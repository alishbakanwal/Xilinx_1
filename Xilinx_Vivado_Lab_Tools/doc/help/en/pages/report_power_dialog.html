Report Power

The Report Power dialog box lets you generate a report of the power consumption of the design. An important consideration for accurate power estimation is that the operating parameters and design switching activity must be realistically characterized.

The Report Power dialog box includes the following tabs and options for characterizing the design:

  • Common Options
  • Environment
  • Power Supply
  • Switching

Common Options

The common options do not appear in a tab but are present on the dialog box at all times.

  • Results name: Specifies the name to apply to the report in Power Results window in the Vivado® IDE.
  • Output text file: Specifies the name of an ASCII file to write the power results.
  • Output XPE file: Specifies the name of an XML file for importing into Xilinx® Power Estimator spreadsheet tool. Refer to Xilinx Power Estimator User Guide (UG440) for more information.

Environment Tab

The Environment tab lets you define the operating environment of the device. These settings relate to the set_operating_conditions Tcl command.

  • Device Settings: Specifies the operating parameters of the target device.
    • Temp grade: Specifies the temperature grade of the target device. Supported values vary by device family.
    • Process: Specifies the manufacturing process corner to analyze. Valid values are "typical" or "maximum".
  • Environment Settings: Specifies the various environmental settings related to the operation of the device or the system it is used in.
    • Output Load: Specifies external capacitance loads driven by the design.
    • Junction temperature: Specifies the device junction temperature. The value is calculated automatically. Enable the checkbox to manually enter a temperature specified in degrees Celsius.
    • Ambient temperature: The ambient temperature of the operating environment in degrees C.
    • Effective Theta-JA: The Theta-JA thermal resistance used for modeling in degrees C/W. The value is calculated automatically, but you can manually enter a value as needed.
    • Airflow: Linear Feet Per Minute (LFM) airflow to be used for modeling cooling. The default setting varies by device family.
    • Heat sink: Specifies the heat sink profile to be used for modeling cooling.
    • Theta-SA: The Theta-SA thermal resistance used during modeling in degrees C/W. The value is determined automatically from the heat sink selected. However, you can manually specify the Theta-SA if you select the custom heat sink.
    • Board selection: The board size to be used for modeling cooling. The valid values are: jedec, small, medium, large, custom.
    • Number of board layers: Estimates the number of layers the system board has. Valid values are: "4to7" for boards with 4 to 7 layers, "8to11" for boards with 8 to 11 layers, "12to15" for boards with 12 to 15 layers, and "16+" for boards with 16 or more layers.
    • Theta-JB: The Theta-JB thermal resistance of the board specified in degrees C/W. The value is calculated automatically, but you can manually enter a value as needed.
    • Board temperature: The board temperature specified in degrees Celsius.

Power Supply Tab

The Power Supply tab lists voltage supply names and their specified values. Supported voltage supply names and their values vary by family. The setting of these voltage supplies also relates to the set_operating_conditions Tcl command.

Switching Tab

The Switching tab lets you define the signal rate and switching probability of the device to be used when performing power estimation on the current design, as well as setting the activity associated with specific device resources. These settings relate to the set_switching_activity Tcl command.

The switching activity of a design affects both the static and dynamic power consumption. The static power is often dependent on logic state transitions, and the dynamic power is directly proportional to the toggle rate.

  • Simulation activity file: Reads a Switching Activity Interchange Format (SAIF) file for use during power analysis or power optimization. The SAIF file will annotate the design nodes with switching activity and the Vivado tool will estimate power appropriately. The Vivado simulator can write an SAIF file, as described at this link in Vivado Design Suite User Guide: Logic Simulation (UG900).
  • Default Activity Settings:
    • Default toggle rate: The default toggle rate to be used in power analysis on the primary inputs of the current design.
    • Default static probability: The default static probability to be used in power analysis on the current design.
  • Enable Rate Settings:
    • BRAM Port Enable: Specify the Static Probability and Toggle Rate for the enable pins (ENARDEN/ENBWREN) of BRAMs.
    • BRAM Write Enable: Specify the Static Probability and Toggle Rate for write enables (WEA/WEBWE) of BRAMs.
    • Bidi Output Port Enable: Specify the Static Probability and Toggle Rate for the enable pin of bidirectional ports.
  • Toggle Rate Settings:
    • Primary Outputs: Specify the Static Probability and Toggle Rate for the output ports of the design.
    • Logic: Specify the Static Probability and Toggle Rate for registers, shift registers, distributed and Block RAMs, LUTs, and DSPs.
    • GTs: Specify the Static Probability and Toggle Rate for GT receivers and transmitters.
  • Constrained Clocks: Lists the clock domains considered for analysis. This is informational only.

Click OK to run the power analysis with the specified values.
