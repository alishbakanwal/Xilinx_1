Report Design Analysis

The Report Design Analysis command provides three modes of operation: timing analysis and physical characteristics of timing paths, complexity analysis of the design to identify timing issues due to routing complexity and LUT distribution, and routing congestion.

The Report Design Analysis dialog box includes the following tabs and common options:

  • Options
  • Advanced
  • Timer Settings
  • Common Options

Options Tab

The Options tab provides the following fields to configure the types of analysis performed by the report:

  • Timing: Performs timing analysis of all the critical paths, or of the specified paths of the design.
    • All paths: A radio button to return timing data for all critical paths in the design.
      • Path delay type: Specifies the type of delay to analyze for all paths. Max delay is for setup analysis of timing paths only. Min delay is for hold analysis only. Choose min_max delay for both setup and hold analysis.
      • Maximum number of paths: Specifies the number of paths returned by the report. By default, the GUI report includes the 10 worst timing paths.
    • Specific paths: A radio button to perform analysis of the specified timing paths.
      Tip: When this option is selected, the entry field to define the timing paths is enabled. You can enter Tcl commands to define the timing path directly, or use the Find Timing Paths command to define the timing paths to report.
    • Extend Analysis: Perform an extended setup analysis of the specified timing paths. The extended analysis includes the path that ends at the start point of the specified timing path, the specified timing path, and the path that starts at the end point of the specified timing path.
      Tip: When Path delay type of min (hold) is specified with the Extend Analysis option, this generates the setup report showing the worst setup paths, and the corresponding hold report for those same paths. This shows hold characteristics with the setup characteristics of the same start and endpoints to make it easier to see if hold and setup fixing are affecting one another.
    • Include logic level distribution: To help identify the longer paths in the design.
      Tip: Remember that logic level distribution is only performed on critical timing paths identified by the tool, or specified by you. If you want the analysis to identify all paths with the greatest number of logic levels in the design, you should use Specific paths to define the timing paths of interest. In addition, you can use the LOGIC_LEVELS property on timing paths to obtain this information.
      • Number of paths: Specifies the number of paths to include in the logic level distribution.
  • Complexity: Enables the complexity analysis reporting the Rent exponent, the average fanout, and a primitive histogram.
    • Cells to analyze: Specify one or more hierarchical modules to perform complexity analysis on. By default the top-level cell of the design is analyzed.
      Tip: When this option is selected, the entry field to specify cells to analyze is enabled. You can enter Tcl commands to specify the cells directly, or use the Specify Top Level Cell command to define the cells.
    • Hierarchical depth: Specify the levels of hierarchy to include in the complexity anlysis.
  • Congestion: Reports the vertical and horizontal routing congestion of a placed or routed design, reported on a per tile basis for the whole design. Congestion analysis is performed on the whole design, even if Complexity analysis is performed on specified cells.

Advanced Tab

  • Write results to file: Writes the results of the Design Analysis report into the specified file. The default is to Overwrite an existing file of the same name. Use Append to append the new report to the specified file.
  • Ignore command errors (quiet mode): Suppresses errors when running the Design Analysis report.
  • Suspend message limits during command: Ignores message limits that are defined when running this report.

Timer Settings Tab

The Timer Settings tab provides the following fields to configure the timing engine prior to generating the report:

  • Interconnect: Specifies the interconnect delays to use in the timing analysis. You can use the full routed delays from an implemented design, estimated delays in the synthesized (or implemented) design, or use no interconnect delays to examine just the logic delays.
  • Speed grade: Changes the speed grade of the device during timing analysis, without actually changing the target part.
  • Multi-corner configuration: Configures the Slow and Fast timing corners in the current design. Variables in the manufacturing process, voltage, and temperature (PVT) determine the delay across the device, and combine to define a timing corner. This option relates to the config_timing_corners Tcl command.
    • Slow: Specifies the type of delay to analyze for the Slow corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
    • Fast: Specifies the type of delay to analyze for the Fast corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
  • Disable flight delays: Ignores the package delay in I/O delay calculations. The flight delay is the package delay that occurs between the package pin and the die pad. This option relates to the config_timing_analysis Tcl command.

Common Options

The common options do not appear in a tab but are present on the dialog box at all times.

  • Results name: Specifies the name to apply to the report in Timing Results window in the Vivado® IDE.
  • Command: Displays the Tcl command with all of the specified options, which is used to generate the report.
  • Open in a new tab: Opens the Timing Summary in a new tab of the Design Analysis window in the Vivado IDE. If this checkbox is not enabled, the new report overwrites the last report that was opened.

Click OK to run the displayed Tcl command, and generate the report.
