Force Clock

This dialog box lets you establish a variable value for a selected signal, wire, or register to be applied at regular intervals, like a clock, for a defined period and duty cycle. The value can be specified in a particular radix format, and can be applied to the object after a specified simulation runtime, and for a specified duration. After the Force Clock command has expired, the object resumes normal operation as defined in your HDL design.
Tip: To apply a fixed value to an object use the Force Constant command.
  • Signal name: Displays the name of the selected HDL object.
  • Value radix: Select the radix format that the value will be specified in.
  • Leading edge value: Define the value to assign to the leading edge of the signal transition.
  • Trailing edge value: Define the value to assign to the trailing edge of the signal transition.
  • Starting after time offset: Specify a time offset to apply before assigning the constant value. The offset should be specified as nanoseconds, and the default offset is 0.
    Important: Both start and cancel time offset values are applied from the current simulation runtime, current_time, at the time the Force Constant command is applied.
  • Cancel after time offset: Specify the time offset to cancel the constant value. The Force Clock command is applied to the object during simulation, until the offset has expired. If no Cancel value is specified, the constant is applied for the duration of simulation runtime.
  • Duty Cycle: The percentage of the specified Period that the signal has the Leading edge value, or the signal is active.
  • Period: The length of the clock pulse, or the duration of one cycle from Leading edge value to Leading Edge value.
