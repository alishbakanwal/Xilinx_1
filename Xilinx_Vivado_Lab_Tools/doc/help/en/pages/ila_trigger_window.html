ILA Trigger Setup

This window is one of five windows displayed in the Hardware ILA dashboard by default. The ILA Trigger window lets you define the trigger condition for the selected debug core, like boolean trigger equations and edge transition triggers for specified probes. The content of the ILA Trigger window is determined by the Trigger Mode defined in the ILA Settings window:

  • Basic type triggers
  • Advanced type triggers

Basic Trigger

The basic trigger is a boolean expression that evaluates the values on each debug probe included in the trigger, and triggers the TRIG_OUT signal and the ILA data capture sequence when the expression evaluates to TRUE. The first step in using the basic trigger mode is to decide what ILA debug probes you will include in the trigger definition. For more information on basic triggers see this link in the Vivado Design Suite User Guide: Programming and Debug (UG908). The ILA Trigger windows lists the probes assigned to the trigger in a table that displays the following:
  • Name: Specifies the name of the probe on the ILA debug core.
  • Operator: Specifies the operator used to evaluate the trigger event on the port. The operator can be changed by clicking in the field and selecting an operator from the drop down menu. The list of available operators depends upon the radix and possible values of the probe.
  • Radix: Specifies the radix for the value on the probe. You can specify the radix by clicking in the field and selecting the radix from the drop down menu.
  • Value: Specifies the probe value to use in the trigger expression. The value and the operator determine if the probe comparator evaluates to true or false.
  • Port: Reports the port in the design that the probe is assigned to.
  • Comparator: Lists the specific probe comparator that the trigger is part of.
The trigger comparators of the ILA debug core evaluate the value on each probe according to the specified operator to determine if the trigger expression is true or false. The trigger of the ILA debug core is the result of a global boolean "AND", "OR", "NAND", or "NOR" expression applied to all of the ILA probe trigger comparator results, as defined by the Set trigger condition command of the toolbar menu.
Important: Only probes that are added to the ILA Trigger window participate in the trigger condition. Any probes that are not in the window are not considered as part of the trigger condition.

The toolbar for the BASIC trigger mode includes the following commands:

Search Opens a text field to enter a search pattern to display specific probes of interest.
Add probes Opens the Add Probes dialog box, which lists the probes associated with the ILA debug core. Select probes and click OK to add them to the window.
Remove selected probes Removes selected probes from the ILA Trigger Setup window.
Set trigger condition Lets you define the global trigger condition of the probes.

Advanced Trigger

The Advanced Trigger Settings let you import an existing trigger state machine, or create a new one. Opening or creating a trigger state machine converts the ILA Trigger Setup window to a Text Editor window, with most of the same commands for editing text files. However, the ILA Trigger Setup window adds the Insert probe command () that opens the Insert Probe dialog box, listing the probes associated with the current ILA debug core, and letting you add them to the trigger state machine. Refer to this link in the Vivado Design Suite User Guide: Programming and Debug (UG908) for more information on advanced triggers, and creating the trigger state machine.
