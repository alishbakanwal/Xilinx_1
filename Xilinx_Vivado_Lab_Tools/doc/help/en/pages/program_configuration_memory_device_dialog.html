Program Configuration Memory Device

The Vivado device programmer can indirectly program flash-based configuration memory devices via JTAG, using a special configuration memory file (cfgmem) created with the Write Configuration Memory File command in the Vivado Hardware Manager. The configuration memory device can be programmed using the options specified in this dialog box.

  • Memory Device: Specify the configuration memory device.
    Tip: This field is automatically populated by the configuration memory device associated with the current hardware device through the Add Configuration Memory Device dialog box. However, you can also specify a different configuration memory device if needed.
  • Configuration file: Specify the name and path of the cfgmem file to use for programming the flash memory device.
  • PRM file: Specify the name and path of the log file that is created along with the cfgmem file. This log file includes checksum data when the Write checksum option is enabled in the Write Configuration Memory File.
  • State of non-config mem I/O pins: Specifies the pull-type termination scheme for unused I/O pins in the cfgmem file.
    Important: This should match the BITSTREAM.CONFIG.UNUSEDPIN property specified in the Edit Device Properties dialog box, which has a default value of PULLDOWN. Refer to this link in the Vivado Design Suite User Guide: Programming and Debug (UG908) for more information.
  • Program Operations
    • Address range: Erase, blank check, program, and verify (as directed below) the complete address range of the configuration memory device, or only program the address range required by the cfgmem file.
    • Erase: Erase the configuration memory device as part of the programming cycle.
    • Blank Check: Perform a blank check of the configuration memory device to ensure it was completely erased as part of the programming cycle.
    • Program: Program the configuration memory device with the specified cfgmem file.
    • Verify: Verify that the configuration memory device contents match the cfgmem file.
    • Verify Checksum: Perform a checksum comparison of the data programmed in the configuration memory device against the checksum value specified in the PRM file.
  • SVF Options
    • Create SVF Only (no program operations): Generate an SVF file for remote programming Xilinx devices, and flash configuration memories. SVF is an industry standard file format that is used to describe JTAG chain operations by describing the information that needs to be shifted into the device chain.
      Tip: The SVF file is created in place of actually programming the configuration memory device.
    • SVF File: Specify the name and path of the SVF file to create.

Click OK to program the configuration memory device, or create an SVF file.
