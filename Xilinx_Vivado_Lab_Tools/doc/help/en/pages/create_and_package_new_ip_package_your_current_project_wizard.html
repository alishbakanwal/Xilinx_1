Package Your Current Project

  • IP Location: Specify the directory location to output the packaged IP. You must add this location as an IP repository to the Vivado® IP Catalog to use the new IP in other projects. Refer to this link in the Vivado Design Suite User Guide: Designing with IP (UG896) for more information on using the Repository Manager.
  • Packaging IP in the project: The following options determine how the IP in your current project are packaged as part of the newly created top-level IP.
    • Include .xci files: Includes only the XCI file in the packaged top-level IP. If you select this option, the required output products for all IP used by the top-level IP must be generated when the IP is used in another design. In addition, included IP must be updated to the latest version to generate the output products in the Vivado tools. Refer to this link in the Vivado Design Suite User Guide: Designing with IP (UG896) for more information on upgrading IP.
      Tip: Updating the included IP in your custom IP might create some change management issues for your custom IP.
    • Include IP generated files: Includes all required output products that support the included IP through simulation and implementation with the packaged top-level IP. If you select this option, the newly packaged IP is self-contained and does not need to be upgraded for use in other designs. This option removes any references to the included IP customizations and treats the included IP as hierarchical project source files in the newly packaged top-level IP.
