Edit Configuration Runs Wizard

This page of the wizard lets you define the implementation runs for the different PR configurations in the design. You can define multiple runs for each configuration, using different implementation strategies to see which yields the best results. Each PR configuration requires at least one run to implement the Reconfigurable Module (RM) for the configuration.

The page displays a Configuration Runs table to manage the implementation runs for the project. The table lists the runs by name, and lets you assign the run to a configuration, define the parent of the run, specify the constraints file and the strategy for the run. When the table is empty, a command link is displayed to automatically create configuration runs. Clicking this link is recommended, and automatically sets up all the implementation runs needed for the PR configurations in the design.

Each RM in the design is synthesized separately as an out-of-context (OOC) synthesis run, and then integrated into the top-level design. Synthesis is configured automatically for the PR flow, as you can see in the Design Runs window after creating a partitionDef or RM. Refer to this link in the Vivado® Design Suite User Guide: Partial Reconfiguration (UG909) for more information.

For implementation runs you can configure the parent of each RM separately so that the configuration is implemented as a complete design, or only the RM is implemented in the context of the previously implemented design. The choices are:

  • Synthesis (synth_1): The synthesis DCP will be used for the RM, and the static logic and RMs will be implemented together as a single design.
  • Implementation (impl_1): The implementation results of the parent will be used, with the netlist for the RM integrated into the configuration, and the RM implemented into the existing implementation results to complete the configuration.

The Configuration Runs table has a toolbar with the following commands:

  • Add (): Define a new implementation run for a PR configuration. This command opens a small dialog box listing the parent run, the configuration name, and a space to define the name of the run. No default run name is provided.
  • Remove (): Remove the selected configuration run from the current project.
  • Show PartitionDef Instances (): Toggle the display of columns listing the partitionDefs and RM assignments in the configuration.

Click Next or Skip to Finish to proceed to the Partial Reconfiguration Summary page of the Partial Reconfiguration wizard.
