Hardware ILA

The Hardware ILA window is a Hardware Manager dashboard that acts as a collection for the settings, status, trigger inputs and outputs, captured data samples, and the waveform activity associated with an ILA debug core programmed into the current device. You can use the ILA core to probe specific signals of the design, to trigger on programmed hardware events, and capture data from the Xilinx® FPGA device in real-time.

The default windows associated with the Hardware ILA window include the following:

  • ILA Settings: Provides access to the various settings configuring the ILA debug core.
  • ILA Status: Provides a real-time visual summary of the state of the ILA debug core.
  • ILA Trigger Setup: Provides access to the probes assigned to the ILA debug core, and lets you manage the triggers on those probes.
  • ILA Capture Setup: Lets you define and manage capture equations associated with debug probes when the ILA debug core capture mode is set to basic.
  • ILA Wave: Displays the waveform activity associated with probes on the ILA debug core.

The Hardware ILA dashboard also includes a Dashboard Options feature that can be accessed from a slim button hidden along the left side of the dashboard. Opening the Dashboard Options displays a list with the current device, the debug cores programmed into the device, and the windows associated with each core. The list has checkboxes to let you add or remove a window to the dashboard.

The Hardware ILA window appears in the Vivado® IDE workspace. For more information on the features of workspace windows refer to this link in the Vivado Design Suite User Guide: Using the Vivado IDE (UG893).
