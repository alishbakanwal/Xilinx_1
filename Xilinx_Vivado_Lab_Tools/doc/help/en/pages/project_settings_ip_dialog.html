Project Settings - IP

The Project Settings dialog box allows you to specify values for various settings used throughout the design flow. The settings are grouped according to the steps in the design flow, such as synthesis, simulation, and implementation.

The IP settings include the following tabs to configure the current project:

  • General
  • Repository Manager
  • Packager

General

The General tab includes the following fields:

  • Use Core Containers for IP: Specifies the use of the compressed core container for IP output products to create a single consolidated file for IP management. Refer to this link in the Vivado Design Suite User Guide: Designing with IP (UG896) for more information.
  • Use Precompiled IP Simulation Libraries: Uses pre-compiled simulation libraries to save time and disk space when generating output products. When this feature is disabled, static IP simulation libraries are created for each design.
  • Automatically Generate Simulation Scripts for IP: Generates simulation scripts for individual IP when generating output products. You can disable this option if generating simulation scripts for individual IP takes too much time or you are not interested in simulating individual IP.
  • Generate Log File: Create a log file reporting details of the IP Upgrade process. The default log file name is ip_upgrade.log.
  • Log File Location: Specifies the location of the log file to generate when upgrading IP. The default location is inside the current project, or the last location you specified.
  • Default IP Location: Specifies the default location to write new IP customizations added to the current project. The default location is inside the current project directory structure, but you can specify a location outside of the project structure to facilitate reuse and source management. Refer to this link in the Vivado Design Suite User Guide: Designing with IP (UG896) for more information.
  • IP Cache: The IP cache stores synthesis results for out-of-context IP and lets you leverage those results when creating IP with the same customization options. You can point multiple projects to the same IP Cache location to avoid re-running synthesis for IP customizations that are the same. See this link in the Vivado Design Suite User Guide: Designing with IP (UG896) for more information.
    • Cache scope: Specifies the synthesis cache as disabled, or local to this project, or remote for use by many projects.
    • Cache location: Specifies the directory location for a remote cache.

Repository Manager

The Repository Manager tab lets you specify folders to add to the Vivado IP Catalog as repositories of user-defined or third-party IP. This tab includes the following fields:

Add Opens a file browser to select folders to add as IP repositories. The Vivado tools scan selected folders and open the Add Repository dialog box to summarize the number of IP and Interfaces found.
Remove Removes the selected folder from the repository.
Move up Moves the selected folder up in the displayed list.
Move down Moves the selected folder down in the displayed list.
Important: The tool issues a warning if the IP repositories contain duplicate IP. IP from the last IP repository listed overwrites any earlier IP definitions.

Packager

The Packager tab lets you specify defaults to use when packaging IP. Refer to this link in the Vivado Design Suite User Guide: Creating and Packaging IP (UG1118) for more information.

  • Default Values: Specifies the default values for user-defined IP.
    • Vendor: Specifies the Vendor name to apply for custom-packaged IP.
    • Library: Specifies the Library name to assign to the IP.
      Tip: The Vendor and Library, along with the Name and Version assigned to the specific packaged IP, define the unique VLNV identifier for the IP.
    • Category: Specifies a category for the IP to be sorted into in the Vivado IP Catalog.
    • IP Location: Specifies the location to write packaged IP to. This location should be added to the Repository Manager.
  • After Packaging: Specifies the tool behavior after packaging the IP.
    • Create archive
    • Add IP to the IP Catalog
    • Close IP Packager window
  • Edit IP in IP Packager: Specifies what to do with the Package IP project when finished packaging the IP.
    • Delete project after packaging
  • File Extensions to Filter on Add Directory: Specifies the file extensions of interest when defining IP file groups.
