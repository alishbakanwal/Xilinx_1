Timing Results

The Timing Results window displays the results of the Report Timing Summary and Report Timing commands. The window is divided into two sections. The left side has a tree view showing Summary, Settings, a list of timing checks performed, and various path reports. The right side shows the summary results for the report or for the selected category of the report. Timing values are presented in nanoseconds.

The Timing Results window is a table window, which presents timing results for timing paths and clock objects in column form. To display and hide columns as well as manage column widths, right-click in the header of any column. You can also sort information in the table by clicking once or twice in the column header. See this link in the Vivado® Design Suite User Guide: Using the Vivado IDE (UG893) for more information.

The right-click menu provides access to commands for exploring selected timing paths, such as Schematic to open a schematic view of the selected path or View Path Report to display the Timing Path Report window for details of the path.

The Timing Results window includes toolbars for the Tree view and for the Table view:

  • The timing tree view toolbar includes the following commands:
    Search Opens a search field to enter a search pattern. Timing checks or paths that match the search pattern are displayed.
    Collapse all Collapses the hierarchy of timing checks displayed in the tree view.
    Expand all Expands the hierarchy of timing checks displayed in the tree view.
    Previous Goes back to select a previously selected timing check. Use this command to quickly scroll through the timing checks that you selected to investigate.
    Next Goes forward to select a previously selected timing check. Use this command to quickly scroll through the timing checks that you selected to investigate.
    Show only failing checks Hides timing checks that reported no violations.
    Rerun report Reruns the report_timing_summary orreport_timing command used to generate the original timing report. You can use this command to regenerate the same report after the design is modified or updated.
    Tip: The General Information section of the timing report lists the original command used to generate the timing report.
    Save Saves the timing report in an RPX file, which you can reload later to reevaluate the timing without rerunning the timing analysis.
  • The timing table view toolbar includes the following commands:
    Search Opens a search field to enter a search pattern. Timing checks or paths that match the search pattern are displayed.
    Show only failing paths Hides timing paths that have no violations.
    Remove selected path Removes the currently selected timing path from the timing report.
    Schematic Opens a schematic view containing the selected timing paths.
    Mark Marks the timing path in other windows, such as the Device or Netlist windows, to make them easier to find.
    Tip: The startpoint is marked in green, the endpoint is marked in red, and any through points are marked in yellow.
    Select Clock Paths when Timing Paths are selected Selects the source and destination clock paths along with the datapath.
