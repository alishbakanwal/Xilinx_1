Power Constraints Advisor

The Power Constraint Advisor reports tool-computed switching activity on all control signals in the design. Control signals include enables and resets (Reset, Set, Clear, and Preset). Providing reasonable switching activity ensures the most accurate power measurements.

Review the report table and modify inaccurate switching activity on critical control signals such as inactive enables and reset signals that are asserted for excessive periods of time. The report also includes the fanout for each control signal, which is the number of driven leaf-level primitives. Signals with higher fanout are most important for review and correction, because they are capable of disabling downstream switching of large portions of the design which may result in severe under-reporting of power. Low-fanout signals with inaccurate switching will have less impact and are therefore not as important. The Deassert button can be used to easily configure switching activity of all resets to be inactive.

Click the checkbox next to a listed net to enable that net for editing. You can modify the Static Probability and Toggle Rate fields for a selected net by clicking the pencil (), which indicates the field can be edited. Click OK to apply the modified values to the selected nets using the set_switching_activity Tcl command. The recommended process for using the Power Constraints Advisor is:

  1. Click the Confidence column to sort it so that LOW are in top.
  2. Hold down the Ctrl key and click on Fanout column twice to sort it by descending values.
  3. Review and define new Static Probability and Toggle Rate for all the control nets which are LOW in confidence with fanout greater than 200.
  4. Click OK to apply the constraints to the design and rerun the Report Power command.

The toolbar menu provides access to the following commands:

Search Opens a search field to enter a search pattern. Nets or values matching the search pattern are displayed.
Deassert Control Signal Sets the Static Probability and Toggle Rate for all control signals. The Static Probability is changed to 0 for active high signals, and changed to 1 for active low signals. Toggle Rate is changed to 0 for all control signals.
Tip: This is not applicable for BRAM enable signals.
Reset constraints to default Restores the dialog box to its original values, resetting any modified values.
Schematic Opens a schematic view containing the selected nets.

Examples:

  • Active high reset with Static Probability 0.9 - This indicates the reset is high 90% of the time, meaning the load cells are reset for 90% of the time, which is excessive. Change the switching activity to indicate the reset is inactive; a more realistic condition. If the reset is active high, set the Static Probability to 0 and Toggle Rate to 0; or if the reset is active low, set the Static Probability to 1 and the Toggle Rate to 0.
  • BRAM Enable with Static Probability 0 and Toggle Rate of 0 - This indicates the BRAM is never enabled, which will cause under reporting of power for BRAM. Assign a more reasonable switching activity on the BRAM Enable, such as a 25% enable rate, by setting the Static Probability to 0.25 and the Toggle Rate to 12.5.

Click OK to apply the constraints to the design.
