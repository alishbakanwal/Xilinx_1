ILA Settings

This window is one of five windows displayed in the Hardware ILA dashboard by default. The ILA Settings window provides configuration options for the selected Integrated Logic Analyzer (ILA) debug core programmed into the current device. You can use the ILA core to probe specific signals of the design, to trigger on programmed hardware events, and capture data from the Xilinx® FPGA device in real-time.

  • Trigger Mode: Specify the type of trigger events for the debug core. Setting the trigger mode to a BASIC type trigger or ADVANCED type trigger determines the commands and options that are available in the ILA Trigger Setup window.
    • BASIC_ONLY: The trigger condition is the result of a boolean expression used to evaluate the values on each of the associated ILA probes.
    • BASIC_OR_TRIG_IN: The ILA core is triggered by a boolean expression considering probe values, or by the TRIG_IN port on the core.
    • ADVANCED_ONLY: The ILA core is configured to have advanced trigger capabilities defined in a user-defined Trigger State Machine (TSM).
    • ADVANCED_OR_TRIG_IN: The ILA core is triggered by the TSM or by the TRIG_IN port on the core.
    • TRIG_IN_ONLY: The ILA core is triggered only by the TRIG_IN port on the core.
  • TRIG OUT mode: Specify how or when to transition the TRIG_OUT port on the ILA core. The TRIG_OUT signal can be used as an input trigger mechanism for other ILA debug cores.
    • DISABLED: Disable the TRIG_OUT port on the ILA core. This is the default.
    • TRIGGER_ONLY: Transition the TRIG_OUT port only when the trigger conditions have been satisfied.
    • TRIG_IN_ONLY: Transition the TRIG_OUT only when the TRIG_IN signal transitions. Use this to pass the trigger event along a chain of ILA cores.
    • TRIGGER_OR_TRIG_IN: Transition the TRIG_OUT when either the trigger conditions are satisfied, or the TRIG_IN signal transitions.
  • Capture Mode: Set to ALWAYS or BASIC. Capture and store a data sample on the probes of the debug core ALWAYS during a given clock cycle, or only if the capture condition evaluates to "true" (BASIC). When set to BASIC, the capture condition for each probe can be defined in the ILA Capture Setup window.
  • Number of windows: Specify the number of data windows to store captured sample data. The first data window will be filled with data from the first trigger event, and each subsequent window will be filled upon subsequent triggering events or capture conditions.
  • Window data depth: Define the number of data samples the selected ILA debug core can capture in a data window. The data depth defaults to the MAX_DATA_DEPTH property of the ILA debug core, which is defined when the core is created. The data depth can be specified as an integer which is a power of two, from 1 to the MAX_DATA_DEPTH of the core.
  • Trigger position in window: An integer value related to the Window data depth that lets you capture sample data ahead of the trigger event. For instance, with a data depth of 1024, a trigger position of 100 allows you to capture 100 data samples ahead of the trigger, and the remaining data samples at and after the trigger event.
  • General Settings: Specify the Refresh rate in milliseconds for the current ILA debug core.
