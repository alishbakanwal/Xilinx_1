Define Constraints and Target

This dialog box lets you define a target constraints file for the current constraint set and also lets you create and add new constraints files to the constraint set. Constraints are read in from the files in the order listed. Constraints defined early in the list can be overwritten by constraints defined later in the list. Refer to this link in the Vivado Design Suite User Guide: Using Constraints (UG903) for more information on constraint precedence.
Important: Changes to existing design constraints are saved to the constraints file they originated from. Refer to this link in the Vivado® Design Suite User Guide: Using Constraints (UG903) for more information on entering and saving constraints.
  • Target: Defines the target constraints file for the current constraint set. New constraints are saved to the target file.
  • Add Files: Opens a file browser in which you can select one or more constraint files (.xdc). Selecting multiple files adds the files in sorted order to the file list.
    Tip: Alternatively, you can click and select Add Files.
  • Create File: Opens the Create Constraints File dialog box. You can add constraints to the empty constraints file by editing the file, by defining constraints using the Timing Constraints Wizard, or by modifying the placement and routing of the design and saving the project or constraints.
    Tip: Alternatively, you can click and select Create Files.
  • Remove (): Removes selected files from the current project. The files are removed from the project but are not removed from the hard drive.
  • Move Up: When one or more files have been selected in the file list display area, the Up button () is activated letting you move the selected file or files up in the listed order.
  • Move Down: When one or more files have been selected in the file list display area, the Down button () is activated letting you move the selected file or files down in the listed order.
  • Copy constraints file into project: This checkbox lets you copy the specified constraints file into the sources folder of the local project directory.
