Find Timing Paths

The Find Timing Paths dialog box lets you specify timing paths to analyze. The dialog box includes the following tabs and common options:

  • Targets
  • Options

Targets Tab

The Targets tab lets you define the timing paths of interest by specifying the following fields:

  • Start Points: Specifies the From objects, or start points, for the timing path. Click the Choose start points button () to open the Specify Start Points dialog box, or enter a Tcl command directly in the text field.
    • Transition: Specifies the edge of signals at the start points to be considered for timing analysis. You can specify the rising, falling, or both the rising and falling edges. If a clock object is specified, only the paths launched by the specified edge of the clock are considered as start points.
  • Through Points: Specifies the Through objects, or through points, for the timing path. Click the Choose through points button () to open the Specify Through Points dialog box, or enter a Tcl command directly in the text field.
    • Transition: Specifies the edge of signals at the through points to be considered for timing analysis. You can specify the rising, falling, or both the rising and falling edges.
    • Add through points: Adds a Through field to enter additional through points for the timing path. You can specify individual through points in sequence to define a specific path through the design for analysis.
  • End Points: Specifies the endpoints, or To objects, for the timing path. Click the Choose end points button () to open the Specify End Points dialog box, or enter a Tcl command directly in the text field.
    • Transition: Specifies the edge of signals at the endpoints to be considered for timing analysis. You can specify the rising, falling, or both the rising and falling edges. If a clock object is specified, only the paths launched by the specified edge of the clock are considered as endpoints.

Options Tab

The Options tab provides the following fields to configure the output of the report:

  • Path delay type: Specifies the type of delay to analyze when running the timing report. Max delay is for setup analysis of timing paths only. Min delay is for hold analysis only. Choose min_max delay for both setup and hold analysis.
  • Path Limits: Limits the number of paths and groups to report.
    • Number of paths per group: Specifies the number of paths returned by the report. By default, the report includes the 10 worst timing paths or the worst paths per path group.
    • Number of paths per endpoint: Specifies the number of timing paths to report per endpoint. This number is limited by the number of paths per clock or group.
    • Limit paths to group: Specifies the path groups to evaluate for timing analysis. Any paths that are not in the specified path groups are not analyzed. Click the Browse button () to open the Specify Path Groups dialog box, or enter a Tcl command directly in the text field.
  • Path Display: Controls the way paths are displayed in the resulting report.
    • Display paths with slack greater than: Eliminates paths from the report that are not of interest to you. This setting does not affect the summary report, only the paths listed in the details.
    • Display paths with slack less than: Eliminates paths from the report that are not of interest to you.
    • Sort paths by: Sorts the paths in the Timing report by path group or by slack values.

Command: Displays the Tcl command which is used to find timing paths.

Click OK to return the displayed Tcl command to the Report Design Analysis dialog.
