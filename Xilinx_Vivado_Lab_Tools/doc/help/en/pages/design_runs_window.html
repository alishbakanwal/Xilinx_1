Design Runs Window

The Design Runs window provides commands to manage, launch, and reset runs for synthesis and implementation. It also displays the status and statistics of the synthesis and implementation runs in a table format, with specific columns displaying processing time for the run, completion rates, resource utilization, and timing results.

Tip: To customize the columns in the Design Runs window, right-click a column header to access various menu commands.

The toolbar includes the following commands:

Search Opens a search field to enter a search pattern. The I/O Ports table is searched for the specified value.
Collapse Collapses the listed runs.
Expand Expands the listed runs.
Launch Runs Launches the selected runs. Specify the number of processors to use, and launch on a remote host.
Reset Run Resets a completed or partially completed run, and optionally delete any data associated with the run.
Launch Next Step Launches the implementation run in incremental steps, such as netlist optimization, place, physical optimization, and routing.
Reset to Previous Step Reverts an implementation run to a previous step so it can be modified and rerun.
Create Runs Add new synthesis and implementation runs to the project. You can specify the target part and strategy for the run to use.
Open Run Opens the selected run as a Synthesized or Implemented design.
Show Percentages Displays the utilization of resources as a percentage of the available resources.

A project that supports multiple synthesis and implementation runs also has an active run, or a run that is currently in-memory in the Vivado® Design Suite. The results for the active run display in the Messages, Compilation, Reports, and Project Summary windows. You can change the active run by selecting an inactive run and using the Make Active command from the right-click menu.

Other commands in the right-click menu let you change design run properties and command settings, view reports and logs, and open run directories to see result files and logs on disk.
