Timing Constraints Summary

The Summary page of the Timing Constraints wizard shows the constraints that are defined and ready to be added to the design. The constraints are saved to the target XDC file when you press Finish to close the Timing Constraints wizard. The constraints were applied to the in-memory design when each page was advanced as you moved through each page of the wizard, however, clicking Cancel will cause the in-memory constraints to be lost as well.

  • View Timing Constraints: This opens the Timing Constraints window for you to examine the constraints that were written.
  • Create Timing Summary Report: This runs the Report Timing Summary command for the current design using the newly defined timing constraints. Timing violations will likely be reported if the period or I/O delay constraints that you entered are too difficult to meet.
  • Create Check Timing Report: This runs the Check Timing report for the current design using the newly defined timing constraints.
  • Create DRC Report using only timing checks: This runs the Report DRC command for the current design, applying only the timing checks.

As discussed in the UltraFast™ Design Methodology Guide for the Vivado® Design Suite (UG949), the process of defining timing constraints can be broken down into iterative steps. It might take multiple passes through the Timing Constraints wizard to complete the definition of constraints.
