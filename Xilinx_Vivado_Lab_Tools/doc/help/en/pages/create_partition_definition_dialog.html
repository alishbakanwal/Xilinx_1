Create Partition Definition

Important: You must first define the project as a Partial Reconfiguration (PR) project using the Tools > Enable Partial Reconfiguration command, or by setting the PR_FLOW property on the project to TRUE.

Partial Reconfiguration allows reconfigurable modules (RMs) to be swapped into and out of a Partition Definition (partitionDef) to create a unique configuration of the design based on the combination of the core design and an RM. The PR design flow requires the implementation of each PR configuration, resulting in partial bitstreams for the RMs, but whole bitstreams for each integrated configuration.

The Create Partition Definition dialog box lets you select a hierarchical module in the current project, create a partitionDef object in a PR project, and create a RM from the hierarchical module. The partitionDef defines the partition hierarchy, and the RM fits into that partitionDef.
Important: All instances of the selected hierarchical module in the design will be converted to partitionDefs and assigned RMs.
  • Partition Definition Name: Enter a text string to name the partition definition.
  • Reconfigurable Module Name: Specify the name for the reconfigurable module you are defining.
    Tip: This is not the name of the hierarchical module you selected for the partitionDef. It is the name to assign to the RM that will be created from that hierarchy.

Click OK to create the partitionDef for the selected hierarchical cell, and create a Reconfigurable Module from that hierarchy.
