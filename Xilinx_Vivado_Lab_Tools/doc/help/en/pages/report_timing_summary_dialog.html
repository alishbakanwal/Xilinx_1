Report Timing Summary

The Report Timing Summary dialog box lets you generate a comprehensive sign-off quality timing report that examines the design from different perspectives. The Timing Summary Report includes timing checks, clock definitions, intra-clock timing paths with the same source and destination clock, inter-clock timing paths that cross clock domains, path group definitions, and detailed timing paths similar to the Report Timing command.

Important: You can only use the Timing Summary report for timing sign-off after the design is completely routed.

By default, the Report Timing Summary dialog box reports the 10 worst timing paths per clock domain or per path group. You can change this to return more or fewer timing paths per path group by setting the Maximum number of paths per clock or path group option. In addition, the Timing Summary reports one path per endpoint object by default. This means that a single endpoint can have multiple failing timing paths that are not reported by default. However, you can also change this by setting the Maximum number of worst paths per endpoint option. These two options interact to determine how many failing timing paths are reported per endpoint and per path group.

The Report Timing Summary dialog box includes the following tabs and common options:

  • Options
  • Advanced
  • Timer Settings
  • Common Options

Options Tab

The Options tab provides the following fields to configure the output of the report:

  • Path delay type: Specifies the type of delay to analyze when running the timing report. Max delay is for setup analysis of timing paths only. Min delay is for hold analysis only. Choose min_max delay for both setup and hold analysis.
  • Report Unconstrained paths: Includes timing on unconstrained paths in the current design. By default, the report only includes constrained paths.
  • Report datasheet: Adds setup and hold times of input I/Os in relation to clocks, max/min delays from clocks to output pads, andskews of input/output buses. This information is also found in the Report Datasheet command.
  • Maximum number of paths per clock or path group: Specifies the number of paths returned by the report. By default, the report includes the 10 worst timing paths or the worst paths per path group.
  • Maximum number of worst paths per endpoint: Specifies the number of timing paths to report per endpoint. This number is limited by the number of paths per clock or group.
  • Display paths with slack less than: Eliminates paths from the report that are not of interest to you. This setting does not affect the summary report, only the paths listed in the details.
    Tip: Click Use default to restore the default setting.
  • Significant digits: Configures the significant digits for values reported in the timing report. The default is 3.

Advanced tab

The Advanced tab provides the following fields:

  • Report from cell: Runs the report from the level of the specified hierarchical cell instance. By default, the report applies to the whole design. Only paths that start, end, or are fully contained inside the specified cell are reported.
  • Show input pins in path: Includes input pins for additional detail in the timing path description returned by the report.
  • Report unique pins: Only reports one timing path through each unique set of pins.
  • Write results to file: Writes the results of the Timing Summary report into the specified file. The default is to Overwrite an existing file of the same name. Use Append to append the new report appended to the specified file.
  • Ignore command errors (quiet mode): Suppresses errors when running the Report Timing Summary command.
  • Suspend message limits during command: Ignores any message limits that may be defined when running this report.

Timer Settings Tab

The Timer Settings tab provides the following fields to configure the timing engine prior to generating the report:

  • Interconnect: Specifies the interconnect delays to use in the timing analysis. You can use the full routed delays from an implemented design, estimated delays in the synthesized (or implemented) design, or use no interconnect delays to examine just the logic delays.
  • Speed grade: Changes the speed grade of the device during timing analysis, without actually changing the target part.
  • Multi-corner configuration: Configures the Slow and Fast timing corners in the current design. Variables in the manufacturing process, voltage, and temperature (PVT) determine the delay across the device, and combine to define a timing corner. This option relates to the config_timing_corners Tcl command.
    • Slow: Specifies the type of delay to analyze for the Slow corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
    • Fast: Specifies the type of delay to analyze for the Fast corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
  • Disable flight delays: Ignores the package delay in I/O delay calculations. The flight delay is the package delay that occurs between the package pin and the die pad. This option relates to the config_timing_analysis Tcl command.

Common Options

The common options do not appear in a tab but are present on the dialog box at all times.

  • Results name: Specifies the name to apply to the report in Timing Results window in the Vivado® IDE.
  • Command: Displays the Tcl command with all of the specified options specified, which are used to generate the report.
  • Open in a new tab: Opens the Timing Summary in a new tab of the Timing Results window in the Vivado IDE. If this checkbox is not enabled, the new report overwrites the last report that was opened.
  • Open in Timing Analysis layout: Changes the current layout view of the Vivado IDE to the Timing Analysis layout view. See this link in the Vivado Design Suite User Guide: Using the Vivado IDE (UG893) for more information.

Click OK to run the displayed Tcl command, and generate the report.
