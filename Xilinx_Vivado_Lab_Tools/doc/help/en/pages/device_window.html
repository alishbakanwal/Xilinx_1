Device Window

The Device window displays a graphical representation of the die for the target device. The Device window shows device resources, such as I/O buffers, block RAMs, DSPs, configurable logic blocks (CLBs), clock buffers, clock regions. You can zoom into specific device resources, perform interactive and automatic placement of the netlist, explore routing, and perform interactive routing from within the Device window.

For interactive placement, you can select objects in the Netlist or I/O Ports window and drag and drop them onto a device resource in the Device window. For controlling placement of objects the toolbar has the following commands:

Cell Drag and Drop Mode Controls whether selected cells will be assigned to a Pblock region they are dropped into, to a SITE resource and assigned a LOC property, or assigned to a BEL resource with a BEL property. Refer to this link in the Vivado Design Suite User Guide: Implementation (UG904) for more information.
Draw Pblock To place cells into a Pblock region, you must first define one with this command. Pblocks let you define regions of the device to place specific elements of the design. This technique is called floorplanning, and it can help you achieve timing closure on challenging designs. Refer to this link in the Vivado Design Suite User Guide: Design Analysis and Closure Techniques (UG906) for more information on floorplanning.
Place Port Modes Defines how you will place selected ports: in an I/O Bank, in an area, or sequentially. Refer to this link in the Vivado Design Suite User Guide: I/O and Clock Planning (UG899) for more information on placing I/O ports.
Autocheck I/O Placement Automatically runs I/O placement design rule checks during interactive placement to prevent errors.
Routing Resources Displays much greater detail of the device, including the various routing resources used for routing the connections in the design. In this detailed view, you can select nets and trace their routing across the device. You can also perform interactive routing of critical nets in the design. Refer to this link in the Vivado Design Suite User Guide: Implementation (UG904) for more information.

You can pan and zoom to display the level of detail needed to see specific device resources. The toolbar menu has the following commands for controlling what is displayed.

Previous Restore the previous view of the Device window, including zoom-level and position.
Next Return from the previous view.
Tip: This command is only available after using the Previous command.
Zoom In Display more detail of the device resources.
Zoom Out Display more area of the device.
Zoom Fit View the whole device.
Select Area Draw a rectangle to select objects in the Device window.
Fit Selection Zoom into and center the Device window around the currently selected objects.
Autofit Selection Automatically pan and zoom the Device window to center an object that has been selected in another window.
The Device window also supports the use of mouse strokes to zoom the display. Click and hold the left mouse button, and draw a line or box to zoom into or out of the device. You can use the scroll bars shown on the right side or bottom of the Device window to scroll the display and see a new area.
Tip: The scroll bars are only visible when the display area is zoomed into the device.

The Device window appears in the Vivado® IDE workspace. For more information on workspace windows refer to this link in the Vivado Design Suite User Guide: Using the Vivado IDE (UG893).
