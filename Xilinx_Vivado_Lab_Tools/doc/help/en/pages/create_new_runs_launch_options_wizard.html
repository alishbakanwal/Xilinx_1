Launch Options

The Launch Options page lets you define the actions to take after creating the synthesis or implementation runs.

  • Launch Directory: Specify the directory to write the run files and results into for the defined runs.
    Important: The default launch directory is inside the current project. If you define a location outside of the current project structure, it can create problems for source control and design portability.
  • Launch runs on local host: Launch the newly created runs immediately on the local host.
    • Number of jobs: Specify the number of design runs to launch concurrently on the local host. The synthesis run must complete prior to launching a dependent implementation run.
  • Launch runs on remote host: This option is only available on Linux machines and launches the new runs on a specified remote computer.
    • Configure Hosts: This launches the Vivado Options dialog box to let you configure the remote computers available on your network.
  • Launch runs using LSF: This option is only available on Linux operating systems and lets you use Load Sharing Facility (LSF) to manage runs on a distributed processing system.
    • Configure LSF: This launches the Vivado Options dialog box to let you configure the bsub command options to launch runs under the control of LSF.
  • Generate scripts only: Generate the scripts for the synthesis or implementation run, but do not launch the run at this time. The script file is written to the run directory and can be launched manually or run in a batch process.
  • Do not launch now: Only create the new synthesis and implementation runs. Do not launch them or create scripts at this time.
