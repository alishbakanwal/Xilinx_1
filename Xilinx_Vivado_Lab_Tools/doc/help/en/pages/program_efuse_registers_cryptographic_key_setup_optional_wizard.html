Cryptographic Key Setup (Optional)

For UltraScale® or UltraScale+® devices, this page of the Program eFUSE Registers wizard lets you program the FUSE_KEY and FUSE_RSA registers of the current device. The Advanced Encryption Standard (AES), or obfuscated AES, and RSA encryption keys will be read from the specified key file and written to the 256-bit registers to enable decrypting the bitstream when programming the device.

The implemented design must have encryption properties assigned prior to generating the bitstream with the write_bitstream command. You can add ENCRYPTION properties to the design using the Encryption page of the Edit Device Properties dialog box in the Vivado IDE.

  • Enable AES key programming:
    • AES key file (.nky): Specify the file name and path of the encryption key file. The NKY file can contain an AES key, an obfuscated AES key, and an RSA key.
      Tip: By default, this file is written to the implementation run directory at the time an encrypted bitstream is generated.
    • AES key: Displays the AES encryption key value, or the obfuscated encryption key value that is read from the file specified above. The check box lets you enable the AES encryption for the device.
    • RSA key: Displays the RSA encryption key value that is read from the file specified above. The checkbox lets you enable the RSA encryption for the device.
Programming a device with an encrypted bitstream is a two-step process that requires first programming the encryption key into the BBR or eFUSE registers, and then programming the encrypted bitstream into the device: 
program_hw_devices -key {efuse} [current_hw_device]
program_hw_device [current_hw_device]
