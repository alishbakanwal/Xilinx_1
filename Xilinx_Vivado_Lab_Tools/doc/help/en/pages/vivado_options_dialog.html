Vivado Options

The Vivado Options dialog box lets you configure tool-level options that control the actions and behavior of the Vivado® Design Suite. The options are separated into the following categories :

  • General: Configures general aspects of the tool, such as launch behaviors, default directories, and auto-save actions.
  • Colors: Defines the color schemes used by the tool, such as foreground, background, and the colors of various design objects displayed in the Vivado IDE.
  • Selection Rules: Specifies the selection behavior for objects when clicking the mouse in different windows, such as the Device or Package windows. This can be very helpful when you have trouble selecting a specific object in a dense window display.
  • Shortcuts: Defines keyboard shortcuts to use for specific actions in the tool.
  • Strategies: Defines and manages strategies for synthesis and implementation runs.
  • Remote Hosts: Configures synthesis and implementation runs for distributed processing through LSF or on remote computers on your network. Refer to this link in the Vivado Design Suite User Guide: Implementation (UG904) for more information.
    Tip: This option is only available on Linux operating systems.
  • Text Editor: Defines the Text Editor to use when opening design source files, Tcl scripts, and other text files. These options also lets you define the standard fonts to use and other features related to the text editor.
  • Window Behavior: Configures the message windows that the Vivado Design Suite displays to warn or ask about specific actions. These confirmation messages can be disabled and enabled from this area.

When you click OK, the options are written to the vivado.ini file and are used each time the tool is launched.
