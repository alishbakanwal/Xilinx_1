Primary Clocks

Understanding Primary Clocks

Figure: Example of a primary clock:



The clk signal controls the launch and capture time of the FD0 and FD1 registers. Creating a clock on the clk input port ensures that the timer checks the setup and hold slack on the timing path from FD0/C to FD1/D. For example: 
create_clock -name clk -period 6.667 [get_ports clk]

The Constraints for Pulse Width Check Only table lists primary clocks that are only needed for pulse width analysis and not setup/hold/recovery/removal. The pulse width checks cover min_period, max_period, min_low_pulse_width, min_high_pulse_width, and skew on primitive clock pins. By default, these constraints are unselected in the wizard.

Understanding the Primary Clocks Page

  • Recommended Constraints: The Timing Constraints wizard identifies candidates for the primary clocks of the design and recommends constraints to define these clocks. Details such as the clock period and phase are left for you to define.
  • Constraints for Pulse Width Check Only: These are clock identified as needed for performing pulse width checks. Pulse width checks do not affect synthesis or implementation but must be run prior to generating a bitstream for the design. Refer to this link in the Vivado Design Suite User Guide: Design Analysis and Closure Techniques (UG906) for more information.
  • Tcl Command Preview: Displays the create_clock commands used to define the primary clocks as listed in Recommended Constraints. These are the constraints that will be added to the design.
  • Existing Constraints: Displays currently defined primary clock constraints.

Each of the preceding sections also has a toolbar menu that provides access to one or more of the following commands:

Search Displays a text entry field to enter a search string.
Select All Selects all recommended constraints.
Edit selected row Opens the Primary Clock Constraints dialog box to define attributes of the clock. You can also edit these attributes by clicking directly on the field in the Recommended Constraints list.
Clock Networks Displays a sub-menu to provide access to the Report Clock Networks command to generate a new clock network report and the View Clock Networks command to open an existing report.
Tip: The clock networks are reported in the Clock Networks window.
