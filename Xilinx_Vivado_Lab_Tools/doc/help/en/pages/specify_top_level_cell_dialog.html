Specify Top Level Cell

The Specify Top Level Cell dialog box lets you define a query to find cell objects in the current design.

The dialog box is divided into two sections:

  • Query
  • Results

Query

  • Options: Provides three unlabeled drop-down menus listing Property, Operator, and Value sets to filter objects in the current design.
    • Property: Lists either the NAME or IS_PRIMITIVE property of cells.
    • Operator: String or boolean operators are listed depending on the Property you have specified.
    • Value: Predefined values are listed for known or available values of the specified Property, or can be a string or value that you enter directly. You can use the question mark (?) and asterisk (*) characters as wildcards in the value field.
    • Add a new criterion (): Adds a new row to the Properties section.
    • Remove this criterion (): Removes the selected row from the Properties section.
    Tip: When there are multiple rows in the Properties section, you must also specify the logical relationship between the properties as either AND or OR. AND returns objects that match all of the specified property values. OR returns objects that match any of the specified property values.
  • Regular Expressions: Defines the Value field as a regular expression (regexp) search pattern. See this link for more information on regular expression syntax.
  • Ignore case: Applies to regular expressions searches only. Search expressions are generally case-sensitive.
  • Search hierarchically: Searches through each level of the design hierarchy. The default search occurs on the current instance, or the top-level of the design hierarchy. The availability of this option is dependent on the type of object specified in the Find field.
  • Of Objects: Limits the Find search to a list of selected objects, rather than searching the whole design. For example, find the cells connected to specific nets, or assigned to specific Pblocks in the design. Use the browse button () to open the Of Objects - Find dialog box, or manually type in the appropriate Tcl command.
  • Find: This button runs the query defined by the Options fields, and returns findings to the Results section of the dialog box.

Results

The Results section has two list boxes with a toolbar menu between them. The list box on the left displays the results returned by the search query you have defined. The list box on the right contains the specific cells you have selected to return from the Specify Top Level Cells dialog box.

The toolbar menu includes the following commands to move cell objects between the left and right side:
Move right Move one or more selected cells from the left side list box to the right side list box.
Move left Move one or more selected cells from the right side list box to the left side list box.
Move all objects right Move all the cell objects from the left side list box (returned) to the right side list box (selected).
Move all objects left Move all the cell objects from the right side list box to the left side list box.
Tip: Timing path startpoints are marked in green, the endpoints are marked in red, and any through points are marked in the selected color.

After you have moved the needed cells from the left list box to the right list box, you can click OK to close the dialog box and return the selected cells.
