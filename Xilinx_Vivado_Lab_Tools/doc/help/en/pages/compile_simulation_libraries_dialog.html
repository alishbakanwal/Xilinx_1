Compile Simulation Libraries

The Vivado® Design Suite includes a set of simulation libraries for use with the Vivado simulator that support the behavioral, functional, and timing simulation of a design targeting a Xilinx® device. However, you must compile these libraries for use with a supported third-party simulator.

The Compile Simulation Libraries dialog box lets you specify the simulator, language preference, and installation information related to your simulator.

  • Simulator: Specifies the third-party simulator to compile libraries for. The list of available simulators is dependant on the operating system you are running. Some simulators are only available on Linux, and will not be listed when running on Windows.
  • Language: Specifies the simulator language to output. Some simulators are language specific, while other support both Verilog and VHDL. For multi-language simulators, both Verilog and VHDL libraries can be compiled.
  • Library: Specifies the Xilinx simulation library to compile. Refer to this link in Vivado Design Suite User Guide: Logic Simulation (UG900) for more information.
  • Family: Specifies the device family to compile simulation libraries for. This opens the Family Options dialog box to choose device families to target in order to restrict the size of the compiled library.
  • Compiled library location: Directory path for saving the compiled library results.
  • Simulator executable path: Specifies the directory to locate the simulator executable. This is required if the target simulator is not defined in the $PATH, or to override the path definition.
  • Miscellaneous options: Use this to specify additional options for the compile_simlib Tcl command.
  • Overwrite the current pre-compiled libraries: Replace existing libraries at the same specified location.
  • Compile 32-bit libraries: Use this to specify 32-bit executable models. By default the libraries are compiled in 64-bit mode.
  • Verbose: Suspend any current message limits and return all messages from the Tcl command.
  • Command: Displays the Tcl command to compile the simulation libraries as configured in this dialog box. You can edit this command directly to change the command that will be run when you click OK.
