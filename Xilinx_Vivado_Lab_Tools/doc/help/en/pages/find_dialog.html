Find

The Find dialog box lets you define a query to locate objects in the current elaborated, synthesized, or implemented design. The query searches for netlist and device objects using specified properties on the objects.

  • Results name: Specifies the name of the results displayed in the Find Results window.
  • Find: Specifies the type of first class objects to find in the current design. The objects are listed as Netlist objects and Device objects.
  • Properties: Provides three unlabeled drop-down menus listing Property, Operator, and Value sets to filter objects in the current design.
    • Property: The listed properties depend on the object type you specified in the Find field. Click More Properties to add to the listed properties.
    • Operator: The listed operators depend on the specified Property and can be a string operator, a boolean function, or a numerical operator.
    • Value: The listed values are defined from the known or available values for the specified Property or can be a string or value that you enter directly. You can use the question mark (?) and asterisk (*) characters as wildcards in the value field.
    • Add a new criterion (): Adds a new row to the Properties section.
    • Remove this criterion (): Removes the selected row from the Properties section.
    Tip: When there are multiple rows in the Properties section, you must also specify the logical relationship between the properties as either AND or OR. AND returns objects that match all of the specified property values. OR returns objects that match any of the specified property values.
  • Regular Expressions: Defines the Value field as a regular expression (regexp) search pattern. See this link for more information on regular expression syntax.
  • Ignore case: Applies to regular expressions searches only. Search expressions are generally case-sensitive.
  • Search hierarchically: Searches through each level of the design hierarchy. The default search occurs on the current instance, or the top-level of the design hierarchy. The availability of this option is dependent on the type of object specified in the Find field.
  • Display unique nets: Shows only the top-level net segment for each net found in the hierarchy. This option is only available when searching for NET type objects.
  • Of objects: Limits the Find search to a list of selected objects, rather than searching the whole design. For example, find the package pins of specific I/O ports, or find the nets on specific cell pins. Use the browse button () to open the Of Objects - Find dialog box, or manually type in the appropriate Tcl command.
  • Command: Displays the Tcl commands that are run when you press the OK button. You can use the displayed command to help create Tcl scripts using the Vivado® IDE.
  • Open in a new tab: Opens the results in a new tab of the Find Results window. If this checkbox is not enabled, the new results overwrites the last search results opened.

Click OK to run the search for the specified objects and open the Find Results window.
