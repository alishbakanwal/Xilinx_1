[Customize Commands] ダイアログ ボックス

[Customize Commands] ダイアログ ボックスでは、カスタム Tcl コマンドまたはプロシージャを Vivado IDE にショートカット キーに追加して、これらのコマンドをすばやく実行できるようにします。追加されたコマンドは、[Tools] → [Customize Commands] から使用できます。

ダイアログ ボックスには、左側にカスタム コマンドのリスト、右側にコマンドの詳細が表示されます。

  • [Custom Commands] : [OK] をクリックすると、Vivado IDE に追加されるコマンドのリストが表示されます。リストには、前に追加したカスタム コマンドと今回追加したコマンドが含まれます。
    Add 新しいカスタム コマンドのメニュー名を入力できます。新しいコマンドは、Enter キーを押すとリストの一番下に追加されます。
    Remove リストから選択したカスタム コマンドが削除されます。
    Move up 選択したコマンドをリストの上方向に移動します。
    ヒント: コマンドの順番は、Vivado IDE の [Customize Commands] メニューに表示される順番です。
    Move down 選択したコマンドをリストの下方向に移動します。
  • [Edit Custom Command] : ダイアログ ボックスの右側で現在選択したコマンドの詳細をカスタマイズできるようになります。
    • [Menu Name] : Vivado IDE メニューに表示されるコマンド名を指定します。
    • [Shortcut] : 現在のコマンドのショートカット キーを指定します。
      • [Add] : 現在のコマンドに使用するショートカット キーを入力できる [Add Shortcut] ダイアログ ボックスが開きます。[OK] をクリックすると、現在のコマンドにショートカットを割り当てることができます。
        重要: 入力したショートカット キーの組み合わせが既に別のコマンドに割り当てられている場合は、[Shortcuts currently used by] の下にそれがレポートされます。ショートカットを変更して [OK] をクリックすると、現在のコマンドにショートカットを割り当てることができます。
      • [Remove] : 現在のコマンドに定義されたショートカット キーが削除されます。
    • [Run Command] : 実行する Tcl proc (プロシージャー) の名前を指定します。Tcl proc はその Tcl ソース ファイルを読み込んで、Vivado Design Suite にあらかじめ読み込んでおく必要があります。Tcl スクリプトの読み込みおよび実行については、『Vivado Design Suite ユーザー ガイド : Tcl スクリプトの使用』 (UG894) のこのセクションを参照してください。
      • [Define Args] : [Define Command Arguments] ダイアログ ボックスが開きます。Tcl proc はコマンドが実行されるときに引数を必要としたり、引数をオプションで受け入れたりすることがあります。これらの引数のデフォルト値を定義したり、引数の名前を定義して、カスタム コマンドが実行されたときに通知されるようにします。
    • [Source Tcl file] : [Run Command] を指定する別の方法で、コマンドが実行されたときに読み込む Tcl ファイルを指定します。この場合、ファイルが読み込まれたときにスクリプトが実行されるので、Tcl proc を定義する必要はありません。
    • [Toolbar Options] : Vivado IDE のメイン ツールバーにオプションで Tcl proc を表示する場合の設定を作成します。
      • [Add to the toolbar] : このチェック ボックスをオンにすると、ツールバーにカスタム コマンドを表示できます。
      • [Tooltip] : ツールバーのコマンド アイコンの上にカーソルを置くと表示されるテキストです。
      • [Icon file path] : ツールバーにアイコンとして表示する小さな (16x16 ピクセル) JPG または PNG 画像ファイルを指定できます。
[OK] をクリックすると、新しいカスタム コマンドが [Customize Commands] メニューに追加されます。
