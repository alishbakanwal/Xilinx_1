[Generate Output Products] ダイアログ ボックス

[Generate Output Products] ダイアログ ボックスには、デザイン内の IP コアまたはブロック デザインがリストされ、それらの IP に必要な出力ファイルを生成する方法を指定できます。出力ファイルには、インスタンシエーション テンプレート、HDL、制約、シミュレーション ファイルなどが含まれます。 アウト オブ コンテキスト合成がオンになっている場合は、合成済みネットリスト (DCP) と、サードパーティ合成ツール用のブラックボックスを推論するためのスタブ ファイルも作成されます。

このダイアログ ボックスでは、デザインおよびブロック デザイン内の IP をいつどのように合成するかも指定できます。Vivado® Design Suite では、グローバル合成フローとデフォルトのアウト オブ コンテキスト デザイン フローの両方がサポートされています。詳細は、『Vivado Design Suite ユーザー ガイド : デザイン フローの概要』 (UG892) のこのセクションを参照してください。

  • [Preview] : 現在のデザインの IP すべて、または [Generate Output Products] コマンドを使用する前に選択した IP のいずれかが表示されます。これらは、このコマンドの影響を受ける IP です。
  • [Synthesis Options] : リストされる IP に対するアウト オブ コンテキスト (OOC) のサポート レベルを示すオプションです。
    • [Global] : リストされた IP が残りの最上位デザインを使用して合成されます。これにより、リストされる IP のアウト オブ コンテキスト フローは無効になります。
    • [Out of context per IP] : リストされた IP が最上位デザインとは別にアウト オブ コンテキストで合成されます。これにより、リストされている IP それぞれに対して合成 run が作成され、これらの IP の合成ネットリストが別々に管理されるようになります。
    • [Out of context per Block Design] : ブロック デザインは最上位デザインから独立してアウト オブ コンテキストで合成されますが、ブロック デザインで使用される IP はすべてブロック デザインの一部として合成されます。
      ヒント: Vivado IP インテグレーター ブロック デザインの出力ファイルを生成する場合にのみ使用できます。
  • [Run Settings] : 複数の合成 run を実行する際に使用するプロセッサの数を指定します。

ツールバー メニューには、次のコマンドが含まれます。

Search 検索パターンを入力する検索フィールドが開きます。[I/O Ports] の表から指定した値が検索されます。
Collapse All リストされた IP の展開が閉じられます。
Expand All リストされた IP が展開されます。

[Apply] をクリックすると、出力ファイルの生成なしで IP がコンフィギュレーションされ、[Generate] をクリックすると、出力ファイルが指定どおりに生成されます。 出力ファイルを後で生成する場合は [Skip] をクリックします。

ヒント: 既存 IP の出力ファイルを生成する場合は、[Skip] の代わりに [Cancel] を使用してください。[Skip] または [Cancel] をクリックした場合は、[Generate Output Products] ダイアログ ボックスの設定への変更は保持されません。
