[Add Sources] ページ

[Add Sources] ページでは、既存の RTL ファイルまたは MEM ファイルを現在のプロジェクトに追加したり、新しい空のファイルを作成できます。

  • [Add Files] : ファイル参照ダイアログ ボックスが開き、プロジェクトに含めるデザイン ソース ファイルを任意の数だけ選択できます。複数のファイルを選択すると、指定した順番でファイル リストに追加されます。
    ヒント: をクリックして [Add Files] をクリックしても同じ操作を実行できます。
  • [Add Directories] : ファイル参照ダイアログ ボックスが開き、デザイン ソース ファイルを検索するディレクトリを任意の数だけ選択できます。指定したディレクトリ内でデザイン ソース ファイルが検索されて現在のプロジェクトに追加されます。
    ヒント: をクリックして [Add Directories] をクリックしても同じ操作を実行できます。
  • [Create File] : [Create Source File] ダイアログ ボックスが開き、ファイル タイプ、ファイル名、ファイルを保存するディレクトリを指定できます。空のソース ファイルが作成されて、プロジェクトに追加されます。新規作成したソース ファイルは、Add Sources ウィザードの終了時にモジュール エディターで開きます。詳細は、[Define Module] ダイアログ ボックスまたは『Vivado® Design Suite ユーザー ガイド : システム レベル デザイン入力』 (UG895) のこのセクションを参照してください。
    ヒント: をクリックして [Create Files] をクリックしても同じ操作を実行できます。
  • [Remove] () : 現在のプロジェクトから選択したファイルが削除されます。ファイルは現在のプロジェクトから削除されますが、ハード ドライブからは削除されません。
  • [Move Up] : ファイル リストでファイルが選択されている場合、上矢印ボタン () が使用できるようになり、リスト内でファイルを上に移動できます。デザイン ファイルはこの順序でデザインに読み込まれて、コンパイルされます。コンパイルの順序および [Hierarch Update] コマンドの詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。
  • [Move Down] : ファイル リストでファイルが選択されている場合、下矢印ボタン () が使用できるようになり、リスト内でファイルを下に移動できます。
  • [Scan and add RTL include files into project] : オンにすると追加した RTL ファイルがスキャンされます。また、参照される Verilog のインクルード ファイルがローカルのプロジェクト ディレクトリに追加されます。
  • [Copy sources into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定したデザイン ソース ファイルをコピーできます。ローカルおよびリモート ソースの使用に関する詳細は、『Vivado® Design Suite ユーザー ガイド : システム レベル デザイン入力』 (UG895) のこのセクションを参照してください。
  • [Add sources from subdirectories] : [Add Direcotories] オプションを使用して指定したディレクトリのサブディレクトリに含まれるデザイン ソース ファイルも追加します。
  • [Target Language] : RTL プロジェクトで使用されるターゲット言語を定義します。デザインに追加された IP はターゲット言語でサポートされる場合に処理されます。
  • [Simulator Language] : デザインの検証に使用するシミュレータでサポートされる HDL 言語を指定します。シミュレーション用に IP 出力ファイルが生成されるときの言語およびサードパーティのシミュレーション スクリプトで使用するファイル タイプを指定します。

[Next] をクリックして [Add Existing IP] ページに進みます。
