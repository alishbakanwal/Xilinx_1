[Add Existing IP] ページ (オプション)

[Add Existing IP] ページでは、既存のカスタマイズ済み IP ファイルを指定して現在のプロジェクトに追加できます。使用できる既存 IP のフォーマットは、Vivado® Design Suite ネイティブ フォーマット (XCIX または XCI) または CORE Generator™ でサポートされるレガシー IP 形式 (XCO) です。デザインの開発時に Vivado IP カタログから IP を直接追加できます。既存 IP を直接プロジェクトまたはデザインに追加することもできます。

  • [Add Files] : ファイル参照ダイアログ ボックスが開き、プロジェクトに含める既存の IP ファイル (.xcix、xci、.xco) を任意の数だけ選択できます。複数のファイルを選択すると、指定した順番でファイル リストに追加されます。
    ヒント: をクリックして [Add Files] をクリックしても同じ操作を実行できます。
  • [Add Directories] : ファイル参照ダイアログ ボックスが開き、既存の IP ファイル (.xcix、xci、.xco) を検索するディレクトリを任意の数だけ選択できます。指定したディレクトリ内で IP ファイルが検索されて現在のプロジェクトに追加されます。
    ヒント: をクリックして [Add Directories] をクリックしても同じ操作を実行できます。
  • [Remove] () : 現在のプロジェクトから選択したファイルが削除されます。ファイルは現在のプロジェクトから削除されますが、ハード ドライブからは削除されません。
  • [Move Up] : ファイル リストでファイルが選択されている場合、左側のメニューから青色の上方向矢印ボタン () が使用できるようになり、リスト内でファイルを上に移動できます。デザイン ファイルはこの順序でデザインに読み込まれて、コンパイルされます。コンパイルの順序および [Hierarch Update] コマンドの詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。
  • [Move Down] : ファイル リストでファイルが選択されている場合、左側のメニューから青色の下方向矢印ボタン () が使用できるようになり、リスト内でファイルを下に移動できます。
  • [Copy sources into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定した IP ファイルをコピーできます。

[Next] をクリックして [Add Constraints] ページに進みます。
