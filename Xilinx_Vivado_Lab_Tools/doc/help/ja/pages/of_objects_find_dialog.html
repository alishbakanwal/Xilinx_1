[Of Objects - Find] ダイアログ ボックス

このダイアログ ボックスでは、そのオブジェクトに指定されたプロパティを使用して、ネットリストおよびデバイス オブジェクトを見つけるための条件を定義できます。この検索条件の結果で [Find] ダイアログ ボックスの -of_objects オプションが定義されます。
ヒント: このダイアログ ボックスでは、入れ子条件を付けてほかのオブジェクトに関連するオブジェクトを見つけることができます。たとえば、特定の I/O バンクの特定の I/O ポートのパッケージ ピンを検出できます。
  • [Find] : 現在のデザインで検出するファースト クラスのオブジェクトのタイプを指定します。オブジェクトは、[Netlist] オブジェクトおよび [Device] オブジェクトとしてリストされます。使用可能なオブジェクトのリストは、その親ダイアログ ボックスで減らすことができます。
  • [Criteria] : ドロップダウン リストからプロパティ、演算子、および値を選択し、現在のデザイン内のオブジェクトを検索します。
    • [Property] : リストされる演算子は、指定したプロパティによって異なり、文字列演算子、ブール関数、または数値演算子のいずれかを選択できます。
    • [Operator] : リストされる演算子は、指定したプロパティによって異なり、文字列演算子、ブール関数、または数値演算子のいずれかを選択できます。
    • [Value] : 指定したプロパティ用にドロップダウン リストに表示される既知の値または使用可能な値から選択するか、文字列または値を直接入力できます。このフィールドには、ワイルドカードとしてクエスチョン マーク (?) およびアスタリスク (*) 文字を使用できます。
    • [Add a new criterion] () : [Criteria] セクションに新しい行が追加されます。
    • [Remove this criterion] () : [Criteria] セクションから選択した行が削除されます。
    ヒント: [Criteria] セクションに複数の行がある場合は、条件間の論理関係を AND または OR のいずれかとして指定する必要もあります。AND の場合は指定した条件すべてに一致するオブジェクトが戻され、OR の場合は指定した条件のいずれかに一致するオブジェクトが戻されます。
  • [Regular Expressions] : [Value] フィールドを正規表現 (regexp) 検索パターンとして定義します。正規表現構文の詳細については、このリンクを参照してください。
  • [Ignore case] : 正規表現の検索にのみ適用され、大文字と小文字を区別しません。通常、検索パターンの大文字/小文字は区別されます。
  • [Search hierarchically] : デザイン階層の各レベルを検索します。デフォルトでは、現在のインスタンス current_instance、またはデザイン階層の最上位が検索されます。このオプションは、[Find] フィールドに指定したオブジェクトのタイプによって異なります。
  • [Of objects] : 選択したオブジェクトのリストに検索を制限します。たとえば、特定の I/O ポートのパッケージ ピンを見つけたり、特定のセル ピンのネットを見つけたりできます。参照ボタン () を開いて検索するか、該当する Tcl コマンドを手動で入力します。
  • [Command] : [OK] をクリックすると、親ダイアログ ボックスに渡された Tcl コマンドが表示されます。

[OK] をクリックすると、親ダイアログ ボックスに Tcl コマンドが渡されます。
