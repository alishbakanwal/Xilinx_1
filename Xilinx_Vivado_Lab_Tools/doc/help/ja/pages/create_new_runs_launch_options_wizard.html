[Launch Options] ページ

[Launch Options] ページでは、合成またはインプリメンテーション run を作成した後の動作を定義します。

  • [Launch Directory] : 定義済みの run に対して、run ファイルと結果を記述するディレクトリを指定します。
    重要: デフォルトのディレクトリは、現在のプロジェクト内に含まれます。現在のプロジェクト構造外に定義した場合は、ソース制御およびデザインのポータビリティに問題が発生する可能性があります。
  • [Launch runs on local host] : ローカル ホストで新しく作成した run がすぐに実行されます。
    • [Number of jobs] : ローカル ホストで同時に実行するデザイン run の数を指定します。合成 run は、依存しているインプリメンテーション run を実行する前に終了している必要があります。
  • [Launch runs on remote host] : このオプションは、Linux マシンでのみ使用でき、指定したリモート コンピュータで新しい run が実行されます。
    • [Configure Hosts] : [Vivado Options] ダイアログ ボックスが開き、ネットワークで使用可能なリモート コンピュータがコンフィギュレーションできるようになります。
  • [Launch runs using LSF] : このオプションは、Linux オペレーティング システムでのみ使用でき、LSF (Load Sharing Facility) を使用して分散されたプロセッシング システムで run を管理できるようになります。
    • [Configure LSF] : [Vivado Options] ダイアログ ボックスが開き、bsub コマンド オプションを使用して LSF 管理下で run を実行できるようになります。
  • [Generate scripts only] : 合成またはインプリメンテーション run のスクリプトが生成されますが、この時点では run は実行されません。スクリプト ファイルは run ディレクトリに書き込まれ、手動で実行するか、バッチ プロセスで実行できます。
  • [Do not launch now] : 新しい合成およびインプリメンテーション run を作成だけします。この時点ではこれらを実行したり、スクリプトを作成したりはされません。
