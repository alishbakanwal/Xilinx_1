[Create Partition Definition] ダイアログ ボックス

重要: まず、[Tools] → [Enable Partial Reconfiguration] コマンドを使用するか、プロジェクトの PR_FLOW プロパティを TRUE に設定して、プロジェクトをパーシャル リコンフィギュレーション (PR) プロジェクトとして定義する必要があります。

パーシャル リコンフィギュレーションでは、コア デザインと RM の組み合わせに基づいて、リコンフィギャラブル モジュール (RM) とパーティション定義 (partitionDef) が入れ替えられて、独自のデザイン コンフィギュレーションが作成されます。PR デザイン フローでは、PR コンフィギュレーションをそれぞれインプリメンテーションする必要があります。 この結果、RM のパーシャル ビットストリームは作成されますが、統合された各コンフィギュレーションのビットストリーム全体は作成されません。

[Create Partition Definition] ダイアログ ボックスでは、現在のプロジェクトの階層モジュールを選択して、PR プロジェクトに partitionDef オブジェクトを作成して、階層モジュールから RM を作成できます。partitionDef ではパーティション階層が定義され、RM はその partitionDef にフィットします。
重要: デザインの選択した階層モジュールのインスタンスすべてが partitionDefs と関連する RM に変換されます。
  • [Partition Definition Name] : パーティション定義の名前をテキスト文字列で入力します。
  • [Reconfigurable Module Name] : 定義するリコンフィギャラブル モジュールの名前を指定します。
    ヒント: これは、partitionDef に選択した階層モジュールの名前ではなく、その階層から作成される RM に割り当てられる名前です。

[OK] をクリックすると、選択した階層セルの partitionDef が作成され、その階層からのリコンフィギャラブル モジュールが作成されます。
