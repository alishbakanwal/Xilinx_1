[Project Settings] ダイアログ ボックス : [Implementation] ページ

[Project Settings] ダイアログ ボックスでは、デザイン フローを通して使用されるさまざまな設定に対して値を指定できます。設定ページは、合成、シミュレーション、インプリメンテーションなどのデザイン フローの手順別に分類されています。

[Implementation] ページでは、制約セットおよびストラテジを定義したり、アクティブなインプリメンテーション run のオプションをカスタマイズしたりできます。選択したストラテジのデフォルト オプションは表形式で表示され、カスタマイズするストラテジのオプションや値が変更できるようになっています。

  • [Default constraint set] : インプリメンテーションで使用する制約セットを指定します。Add Sources ウィザードを使用すると、新しい制約セットを作成できます。
  • [Incremental Compile] : 次のインプリメンテーション run のリファレンスとして使用するデザイン チェックポイント (DCP) を指定します。基準デザインは、通常はデザインの合成、配置、配線が完了した以前のバージョンです。ただし、部分的に配置配線されただけのチェックポイントを使用することもできます。詳細は、『Vivado Design Suite ユーザー ガイド : インプリメンテーション』 (UG904) のこのセクションを参照してください。
  • [Strategy] : 定義済み Vivado® Design Suite ストラテジを指定するか、選択したストラテジのオプションを変更して独自のストラテジを作成できます。
    ヒント: 選択したストラテジのこのオプションを変更した場合は (ストラテジ名にアスタリスク (*) で表示)、[Save Strategy As] ボタン () をクリックすると、それらのオプションを新しいユーザー定義のストラテジとして保存できます。
  • [Description] : 選択したストラテジの目的のサマリを簡単に記述しておきます。この部分は、ユーザー定義のストラテジを保存する際に定義できます。
  • [Options] : 選択したインプリメンテーション ストラテジで使用可能なオプションが表形式で表示されます。表のオプションを選択すると、そのオプションの詳細が表示されます。
    ヒント: インプリメンテーション ストラテジは、最適化、配置、および配線プロセスなどのさまざまな段階に分割されます。詳細は、『Vivado Design Suite ユーザー ガイド : インプリメンテーション』 (UG904) のこのセクションを参照してください。
