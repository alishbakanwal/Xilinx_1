[Report Timing] ダイアログ ボックス

[Report Timing] ダイアログ ボックスでは、デザイン内の特定パスを検証するためのタイミング レポートを生成できます。

デフォルトでは、[Report Timing] ダイアログ ボックスでクロック ドメインごとまたはパス グループごとに 10 個のワースト タイミング パスがレポートされます。これは、[Number of paths per path group] オプションを設定すると、パス グループごとにより多くのタイミング パスやより少ないタイミング パスが表示されるように変更できます。また、タイミング レポートには、デフォルトで各終点オブジェクトごとに 1 つのパスが表示されます。これは、1 つの終点にデフォルトではレポートされない複数のエラーになるタイミング パスを含めることができることを意味します。ただし、これは [Number of paths per endpoint] オプションを設定すると変更できます。これら 2 つのオプションにより、終点ごとおよびパス グループごとにレポートされるエラーになるタイミング パスの数が決定されます。

[Report Timing Summary] ダイアログ ボックスには、次のタブと共有オプションが含まれます。

  • [Targets] タブ
  • [Options] タブ
  • [Advanced] タブ
  • [Timer Settings] タブ
  • 共通オプション

[Targets] タブ

[Targets] タブでは、次のフィールドを指定することで該当するタイミング パスを定義できます。

  • [Start Points] : タイミング パスの [From] オブジェクトまたは始点を指定します。[Choose start points] ボタン () をクリックすると、[Specify Start Points] ダイアログ ボックスを開いたり、テキスト フィールドに Tcl コマンドを直接入力できます。
    • [Transition] : 始点での信号エッジを指定して、タイミング解析で考慮されるようにします。立ち上がりエッジ、立ち下がり エッジのいずれか、または両方を指定できます。クロック オブジェクトを指定すると、そのクロックの指定したエッジで開始されるパスのみが始点として考慮されます。
  • [Through Points] : タイミング パスの [Through] オブジェクトまたは通過点を指定します。[Choose through points] ボタン () をクリックすると、[Specify Through Points] ダイアログ ボックスを開いたり、テキスト フィールドに Tcl コマンドを直接入力できます。
    • [Transition] : 通過点での信号エッジを指定して、タイミング解析で考慮されるようにします。立ち上がりエッジ、立ち下がり エッジのいずれか、または両方を指定できます。
    • [Add through points] : タイミング パスの通過点を追加するための [Through] フィールドが追加されます。個別の通過点を順次指定し、解析用にデザインを通過する特定のパスを定義できます。
    • [Remove through points] : 選択した [Through] フィールドがダイアログ ボックスから削除されます。このオプションは、スルー ポイントを追加した場合にのみ使用できます。
  • [End Points] : タイミング パスの [To] オブジェクトまたは終点を指定します。[Choose end points] ボタン () をクリックすると、[Specify End Points] ダイアログ ボックスを開いたり、テキスト フィールドに Tcl コマンドを直接入力できます。
    • [Transition] : 終点での信号エッジを指定して、タイミング解析で考慮されるようにします。立ち上がりエッジ、立ち下がり エッジのいずれか、または両方を指定できます。クロック オブジェクトを指定すると、そのクロックの指定したエッジで開始されるパスのみが終点として考慮されます。

[Options] タブ

[Options] タブには、レポートの出力を設定するための次のフィールドが含まれます。

  • [Path delay type] : タイミング レポートを実行する際に解析する遅延のタイプを指定します。最大遅延はタイミング パスのセットアップ解析用で、最小遅延はホールド解析用です。セットアップおよびホールド解析両方の min_max 遅延を選択します。
  • [Do not report unconstrained paths] : 制約の付いていないパスのタイミングをレポートから除外します。デフォルトでは、レポートに制約の付いていないパスが含まれます。
  • [Number of paths per group] : レポートに表示されるパス数を指定します。デフォルトでは、10 個のワースト タイミング パス、またはパス グループごとにワースト パスがレポートされます。
  • [Number of paths per endpoint] : 終点ごとにレポートされるタイミング パス数を指定します。この数は、クロックまたはグループごとのパス数によって制限されます。
  • [Limit paths to group] : タイミング解析で評価するパス グループを指定します。指定したパス グループに含まれないパスは解析されません。[Browse] ボタン () をクリックすると、[Specify Path Group] ダイアログ ボックスを開いたり、テキスト フィールドに Tcl コマンドを直接入力できます。
  • [Display paths with slack greater than] : 必要のないパスがレポートから削除されます。この設定はサマリ レポートには影響せず、詳細にリストされるパスにのみ影響します。
  • [Display paths with slack less than] : 必要のないパスがレポートから削除されます。
  • [Significant digits] : タイミング レポートでレポートされる値の有効桁数を指定します。デフォルト値は 3 です。
  • [Sort paths by] : タイミング レポートのパスがパス グループ別またはスラック値別に並び替えられます。

[Advanced] タブ

[Advanced] タブには、次のフィールドが含まれます。

  • [Report from cell] : 指定した階層セル インスタンスのレベルからレポートを実行します。デフォルトでは、レポートはデザイン全体に適用されます。指定したセル内で開始、終了するパス、またはセルに完全に含まれるパスのみがレポートされます。
  • [Show input pins in path] : レポートに表示されるタイミング パスの詳細な説明に入力ピンを含めます。
  • [Report unique pins] : ピンの固有のセットそれぞれを使用して 1 つのタイミング パスがレポートされます。
  • [Report user-ignored paths] : set_false_path または set_clock_groups 制約があるため、通常タイミングで無視されるパスのみがレポートされます。
  • [Write results to file] : タイミング レポートの結果が指定ファイルに書き込まれます。デフォルトは [Overwrite] で、同じ名前の既存ファイルが上書きされます。[Append] をクリックすると、指定ファイルに新しいレポートが追加されます。
  • [Ignore command errors (quiet mode)] : [Report Timing Summary] コマンドを実行する際に、エラーが表示されないようになります。
  • [Suspend message limits during command] : このレポートを実行する際に定義されるメッセージ制限が無視されます。

[Timer Settings] タブ

[Timer Settings] タブには、レポートを生成する前にタイミング エンジンを設定するための次のフィールドが含まれます。

  • [Interconnect] : タイミング解析で使用するインターコネクト遅延を指定します。インプリメント済みデザインからすべての配線済み遅延を使用したり、合成またはインプリメント済みデザインで見積もり遅延を使用したり、インターコネクト遅延を使用しないようにして、ロジック遅延のみを検証できます。
  • [Speed grade] : 実際にターゲット パーツを変更することなく、タイミング解析中にデバイスのスピード グレードを変更できます。
  • [Multi-corner configuration] : 現在のデザインの Slow および Fast タイミング コーナーを設定できます。製造プロセス、電圧、および温度 (PVT) の変数により、デバイスの遅延が決定され、まとめられてタイミング コーナーが定義されます。これは、Tcl コマンドの config_timing_corners に関連します。
    • [Slow] : Slow コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延を使用します。タイミング解析からこのコーナーを除外するには、none を使用します。
    • [Fast] : Fast コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延を使用します。タイミング解析からこのコーナーを除外するには、none を使用します。
  • [Disable flight delays] : I/O 遅延の算出でパッケージ遅延が無視されます。フライト遅延は、パッケージ ピンとダイ パッド間で発生するパッケージ遅延です。これは、Tcl コマンドの config_timing_analysis に関連しています。

共通オプション

共通オプションのタブはありませんが、次はダイアログ ボックス内に常に表示されます。

  • [Results name] : Vivado® IDE の [Timing Results] ウィンドウのレポートに適用する名前を指定します。
  • [Command] : レポートを生成するために使用した Tcl コマンドと指定したオプションすべてが表示されます。
  • [Open in a new tab] : Vivado IDE の [Timing Results] ウィンドウに新しいタブでタイミング サマリが開きます。オフの場合は、新しいレポートで最後に開いたレポートが上書きされます。
  • [Open in Timing Analysis layout] : Vivado IDE の現在のレイアウト ビューがタイミング解析用のレイアウト ビューに変更されます。詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。

[OK] をクリックすると、表示された Tcl コマンドが実行されてレポートが生成されます。
