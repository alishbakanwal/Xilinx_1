[Configure Synthesis Runs] ページ

[Configure Synthesis Runs] ページでは、現在のプロジェクトに新しい合成 run を追加してコンフィギュレーションできます。最初は、デフォルトの合成 run が表示されます。

  • [Add] () : コンフィギュレーションされる run のリストに新しい合成 run が追加できます。
  • [Remove] () : コンフィギュレーションされる run のリストから合成 run を削除できます。このボタンは、複数の合成 run が run のリスト内で定義されていて、それらの run の 1 つまたは複数の run が選択されている場合にアクティブになります。
    ヒント: 最後の run は削除できません。run が 1 つも必要ない場合は、[Cancel] をクリックします。
  • [Name] : 作成してコンフィギュレーションする合成 run の名前を指定します。デフォルト名は、現在定義されている合成の名前に基づいて付けられますが、必要に応じて変更できます。
  • [Constraint set] : 合成 run で使用する制約セットを指定します。既存の制約セットのリストから選択する必要があります。新しい制約セットを使用する場合は、[Sources] ウィンドウから [Edit Constraints Sets] コマンドを使用して定義しておく必要があります。
  • [Part] : デザインのターゲット パーツまたはプラットフォーム ボードを指定します。現在のプロジェクトのデフォルト パーツが自動的に選択されます。[Select Device] ダイアログ ボックスから新しいターゲット パーツまたはボードを選択するには [Part] フィールドをクリックします。
  • [Strategy] : run に使用する合成ストラテジを指定します。メイン メニューから [Tools] → [Options] → [Strategies] を使用すると、run のストラテジを定義できます。
  • [Make Active] : 選択した run をアクティブ run にし、[Run Synthesis] コマンドが使用されたときに自動的に起動されるようにします。アクティブ run の結果が [Messages]、[Compilation]、[Reports]、[Project Summary] ウィンドウに表示されます。
