[Report Power] ダイアログ ボックス

[Report Power] ダイアログ ボックスでは、デザインの消費電力のレポートを生成できます。正確な消費電力の見積もりには、動作パラメーターとデザインのスイッチング アクティビティが現実的に設定されている必要があります。

[Report Power] ダイアログ ボックスには、デザインを設定するための次のタブとオプションが含まれます。

  • 共通オプション
  • [Environment] タブ
  • [Power Supply] タブ
  • [Switching] タブ

共通オプション

共通オプションのタブはありませんが、次はダイアログ ボックス内に常に表示されます。

  • [Results name] : Vivado® IDE の [Power Results] ウィンドウのレポートに適用する名前を指定します。
  • [Output text file] : 消費電力結果を書き出す ASCII ファイル名を指定します。
  • [Output XPE file] : Xilinx® Power Estimator スプレッドシート ツールにインポートするための XML ファイル名を指定します。詳細は、『Xilinx Power Estimator ユーザー ガイド』 (UG440) を参照してください。

[Environment] タブ

[Environment] タブでは、デバイスの動作環境を定義できます。これらの設定は、Tcl コマンドの set_operating_conditions に関連しています。

  • [Device Settings] : ターゲット デバイスの動作パラメーターを指定します。
    • [Temp grade] : ターゲット デバイスの温度グレード動作を指定します。サポートされる値は、デバイス ファミリによって異なります。
    • [Process] : 解析する製造プロセス コーナーを指定します。有効な値は、typical または maximum です。
  • [Environment Settings] : それが使用されるデバイスまたはシステムの動作に関連するさまざまな環境設定を指定できます。
    • [Output Load] : デザインで駆動される外部負荷容量を指定します。
    • [Junction temperature] : デバイスのジャンクション温度を指定します。値は、自動的に計算されます。チェック ボックスをオンにすると、温度 (摂氏) を手動で入力できます。
    • [Ambient temperature] : 動作環境の周囲温度 (摂氏) を指定します。
    • [Effective Theta-JA] : モデリングに使用する Theta-JA 熱抵抗を C/W で指定します。値は自動的に計算されますが、必要に応じて手動で入力することもできます。
    • [Airflow] : 冷却モデリングに使用するエアフローを LFM (リニア フィート/分) で指定します。デフォルト値はデバイス ファミリによって異なります。
    • [Heat sink] : 冷却モデリングに使用するヒートシンク プロファイルを指定します。
    • [Theta-SA] : モデリングに使用する Theta-SA 熱抵抗を C/W で指定します。値は、選択したヒート シンクから自動的に決定されますが、カスタム ヒート シンクを選択した場合は手動で Theta-SA を指定できます。
    • [Board selection] : 冷却モデリングに使用するボード サイズを指定します。有効な値は jedec、small、medium、large、custom です。
    • [Number of board layers] : システム ボードに含まれるレイヤー数を見積もります。有効な値は、4 ～ 7 層のボードでは [4to7]、8 ～ 11 層のボードでは [8to11]、12 ～ 15 層のボードでは [12to15]、16 層以上のボードでは 16+ です。
    • [Theta-JB] : ボードの Theta-JB 熱抵抗を C/W で指定します。値は自動的に計算されますが、必要に応じて手動で入力することもできます。
    • [Board temperature] : ボード温度を摂氏 (C) で指定します。

[Power Supply] タブ

[Power Supply] タブには、電圧電源の名前とそれに指定した値がリストされます。サポートされる電源名およびその値は、デバイス ファミリによって異なります。これらの電圧電源の設定は、Tcl コマンドの set_operating_conditions にも関連しています。

[Switching] タブ

[Switching] タブでは、現在のデザインの消費電力の見積もりを実行する際に使用されるデバイスの信号レートとスイッチング確立を定義できます。これらの設定は、Tcl コマンドの set_switching_activity に関連しています。

デザインのスイッチング アクティビティは、静的および動的消費電力の両方に影響します。静的消費電力はロジック ステートの遷移によって異なり、動的消費電力はトグル レートに正比例します。

  • [Simulation activity file] : 消費電力解析または消費電力最適化に使用する SAIF (Switching Activity Interchange Format) ファイルを読み込みます。SAIF ファイルではスイッチング アクティビティを使用してデザイン ノードがアノテートされているので、Vivado ツールで消費電力を正しく見積もることができるようになります。Vivado シミュレータでは SAIF ファイルが出力されます。 詳細は、『Vivado Design Suite ユーザー ガイド : ロジック シミュレーション』 (UG900) のこのセクションを参照してください。
  • [Default Activity Settings] :
    • [Default toggle rate] : 消費電力解析で現在デザインのプライマリ入力に使用するデフォルトのトグル レートを指定します。
    • [Default static probability] : 現在のデザインの消費電力解析で使用するデフォルトのスタティック確率を指定します。
  • [Enable Rate Settings] :
    • [BRAM Port Enable] : BRAM のイネーブル ピン (ENARDEN/ENBWREN) のスタティック確率とトグル レートを指定します。
    • [BRAM Write Enable] : BRAM の書き込みイネーブル (WEA/WEBWE) のスタティック確率とトグル レートを指定します。
    • [Bidi Output Port Enable] : 双方向ポートのイネーブル ピンのスタティック確率とトグル レートを指定します。
  • [Toggle Rate Settings] :
    • [Primary Outputs] : デザインの出力ポートのスタティック確率とトグル レートを指定します。
    • [Logic] : レジスタ、シフト レジスタ、分散およびブロック RAM、DSP のスタティック確率とトグル レートを指定します。
    • [GTs] : GT レシーバーとトランスミッターのスタティック確率とトグル レートを指定します。
  • [Constrained Clocks] : 解析で考慮されるクロック ドメインがリストされます。これは、情報提供のみを目的に表示されます。

[OK] をクリックすると、指定した値で消費電力解析が実行されます。
