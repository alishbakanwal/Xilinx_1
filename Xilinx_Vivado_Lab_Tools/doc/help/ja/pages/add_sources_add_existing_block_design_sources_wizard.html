[Add Existing Block Design Sources] ページ

[Add Existing Block Design Sources] ページでは、現在のプロジェクトに追加する既存のブロック デザイン ファイル (.bd) 指定できます。ブロック デザインは、Vivado® IP インテグレーター機能で作成されます。

  • [Add Files] : ファイル参照ダイアログ ボックスが開き、ブロック デザイン (.bd) を任意の数だけ選択できます。複数のファイルを選択すると、指定した順番でファイル リストに追加されます。
    ヒント: をクリックして [Add Files] をクリックしても同じ操作を実行できます。
  • [Remove] () : 現在のプロジェクトから選択したファイルを削除します。ファイルは現在のプロジェクトから削除されますが、ハード ドライブからは削除されません。
  • [Move Up] : ファイル リストでファイルが選択されている場合、左側のメニューから青色の上方向矢印ボタン () が使用できるようになり、リスト内でファイルを上に移動できます。デザイン ファイルはこの順序でデザインに読み込まれて、コンパイルされます。コンパイルの順序および [Hierarch Update] コマンドの詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。
  • [Move Down] : ファイル リストでファイルが選択されている場合、左側のメニューから青色の下方向矢印ボタン () が使用できるようになり、リスト内でファイルを下に移動できます。
  • [Copy sources into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定したブロック デザインをコピーできます。
