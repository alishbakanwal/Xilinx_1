[Find] ダイアログ ボックス

[Find] ダイアログ ボックスでは、現在のエラボレート、合成、インプリメント済みデザインのオブジェクトを見つけるための条件を定義できます。オブジェクトに指定されたプロパティを使用して、ネットリストとデバイス オブジェクトが検索できます。

  • [Results name] : [Find Results] ウィンドウに表示される結果の名前を指定します。
  • [Find] : 現在のデザインで検出するファースト クラスのオブジェクトのタイプを指定します。オブジェクトは、[Netlist] オブジェクトおよび [Device] オブジェクトとしてリストされます。
  • [Properties] : ドロップダウン リストからプロパティ、演算子、および値を選択し、現在のデザイン内のオブジェクトを検索します。
    • プロパティ : リストされる演算子は、指定したプロパティによって異なり、文字列演算子、ブール関数、または数値演算子のいずれかを選択できます。[More Properties] をクリックすると、リストにプロパティを追加できます。
    • 演算子 : リストされる演算子は、指定したプロパティによって異なり、文字列演算子、ブール関数、または数値演算子のいずれかを選択できます。
    • 値 : リストされる値は、既知の値、または指定した [Property] に対して使用可能な値から定義されるか、直接入力した文字列または値にできます。このフィールドには、ワイルドカードとしてクエスチョン マーク (?) およびアスタリスク (*) 文字を使用できます。
    • [Add a new criterion] () : [Options] セクションに新しい行を追加します。
    • [Remove this criterion] () : [Options] セクションから選択した行を削除します。
    ヒント: [Options] セクションに複数の行がある場合は、条件間の論理関係を AND または OR のいずれかとして指定する必要もあります。AND の場合は指定した条件すべてに一致するオブジェクトが戻され、OR の場合は指定した条件のいずれかに一致するオブジェクトが戻されます。
  • [Regular Expressions] : [Value] フィールドを正規表現 (regexp) 検索パターンとして定義します。正規表現構文の詳細については、このリンクを参照してください。
  • [Ignore case] : 正規表現の検索にのみ適用され、大文字と小文字を区別しません。通常、検索パターンの大文字/小文字は区別されます。
  • [Search hierarchically] : デザイン階層の各レベルを検索します。デフォルトでは、現在のインスタンス current_instance、またはデザイン階層の最上位が検索されます。このオプションは、[Find] フィールドに指定したオブジェクトのタイプによって異なります。
  • [Display unique nets] : 階層で検出された各ネットの最上位ネット セグメントのみが表示されます。このオプションは、NET タイプのオブジェクトを検索する際にのみ使用できます。
  • [Of objects] : [Find] 検索をデザイン全体ではなく、選択したオブジェクトのみに限定します。たとえば、特定の I/O ポートのパッケージ ピンを見つけたり、特定のセル ピンのネットを見つけたりできます。参照ボタン () をクリックして [Of Objects - Find] ダイアログ ボックスを開いて検索するか、該当する Tcl コマンドを手動で入力します。
  • [Command] : [OK] ボタンをクリックすると実行される Tcl コマンドが表示されます。表示されたコマンドを使用すると、Vivado® IDE で使用される Tcl コマンドを作成しやすくなります。
  • [Open in a new tab] : 結果を [Find Results] ウィンドウの新しいビューに表示します。オフの場合は、新しい結果で開いている検索結果が上書きされます。

[OK] をクリックすると、指定したオブジェクトの検索が実行され、[Find Results] ウィンドウが開きます。
