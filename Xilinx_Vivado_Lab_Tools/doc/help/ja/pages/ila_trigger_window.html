[ILA Trigger Setup] ウィンドウ

このウィンドウは、[Hardware ILA] ダッシュボードに表示されるデフォルトの 5 つのウィンドウの 1 つで、指定したプローブのトリガー ブール式およびエッジ遷移トリガーなどの、選択したデバッグ コアのトリガー条件が定義できます。[ILA Trigger] ウィンドウの内容は [Trigger Mode] の設定によって異なります。

  • ベーシック タイプのトリガー
  • アドバンス タイプのトリガー

ベーシック トリガー

ベーシック トリガー条件は、トリガーに含まれる各デバッグ プローブの値を評価するブール式で、結果が true の場合に TRIG_OUT 信号と ILA データ キャプチャ シーケンスがトリガーされます。ベーシック トリガー モードを使用する場合は、まずトリガー定義にどの ILA デバッグ プローブを含めるかを決定します。ベーシック トリガーの詳細は、『Vivado Design Suite ユーザー ガイド : プログラムおよびデバッグ』 (UG908) のこのセクションを参照してください。[ILA Trigger] ウィンドウにはトリガーに関連付けられたプローブが次を含む表形式でリストされます。
  • [Name] : ILA デバッグ コアのプローブの名前を指定します。
  • [Operator] : ポートのトリガー イベントを評価するのに使用される演算子を指定します。演算子は、フィールドでクリックしてドロップダウン メニューから選択すると変更できます。使用可能な演算子は、そのプローブの基数と有効な値によって異なります。
  • [Radix] : プローブの値の基数を指定します。基数は、フィールドでクリックしてドロップダウン メニューから選択すると変更できます。
  • [Value] : トリガー式で使用するプローブ値を指定します。この値と演算子で、プローブ コンパレータが true か false かが決まります。
  • [Port] : プローブが割り当てられたポートがレポートされます。
  • [Comparator] : トリガーをその一部として含む特定のプローブ コンパレータがリストされます。
ILA デバッグ コアのトリガー コンパレータでは、指定した演算子に従って各プローブの値が評価され、トリガー式が true なのか false なのかが決定されます。ILA デバッグ コアのトリガーは、すべての ILA プローブのトリガー コンパレータの結果に適用されるグローバル ブール式 (AND、OR、NAND、または NOR) の結果で、ツールバー メニューの [Set trigger condition] で定義します。
重要: トリガー条件には、[ILA Trigger] ウィンドウ内のプローブのみを含めることができます。ウィンドウに含まれないプローブは、トリガー条件には含まれません。

[BASIC] トリガー モードのツールバーには、次のコマンドが含まれます。

Search 特定のプローブが表示されるように、検索パターンを入力するテキスト フィールドが開きます。
Add probes ILA デバッグ コアに関連するプローブがリストされた [Add Probes] ダイアログ ボックスが開きます。プローブを選択して [OK] をクリックすると、ウィンドウにそのプローブが追加されます。
Remove selected probes [ILA Trigger Setup] ウィンドウから選択したプローブが削除されます。
Set trigger condition プローブのグローバル トリガー条件を定義できます。

アドバンス トリガー

アドバンス トリガー設定を使用すると、既存のトリガー ステート マシンをインポートしたり、新しいものを作成したりできます。トリガー ステート マシンを開くか作成すると、[ILA Trigger Setup] ウィンドウがテキスト エディターに変換され、ほとんど同じコマンドが含まれ、テキスト ファイルを編集できるようになります。ただし、[ILA Trigger Setup] ウィンドウには、現在の ILA デバッグ コアに関連するプローブをリストして、それらをトリガー ステート マシンに追加できる [Insert Probe] ダイアログ ボックスを開く [Insert probe] コマンド () があります。アドバンス トリガーとトリガー ステート マシンの作成に関する詳細は、『Vivado Design Suite ユーザー ガイド : プログラムおよびデバッグ』 (UG908) のこのセクションを参照してください。
