[Report Design Analysis] ダイアログ ボックス

[Report Design Analysis] コマンドには、タイミング パスのタイミング解析と物理特性の解析、配線の複雑性と LUT 分散によるタイミング問題を識別するデザインの複雑性解析、配線密集の解析の 3 つの動作モードがあります。

[Report Design Analysis] ダイアログ ボックスには、次のタブと共有オプションが含まれます。

  • [Options] タブ
  • [Advanced] タブ
  • [Timer Settings] タブ
  • 共通オプション

[Options] タブ

[Options] タブには、レポートで実行される解析タイプを設定するための次のフィールドが含まれます。

  • [Timing] : すべてのクリティカル パスまたは指定したパスのタイミング解析が実行されます。
    • [All paths] : デザインのすべてのクリティカル パスのタイミング データが戻されます。
      • [Path delay type] : すべてのパスに対して解析する遅延のタイプを指定します。最大遅延はタイミング パスのセットアップ解析用で、最小遅延はホールド解析用です。セットアップおよびホールド解析両方には min_max 遅延を選択します。
      • [Maximum number of paths] : レポートに表示されるパス数を指定します。デフォルトでは、GUI レポートに 10 個のワースト タイミング パスが含まれます。
    • [Specific paths] : 指定したタイミング パスの解析が実行されます。
      ヒント: オンにすると、タイミング パスを定義するフィールドが入力できるようになります。Tcl コマンドを入力すると、タイミング パスを直接定義できます。 または [Find Timing Paths] コマンドを使用してレポートするタイミング パスを定義します。
    • [Extend Analysis] : 指定したタイミング パスのより詳細な解析が実行されます。詳細な解析には、指定したタイミング パスの始点で終了するパスと、指定したタイミング パスの終点で開始するパスが含まれます。
      ヒント: 最小 (ホールド) の [Path delay type] (パス遅延タイプ) が [Extend Analysis] オプションに指定されると、ワースト セットアップ パスを示すセットアップ レポートと、その同じパスに該当するホールド レポートが生成されます。このレポートには、ホールドおよびセットアップ修正が相互に影響するかどうかを確認しやすくするために、同じ始点と終点のセットアップ特性を持つホールド特性が示されます。
    • [Include logic level distribution] : デザイン内の長いパスが見つけやすくなります。
      ヒント: ロジック レベル分布 (logic level distribution) は、ツールで検出された、またはユーザーの指定したクリティカル タイミング パスにのみ実行できます。解析でデザインのロジック レベルの数が多いすべてのパスを見つけるには、[Specific paths] を使用して該当するタイミング パスを定義する必要があります。また、タイミング パスに LOGIC_LEVELS プロパティを使用してもこの情報を取得できます。
      • [Number of paths] : ロジック レベル分布に含めるパス数を指定します。
  • [Complexity] : Rent 指数、平均ファンアウト、およびプリミティブ ヒストグラムをレポートする複雑性解析が実行されるようになります。
    • [Cells to analyze] : 複雑性解析を実行する 1 つまたは複数の階層モジュールを指定します。デフォルトでは、デザインの最上位セルが解析されます。
      ヒント: オンにすると、解析するセルを指定するフィールドが入力できるようになります。Tcl コマンドで直接セルを指定できるほか、[Specify Top Level Cell] コマンドを使用してセルを定義できます。
    • [Hierarchical depth] : 複雑性解析に含める階層レベルを指定します。
  • [Congestion] : 配置または配線済みデザインの水平方向および垂直方向の配線密集がタイルごとにレポートされます。密集度解析は、複雑性解析が指定したセルで実行されている場合でも、デザイン全体に対して実行されます。

[Advanced] タブ

  • [Write results to file] : デザイン解析レポートの結果が指定ファイルに書き込まれます。デフォルトは [Overwrite] で、同じ名前の既存ファイルが上書きされます。[Append] をクリックすると、指定ファイルに新しいレポートが追加されます。
  • [Ignore command errors (quiet mode)] : デザイン解析レポートを実行する際に、エラーが表示されないようになります。
  • [Suspend message limits during command] : このレポートを実行する際に定義されるメッセージ制限が無視されます。

[Timer Settings] タブ

[Timer Settings] タブには、レポートを生成する前にタイミング エンジンを設定するための次のフィールドが含まれます。

  • [Interconnect] : タイミング解析で使用するインターコネクト遅延を指定します。インプリメント済みデザインからすべての配線済み遅延を使用したり、合成またはインプリメント済みデザインで見積もり遅延を使用したり、インターコネクト遅延を使用しないようにして、ロジック遅延のみを検証できます。
  • [Speed grade] : 実際にターゲット パーツを変更することなく、タイミング解析中にデバイスのスピード グレードを変更できます。
  • [Multi-corner configuration] : 現在のデザインの Slow および Fast タイミング コーナーを設定できます。製造プロセス、電圧、および温度 (PVT) の変数により、デバイスの遅延が決定され、まとめられてタイミング コーナーが定義されます。これは、Tcl コマンドの config_timing_corners に関連します。
    • [Slow] : Slow コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延を使用します。タイミング解析からこのコーナーを除外するには、none を使用します。
    • [Fast] : Fast コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延を使用します。タイミング解析からこのコーナーを除外するには、none を使用します。
  • [Disable flight delays] : I/O 遅延の算出でパッケージ遅延が無視されます。フライト遅延は、パッケージ ピンとダイ パッド間で発生するパッケージ遅延です。これは、Tcl コマンドの config_timing_analysis に関連しています。

共通オプション

共通オプションのタブはありませんが、次はダイアログ ボックス内に常に表示されます。

  • [Results name] : Vivado® IDE の [Timing Results] ウィンドウのレポートに適用する名前を指定します。
  • [Command] : レポートを生成するために使用した Tcl コマンドと指定したオプションすべてが表示されます。
  • [Open in a new tab] : Vivado IDE の [Timing Results] ウィンドウに新しいタブでタイミング サマリが開きます。オフの場合は、新しいレポートで最後に開いたレポートが上書きされます。

[OK] をクリックすると、表示された Tcl コマンドが実行されてレポートが生成されます。
