[Define Command Arguments] ダイアログ ボックス

このダイアログ ボックスでは、カスタム コマンドに割り当てた Tcl proc (プロシージャー) で使用される必須またはオプションの引数を定義したり、これらの引数のデフォルト値を定義したりできます。たとえば、次のコマンド例を実行するのに必要な引数を定義できます。

example_proc {arg1 arg2 {arg3}}

この例の場合、1 つ目と 2 つ目の引数 (arg1 および arg2) が必須の引数で、arg3 がオプションの引数です。このダイアログ ボックスでは、引数の名前、コメント、デフォルト値を定義し、次に示す機能を使用して arg3 をオプションとして指定します。

Add Tcl proc で必要とされる引数の表示名を入力できます。新しい引数は、Enter キーを押すとリストの一番下に追加されます。
注: 指定した名前は表示およびリファレンス用です。引数はその位置で Tcl proc に渡されるので、Tcl プロシージャーのコマンド フットプリントで必要とされる順番どおりにリストされている必要があります。
Remove リストから選択した引数が削除されます。
Move up 選択した引数がリストの上方向に移動されます。
ヒント: 引数はリストされる順番で Tcl コマンドに渡されます。example_proc の場合、最初にリストされる引数が arg1 として、2 つ目にリストされる引数が arg2 として、最後にリストされる引数が arg3 として渡されます。
Move down 選択したコマンドがリストの下方向に移動されます。
  • [Name] : 引数の表示名を指定します。名前はこのフィールドで直接編集することもできます。
  • [Comments] : 引数を使用してカスタム コマンドが実行されると開く [Custom Commands Arguments Entry] ダイアログ ボックスで表示されるコメントを指定します。コメントは、引数を選択するとダイアログ ボックスの一番下に表示され、その引数の説明や予測される値などが表示されます。
  • [Default value] : コマンドを実行したときにユーザーに上書きされる値がない場合に、カスタム コマンドに渡されるデフォルト値を指定します。
  • [Optional] : 引数がオプションであることを指定するチェック ボックスです。
    注: オプションの引数は、Tcl proc の引数リストの最後、必須の引数の後に表示されます。

[OK] をクリックすると、引数が定義され、ダイアログ ボックスが閉じます。
