[Sources] ウィンドウ

[Sources] ウィンドウには、現在のプロジェクトに関連するデザイン ソースが複数のビューで表示されます。次のビューがあります。

  • [Hierarchy] : ソースがデザインの最上位から下位の順に表示されます。
  • [IP Sources] : IP またはブロック デザインを含むデザインの IP ソース ファイルおよび生成された出力が表示されます。
  • [Libraries] : ライブラリ別に分類されたソースが表示されます。
  • [Compile Order] : ツールバーすぐ下の [Used In] の選択に基づいて、合成、インプリメンテーション、またはシミュレーション用にコンパイルされる順番にソースを並べ替えます。
    ヒント: パーシャル リコンフィギュレーション フローが使用される場合、[Sources] ウィンドウには [Partition Definitions] ビューも含まれるようになります。PR フローの詳細は、『Vivado® Design Suite ユーザー ガイド : パーシャル リコンフィギュレーション』 (UG909) を参照してください。

[Sources] ウィンドウでは、HDL デザイン ソース、IP とブロック デザイン ソース、制約ファイルと制約セット、およびテストベンチなどのシミュレーション ソース ファイルを管理できます。[Sources] ウィンドウの右クリック メニューには、現在のプロジェクトからファイルを追加または削除したり、ライブラリおよびコンパイル順を変更したり、特定のファイルセットをアクティブにしたり、パーシャル リコンフィギュレーションを管理したり、デザイン制約のターゲットを設定したりできるコマンドが含まれます。

[Sources] ウィンドウのツールバーには、次のコマンドが含まれます。

Search 検索パターンを入力する検索フィールドが開きます。
Collapse all すべてのソース フォルダー階層の展開が閉じられます。
Expand all すべてのソース フォルダー階層が展開されます。
Open Selected 選択したソース ファイルが開きます。
ヒント: このコマンドは、ソース ファイルが選択されている場合にのみアクティブになります。
[Add Sources] ページ Add Sources ウィザードを開いて、さまざまなタイプのソース ファイルをデザインに追加できます。
Show Only Missing Sources [Hierarchy] タブから使用でき、不足しているソース ファイルのみが表示され、問題をわかりやすくなります。
ヒント: このコマンドは、足りないソース ファイルがプロジェクトにある場合にのみ使用できます。
Sort [Libraries] タブから使用でき、リストされたフォルダーおよびソース ファイルをアルファベット昇順で並べ替えます。
Scroll オブジェクトを [Device] ウィンドウなどの別のウィンドウで選択した場合に、[Sources] ウィンドウでその選択したオブジェクトをスクロールして表示します。
