[Add Configuration Memory Device] コマンド

ザイリンクス デバイスにデザイン特定のデータを読み込む (プログラムする) プロセスは、コンフィギュレーションと呼ばれます。ザイリンクス デバイスは、外部フラッシュ メモリ デバイスからコンフィギュレーションするか、マイクロプロセッサ、DSP プロセッサ、ボード テスターなどの外部スマート ソースからコンフィギュレーションできます。コンフィギュレーション データパスには、コンフィギュレーションのデバイス ピン要件を最小限にするために使用されるシリアル ペリフェラル インターフェイス (SPI) とパフォーマンスのより高いコンフィギュレーションに使用されるパラレル バイト ペリフェラル インターフェイス (BPI) の 2 つがあります。[Add Configuration Memory Device] ダイアログ ボックスでは、ハードウェア デバイスをコンフィギュレーションおよび起動するためにの必要な現在のデザインのビットストリームを格納するコンフィギュレーション メモリを選択できます。

Vivado ハードウェア マネージャーの [Write Configuration Memory File] コマンドを使用すると、特別なコンフィギュレーション メモリ ファイル (CFGMEM) が作成されます。 Vivado デバイス プログラマでは、このコンフィギュレーション メモリ ファイルが使用されて、JTAG を介して間接的にフラッシュ ベースのコンフィギュレーション メモリ デバイスがプログラムされます。[Program Configuration Memory Device] コマンドを使用すると、コンフィギュレーション メモリ デバイスをビットストリームと CFGMEM ファイルからのその他のデータでプログラムできます。

  • [Filter] : 次の条件に従って、コンフィギュレーション メモリ デバイスのリストがフィルターできます。
    • [Manufacturer] : コンフィギュレーション メモリ デバイスのサポートされる製造元の 1 つを選択します。
    • [Density] : デバイスのメモリ容量を選択します。
    • [Type] : バイト ペリフェラル インターフェイス (BPI) またはシリアル ペリフェラル インターフェイス (SPI) のみを表示するか、すべてのタイプを表示するか選択できます。
    • [Width] : デバイスのプログラミング データ幅を選択します。
    ヒント: [Reset Filters] コマンドを使用すると、フィルターが削除され、デフォルトの設定に戻ります。
  • [Search] : デバイスを指定した検索文字列に一致するもののみに表示を絞りこむことができます。

[Select Configuration Memory Part] の表には、適用したフィルターか検索文字列パターンに基づいて使用可能なデバイスがリストされます。結果の表には、名前、パーツ、製造元、タイプなどの情報を示す複数の列が含まれます。結果の表を並び替えるには、列ヘッダーをクリックします。1 回クリックした場合は昇順に、２ 回クリックした場合は降順に列が並べ替えられます。複数の列を並び替えるには、Ctrl キーを押しながら列ヘッダーをクリックしていきます。

コンフィギュレーション メモリ デバイスを選択し、[OK] をクリックします。コンフィギュレーション メモリ デバイスが現在のデバイスに追加され、[Hardware] ウィンドウに表示されます。
