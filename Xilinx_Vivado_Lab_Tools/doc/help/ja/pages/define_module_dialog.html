[Define Module] ダイアログ ボックス

[Define Module] ダイアログ ボックスのフィールドは、定義するモジュールまたはエンティティの言語 (Verilog、SystemVerilog、または VHDL) によって異なります。言語は、[Create Source File] ダイアログ ボックスの [File Type] 設定で決まります。

  • [Module name/Entity name] : [Create Source File] ダイアログ ボックスの [File Name] を使用して自動的に生成されます。 ファイル拡張子は付きません。ただし、モジュール名はファイル名と一致している必要はなく、このフィールドで変更できます。
  • [Architecture name] : 定義された VHDL モジュールのアーキテクチャを指定します。デフォルトは [behavioral] ですが、変更できます。アーキテクチャでは回路の内部表示が提供され、回路の動作または構造が記述されます。
    ヒント: 指定したアーキテクチャはモジュール宣言で宣言されますが、アーキテクチャの詳細はユーザーが定義する必要があります。
  • [IO Port Definitions] : 現在定義されているモジュールのポートをリストします。この表には、各ポートの [Port Name]、[Direction]、[Bus]、[MSB] および [LSB] が表示されます。表内のポートのこれらのフィールドはそれぞれ編集できます。
    • [Port Name] : ポートの名前を指定します。テキスト フィールドに直接入力できます。名前定義のないポートはモジュールに記述されないことに注意してください。
    • [Direction] : ポートを入力、出力、双方 (InOut) ポートとして指定します。
    • [Bus] : ポートがバス ポートであることを示します。[MSB] および [LSB] オプションを使用してバスの幅を定義します。
    • [MSB] : 最上位ビット (MSB) を定義します。これと [LSB] フィールドを併せて、定義されるバスの幅が決定されます。
    • [LSB] : 最下位ビット (LSB) を定義します。
    新しいポートをモジュールに追加したり、既存のポートを削除したり、ポートの順番を次のように変更できます。
    • [Add] : 緑のプラス マーク () をクリックすると、新しいポート定義が表に追加できます。また、表の空の行をクリックすると、表に新しい行が追加されます。
    • [Remove] : 赤のマイナス マーク () をクリックすると、表からポートを削除できます。このコマンドは、表の行を選択すると使用できるようになります。選択したポートがモジュール定義から削除されます。
    • [Move Up] : 青い上向き矢印ボタン () をクリックすると、選択したポートをリストの上の方に移動できます。このコマンドは、表の行を選択すると使用できるようになります。
    • [Move Down] : 青い下向き矢印ボタン () をクリックすると、選択したポートをリストの下の方に移動できます。このコマンドは、表の行を選択すると使用できるようになります。
  • [New Source Files] : このセクションは、モジュール宣言が必要な複数の新しいソース ファイルがリストされます。このリストでファイルを選択すると、モジュールを宣言できます。
    ヒント: [New Source Files] セクションは、複数の新しいソース ファイルが [Add Sources] ダイアログ ボックスまたは Add Sources ウィザードで指定される場合にのみ表示されます。1 つのモジュールを定義するためには、使用できません。
[OK] をクリックすると、新しいモジュールまたはエンティティ定義がソース ファイルに書き込まれます。 このファイルは現在のプロジェクトに追加されます。
重要: 複数のソース ファイルが [New Source Files] セクションにリストされている場合は、すべての新しいモジュール宣言が同時に書き込まれます。[OK] をクリックする前にリストされている全ソース ファイルに対してすべてのモジュールを定義しておかないと、未定義のモジュールを手動で定義する必要があります。
