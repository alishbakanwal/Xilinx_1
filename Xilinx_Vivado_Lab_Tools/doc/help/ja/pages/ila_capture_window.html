[ILA Capture Setup] ウィンドウ

このウィンドウは、[Hardware ILA] ダッシュボードに表示されるデフォルトの 5 つのウィンドウの 1 つで、内容は [ILA Settings] ウィンドウの [Capture Mode] の設定によって異なります。[Capture Mode] が [ALWAYS] の場合は定義するキャプチャ条件がないので、[ILA Capture] ウィンドウは空白にできます。[Capture Mode] が [BASIC] の場合は、ウィンドウには次の機能が含まれます。

ベーシック キャプチャ

ベーシック キャプチャ条件は、キャプチャ条件に含まれる各デバッグ プローブの値を評価するブール式で、結果が true の場合に ILA データ キャプチャ シーケンスがトリガーされます。ILA デバッグ コアのキャプチャ条件を定義するには、まずどのデバッグ プローブを含めるかを決める必要があります。[ILA Capture Setup] ウィンドウにはキャプチャ条件に割り当てられたプローブが次の情報を含む表形式でリストされます。
  • [Name] : ILA デバッグ コアのプローブの名前を指定します。
  • [Operator] : プローブのキャプチャ条件を評価するのに使用される演算子を指定します。演算子は、フィールドでクリックしてドロップダウン メニューから選択すると変更できます。使用可能な演算子は、そのプローブの基数と有効な値によって異なります。
  • [Radix] : プローブの値の基数を指定します。基数は、フィールドでクリックしてドロップダウン メニューから選択すると変更できます。
  • [Value] : キャプチャ条件で使用するプローブ値を指定します。[Value] および [Operator] で定義されるブール式によって、プローブのキャプチャ条件が true になるか false になるかが決まります。
  • [Port] : プローブが割り当てられたポートがレポートされます。
各プローブのブール式が指定した演算子および値に従って評価され、式が true なのか false なのかが決まります。ILA デバッグ コアのキャプチャ条件は、すべての ILA プローブのキャプチャ コンパレータの結果に適用されるグローバル ブール式 (AND、OR、NAND、または NOR) の結果で、ツールバー メニューの [Set capture condition] で定義します。
重要: キャプチャ条件には、[ILA Capture Setup] ウィンドウに追加されたプローブのみが含まれます。ウィンドウに含まれないプローブは、キャプチャ条件には含まれません。

[ILA Capture Setup] ウィンドウのツールバーには、次のコマンドが含まれます。

Search 特定のプローブが表示されるように、検索パターンを入力するテキスト フィールドが開きます。
Add probes ILA デバッグ コアに関連するプローブがリストされた [Add Probes] ダイアログ ボックスが開きます。プローブを選択して [OK] をクリックすると、キャプチャ条件にそのプローブが追加されます。
Remove selected probes [ILA Capture Setup] ウィンドウから選択したプローブが削除されます。
Set capture condition すべてのプローブのグローバル キャプチャ条件を定義できます。
