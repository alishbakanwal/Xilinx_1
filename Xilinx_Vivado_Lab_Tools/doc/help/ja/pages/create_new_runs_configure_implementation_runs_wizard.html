[Configure Implementation Runs] ページ

[Configure Implementation Runs] ページでは、現在のプロジェクトに新しいインプリメンテーション run を追加してコンフィギュレーションできます。最初は、デフォルトのインプリメンテーション run が表示されます。

  • [Add] () : コンフィギュレーションされる run のリストに新しいインプリメンテーション run を追加できます。
  • [Remove] () : コンフィギュレーションされる run のリストからインプリメンテーション run を削除できます。このボタンは、複数のインプリメンテーション run が run のリスト内で定義されていて、それらの run の 1 つまたは複数の run が選択されているとアクティブになります。
    ヒント: 最後の run は削除できません。run が 1 つも必要ない場合は、[Cancel] をクリックします。
  • [Name] : 作成してコンフィギュレーションするインプリメンテーション run の名前を指定します。デフォルト名は、現在定義されているインプリメンテーションの名前に基づいて付けられますが、必要に応じて変更できます。
  • [Synth Name] : インプリメンテーション run に関連付ける合成 run の名前を指定します。
  • [Constraint set] : インプリメンテーション run で使用する制約セットを指定します。これで、配置配線を駆動するために使用される制約とインプリメンテーション終了後に書き込まれる制約セットの両方が決定されます。既存の制約セットのリストから選択する必要があります。新しい制約セットを使用する場合は、[Sources] ウィンドウから [Edit Constraints Sets] コマンドを使用して定義しておく必要があります。
  • [Part] : デザインのターゲット パーツまたはプラットフォーム ボードを指定します。現在のプロジェクトのデフォルト パーツが自動的に選択されます。[Select Device] ダイアログ ボックスから新しいターゲット パーツまたはボードを選択するには [Part] フィールドをクリックします。
  • [Strategy] : run に使用するインプリメンテーション ストラテジを指定します。メイン メニューから [Tools] → [Options] → [Strategies] を使用して run のストラテジを定義できます。
  • [Make Active] : 選択した run をアクティブ run にし、[Run Implementation] コマンドが使用されたときに自動的に起動されるようにします。アクティブ run の結果が [Messages]、[Compilation]、[Reports]、[Project Summary] ウィンドウに表示されます。
