外部フィードバック遅延の理解

Figure: 外部 MMCM 補正の例 :



MMCM の COMPENSATION プロパティは EXTERNAL に設定され、フィードバック ピンの CLKFBIN と CLKFBOUT はボード上の FPGA デバイス外で一緒に接続されています。フィードバック ループ ポート (上記の例の場合 clkFbIn および clkFbOut) 間のボード遅延に set_external_delay コマンドで制約を付けると、MMCM の補正遅延がタイミング解析中に正しくレポートされます。次は、その例です。
set_external_delay -from [get_ports clkFbOut] -to [get_ports clkFbIn] -max 1.000
set_external_delay -from [get_ports clkFbOut] -to [get_ports clkFbIn] -min 0.500

[External Feedback Delays] ページのフィールドの理解

  • [Recommended Constraints] : タイミング制約ウィザードでは、MMCM セルのフィードバック ループ接続が解析され、CLKFBIN ピンと CLKFBOUT ピンが I/O バッファーを介してポートに接続される場合、外部遅延制約が推奨されます (EXTERNAL 補正)。
  • [Tcl Command Preview] : 遅延を定義するのに使用した set_external_delay コマンドを [Recommended Constraints] にリストされるように表示します。これらの制約がデザインに追加されます。
  • [Existing Constraints] : 現在定義されている外部遅延制約が表示されます。

上記のセクションにはそれぞれツールバーもあり、次のコマンドの 1 つまたは複数を実行できます

Search 検索文字列を入力するためのテキスト入力フィールドが表示されます。
Select All 推奨される制約すべてを選択します。
Edit selected row 選択した制約の最小遅延および最大遅延の値を定義するための [External Feedback Constraints] ダイアログ ボックスが開きます。これらの属性は、[Recommended Constraints] リストのフィールドを直接クリックしても編集できます。
Report Clock Networks [Report Clock Networks] コマンドを実行して、新しいクロック ネットワーク レポートが生成されます。
Tip: クロック ネットワークは、[Clock Networks] ウィンドウにレポートされます。
