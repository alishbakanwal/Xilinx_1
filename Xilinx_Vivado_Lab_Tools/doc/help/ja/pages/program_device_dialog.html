[Program Device] ダイアログ ボックス

このダイアログ ボックスでは、ビットストリーム ファイルとプローブ ファイルを指定して、現在のデバイスにダウンロードできます。プローブ ファイルは、デザインに含まれるデバッグ コアに対応しています。

ターゲット ハードウェアに接続した後、デバイスをプログラムするには、インプリメント済みデザインからのビットストリーム ファイル (.bit、.rbt、.bin) をデバイスと関連付ける必要があります。ビットストリーム ファイルは、[Flow] → [Generate Bitstream] コマンドまたは Tcl コマンドの write_bitstream を使用してインプリメンテーション後に作成します。このダイアログ ボックスでは、ビットストリームを定義する PROGRAM.FILE プロパティと現在のデバイスの PROBES.FILE プロパティを設定します。

  • [Bitstream file] : 現在のデザインに関連するビットストリーム ファイルの名前とパスを指定します。
    重要: ビットストリーム ファイルがデザイン IP を保護するために暗号化されたら、まず暗号キーを BBR または eFUSE キー レジスタに読み込んで、デバイスをイネーブルにして、ビットストリームの暗号を解読させます。これには、[Program BBR Key] ダイアログ ボックスまたは Program Efuse Registers ウィザードを使用します。詳細は、『Vivado Design Suite ユーザー ガイド : プログラムおよびデバッグ』 (UG908) のこのセクションを参照してください。
  • [Debug probes file] : 現在のデザインのデバッグ コアに関連するプローブ ファイルの名前とパスを指定します。
  • [Enable end of startup check] : プログラムが完了したときにデバイスのスタートアップ ステータスがチェックされます。オンにすると、ハードウェア マネージャーでプログラムがうまくいかなかった場合にエラー メッセージが表示されます。

[OK] をクリックすると、現在のデバイスの PROGRAM.FILE および PROBES.FILE プロパティが設定され、指定したビットストリームを使用してデバイスがプログラムされます。
