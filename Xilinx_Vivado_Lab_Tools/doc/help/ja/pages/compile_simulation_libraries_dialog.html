[Compile Simulation Libraries] ダイアログ ボックス

Vivado® Design Suite には、Vivado シミュレータで使用されるシミュレーション ライブラリ セットが含まれています。 Vivado シミュレータでは、ザイリンクス デバイスをターゲットにするデザインのビヘイビアー、論理、およびタイミング シミュレーションがサポートされます。しかし、サポートされるサードパーティ シミュレータを使用する場合は、ライブラリをコンパイルする必要があります。

[Compile Simulation Libraries] ダイアログ ボックスでは、シミュレータに関するシミュレータ、言語プリファレンス、インストール情報を指定できます。

  • [Simulator] : ライブラリをコンパイルするサードパーティ シミュレータを指定します。使用可能なシミュレータのリストは、実行している OS によって異なります。シミュレータの中には Linux でしか使用できず、Windows で実行している場合はリストされないものもあります。
  • [Language] : 出力するシミュレータ言語を指定します。シミュレータの中には特定の言語限定のものもありますが、それ以外では Verilog と VHDL の両方がサポートされます。混合言語シミュレータの場合は、Verilog および VHDL ライブラリがコンパイルできます。
  • [Library] : コンパイルするザイリンクス シミュレーション ライブラリを指定します。詳細は、『Vivado Design Suite ユーザー ガイド : ロジック シミュレーション』 (UG900) のこのセクションを参照してください。
  • [Family] : シミュレーション ライブラリをコンパイルするデバイス ファミリを指定します。ターゲットにするデバイス ファミリを選択する [Family Options] ダイアログ ボックスが開き、コンパイル済みライブラリのサイズを制限できます。
  • [Compiled library location] : コンパイルしたライブラリの結果を保存するディレクトリ パスを指定します。
  • [Simulator executable path] : シミュレータ実行ファイルのディレクトリを指定します。これはターゲット シミュレータが $PATH パスで定義されていない場合やパス定義を上書きする場合に必要です。
  • [Miscellaneous options] : Tcl コマンドの compile_simlib にさらにオプションを指定する場合に使用します。
  • [Overwrite the current pre-compiled libraries] : 同じ指定したディレクトリにある既存のライブラリに置き換えられます。
  • [Compile 32-bit libraries] : 32 ビットの実行可能モデルを指定するために使用します。デフォルトでは、ライブラリは 64 ビット モードでコンパイルされます。
  • [Verbose] : 現在のメッセージの制限が解除され、Tcl コマンドからのすべてのメッセージが表示されます。
  • [Command] : このダイアログ ボックスで設定したシミュレーション ライブラリをコンパイルする Tcl コマンドが表示されます。このコマンドを直接変更すると、[OK] をクリックしたときに実行されるコマンドを変更できます。
