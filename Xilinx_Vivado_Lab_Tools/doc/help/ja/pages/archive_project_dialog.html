[Archive Project] ダイアログ ボックス

[Archive Project] ダイアログ ボックスでは、現在のプロジェクトの ZIP ファイル アーカイブを作成できます。ZIP ファイルは、リビジョン管理に使用したり、チーム メンバーと共有したり、デザインを移動したりする際に使用できます。[Archive Project] コマンドでは、デザイン階層が検証されて、リモート ソース ファイルとインクルード ファイルがローカル ディレクトリにコピーされ、デザイン制約がコピーされ、合成、シミュレーション、インプリメンテーション run の結果がオプションでコピーされます。

  • [Archive name] : 作成するプロジェクト アーカイブ名を指定します。
  • [Archive location] : プロジェクト アーカイブのディレクトリを指定します。
  • [Archive will be created at] : 作成されるプロジェクト アーカイブのファイル名とパスが表示されます。
  • [Temporary location] : プロジェクト アーカイブの作成中にファイルおよびフォルダーをコピーして管理するための一時ファイル ディレクトリを指定します。一時ディレクトリは、アーカイブ処理が終了したら空になります。Vivado では、現在の作業ディレクトリ内にデフォルトで一時ディレクトリが作成されます。
    ヒント: このオプションは Windows のみで使用でき、オペレーティング システムのファイル名の制限をなくすために使用できます。
  • [Include configuration settings] : アーカイブの config_settings フォルダーのアーカイブ ファイルに初期化 Tcl コマンド (init.tcl) を追加します。
  • [Include run results] : 合成またはインプリメンテーション run の結果が含まれます。このオプションを使用すると、プロジェクト アーカイブの容量は大幅に増加しますが、必要であれば合成ネットリストおよびインプリメント済みデザインも保持されます。

[OK] をクリックすると、プロジェクト アーカイブが作成されます。
