[Project Settings] ダイアログ ボックス : [IP] ページ

[Project Settings] ダイアログ ボックスでは、デザイン フローを通して使用されるさまざまな設定に対して値を指定できます。設定ページは、合成、シミュレーション、インプリメンテーションなどのデザイン フローの手順別に分類されています。

[IP] ページには、現在のプロジェクトを設定するための次のタブが含まれます。

  • [General] タブ
  • [Repository Manager] タブ
  • [Packager] タブ

[General] タブ

[General] タブには、次のフィールドが含まれます。

  • [Use Core Containers for IP] : IP 出力ファイルに圧縮コア コンテナーの使用を指定して、IP 管理用の 1 つのまとまったファイルを作成します。詳細は、『Vivado Design Suite ユーザー ガイド : IP を使用した設計』 (UG896) のこのセクションを参照してください。
  • [Use Precompiled IP Simulation Libraries] : プリコンパイル シミュレーション ライブラリを使用することで、出力ファイルを生成する際の時間とディスク空間を節約できます。これをオフにすると、スタティック IP シミュレーション ライブラリがデザインごとに作成されます。
  • [Automatically Generate Simulation Scripts for IP] : 出力ファイルを生成する際に各 IP に対してシミュレーション スクリプトが生成されます。各 IP に対してシミュレーション スクリプトを生成するのに時間がかかりすぎる場合や、IP を個別にシミュレーションしない場合は、このオプションをオフにします。
  • [Generate Log File] : IP アップグレード プロセスの詳細をレポートするログ ファイルが作成されます。デフォルトのログ ファイル名は ip_upgrade.log です。
  • [Log File Location] : IP をアップグレードする際に生成するログ ファイルのディレクトリを指定します。デフォルトのディレクトリは、現在のプロジェクト内または最後に指定したディレクトリになります。
  • [Default IP Location] : 現在のプロジェクトに追加する新しい IP カスタマイズを書き込むデフォルト ディレクトリを指定します。デフォルトのディレクトリは現在のプロジェクト ディレクトリ構造内にありますが、ディレクトリをプロジェクト構造外に指定すると、再利用およびソース管理をしやすくできます。詳細は、『Vivado Design Suite ユーザー ガイド : IP を使用した設計』 (UG896) のこのセクションを参照してください。
  • [IP Cache] : IP キャッシュにアウト オブ コンテキスト IP の合成結果を格納して、同じカスタマイズ オプションで IP を作成する際にこれらの結果を利用できるようにします。複数のプロジェクトを同じ IP キャッシュ ディレクトリに指定すると、同じ IP カスタマイズの合成が実行し直されないようにできます。詳細は、『Vivado Design Suite ユーザー ガイド : IP を使用した設計』 (UG896) のこのセクションを参照してください。
    • [Cache scope] : 合成キャッシュをディスエーブルにするか、このプロジェクト特有にするか、または多くのプロジェクトで使用するようにリモートに設定されます。
    • [Cache location] : リモート キャッシュのディレクトリ ロケーションを指定します。

[Repository Manager] タブ

[Repository Manager] タブでは、Vivado IP カタログをユーザー定義またはサードパーティ IP のリポジトリとして追加するためのフォルダーを指定できます。このタブには、次のフィールドが含まれます。

Add IP リポジトリとして追加するフォルダーを選択するためのファイル ブラウザーが開きます。Vivado ツールでは選択したフォルダーがスキャンされ、見つかった IP およびインターフェイスの数をまとめるための [Add Repository] ダイアログ ボックスが開きます。
Remove リポジトリから選択したフォルダーが削除されます。
Move up 選択したフォルダーが表示リストの上の方へ移動されます。
Move down 選択したフォルダーが表示リストの下の方へ移動されます。
重要: IP リポジトリで IP が重複している場合は、警告メッセージが表示されます。リストされている最新の IP リポジトリからの IP でそれ以前の IP 定義が上書きされます。

[Packager] タブ

[Packager] タブでは、IP をパッケージする際に使用するデフォルト設定を指定できます。詳細は、『Vivado Design Suite ユーザー ガイド : カスタム IP の作成およびパッケージ』 (UG1118) のこのセクションを参照してください。

  • [Default Value] : ユーザー定義 IP のデフォルト値を指定します。
    • [Vendor] : カスタム パッケージ IP に適用するベンダー名を指定します。
    • [Library] : IP に割り当てるライブラリ名を指定します。
      ヒント: [Vendor] および [Library] がその特定のパッケージ IP に割り当てられた [Name] および [Version] と共に使用され、その IP の固有の VLNV 識別子が定義されます。
    • [Category] : Vivado IP カタログで分類する IP のカテゴリを指定します。
    • [IP Location] : パッケージ IP を書き込むディレクトリを指定します。このディレクトリは、Repository Manager に追加する必要があります。
  • [After Packaging] : IP をパッケージした後のツール動作を指定します。
    • アーカイブを作成
    • IP を IP カタログへ追加
    • [IP Packager] ウィンドウを閉じる
  • [Edit IP in IP Packager] : IP をパッケージした後に Package IP プロジェクトをどう処理するかを指定します。
    • パッケージ後にプロジェクトを削除
  • [File Extensions to Filter on Add Directory] : IP ファイル グループを定義する際のファイルの拡張子を指定します。
