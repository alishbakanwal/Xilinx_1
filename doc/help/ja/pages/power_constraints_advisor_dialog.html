[Power Constraints Advisor] ダイアログ ボックス

消費電力制約アドバイザー (Power Constraint Advisor) からは、ツールで計算されたデザイン内のすべての制御信号のスイッチング アクティビティがレポートされます。制御信号には、イネーブルおよびリセット (リセット、セット、クリア、プリセット) が含まれます。スイッチング アクティビティが妥当であれば、消費電力解析が最も正確になります。

レポートの表を確認し、余分な時間アサートされる非アクティブなイネーブルおよびリセット信号といったクリティカルな制御信号に不正確なスイッチング アクティビティがあれば、修正します。レポートには、各制御信号のファンアウト (駆動されるリーフ レベルのプリミティブの数) も含まれます。ファンアウトが多い信号は、確認して修正する必要性が最も高くなります。 このような信号があると、デザインの大部分のダウンストリーム スイッチングがディスエーブルになってしまう可能性があり、消費電力のレポートがかなり不足な状態になってしまうからです。不正確なスイッチングを持つファンアウトの少ない信号の方が影響は少ないので、重要度は低くなります。[Deassert] ボタンを使用すると、簡単にすべてのリセットのスイッチング アクティビティを非アクティブに設定できます。

リストされているネットの横のチェック ボックスをオンにすると、そのネットを編集できます。フィールドが編集できる場合は、鉛筆マーク が表示されます。 これをクリックすると、選択したネットの [Static Probability] および [Toggle Rate] フィールドを変更できます。[OK] をクリックすると、Tcl コマンドの set_switching_activity を使用して選択したネットに変更した値が適用されます。[Power Constraint Advisor] は、次のように使用することをお勧めします。

  1. [Confidence] 列をクリックして LOW が一番上になるように並べ替えます。
  2. Ctrl キーを押しながら [Fanout] 列を 2 回クリックして、高い値から低い値の順に並べ替えます。
  3. 信頼度 LOW でファンアウト 200 を超える制御ネットすべてを確認して、新しい [Static Probability] と [Toggle Rate] を定義します。
  4. [OK] をクリックすると、デザインに制約が適用され、[Report Power] コマンドが再実行できます。

ツールバー メニューからは、次のコマンドを実行できます。

Search 検索パターンを入力する検索フィールドが開きます。検索パターンと一致するネットまたは値が表示されます。
Deassert Control Signal すべての制御信号の [Static Probability] と [Toggle Rate] を設定します。[Static Probability] は、アクティブ High 信号の場合は 0 に、アクティブ Low 信号の場合は 1 に変更されます。[Toggle Rate] は、すべての制御信号に対して 0 に変更されます。
ヒント: これは、BRAM イネーブル信号には適用されません。
Reset constraints to default 変更した値をリセットして、ダイアログ ボックスを元の値に戻します。
Schematic 選択したネットを含む回路図が開きます。

例 :

  • [Active high reset with Static Probability 0.9] : リセット が 90% の時間 High であること、つまりロード セルが 90% の時間リセットされ、多すぎることを意味します。スイッチング アクティビティを変更して、リセットがアクティブでないこと (より現実的な条件) を示します。リセットがアクティブ High の場合は [Static Probability] を 0、[Toggle Rate] を 0 に設定し、リセットがアクティブ Low の場合は [Static Probability] を 1、[Toggle Rate] を 0 に設定します。
  • [BRAM Enable with Static Probability 0 and Toggle Rate of 0] : BRAM が絶対にイネーブルにならないことを示します (BRAM の消費電力のレポートが不足します)。[Static Probability] を 0.25 に、[Toggle Rate] を 12.5 に設定して、25% のイネーブル レートなど、より合理的なスイッチング アクティビティを BRAM イネーブルに割り当ててください。

[OK] をクリックすると、デザインに制約が適用されます。
