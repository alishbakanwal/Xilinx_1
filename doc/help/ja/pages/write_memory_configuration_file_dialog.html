[Write Configuration Memory File] ダイアログ ボックス

Vivado ハードウェア マネージャの [Generate Configuration Memory File] コマンドを使用すると、特別なコンフィギュレーション メモリ ファイル (.cfgmem) が作成されます。 Vivado デバイス プログラマでは、このコンフィギュレーション メモリ ファイルが使用されて、JTAG を介して間接的にフラッシュ ベースのコンフィギュレーション メモリ デバイスがプログラムされます。このダイアログ ボックスでは、CFGMEM ファイルの書き込みに関するオプションを設定できます。

  • [Format] : コンフィギュレーション メモリ ファイルの形式を指定します。デフォルト フォーマットは MCS です。
  • [Memory Part] : コンフィギュレーション メモリ デバイスを指定します。
    ヒント: このフィールドには、[Add Configuration Memory Device] ダイアログ ボックスに基づいたハードウェア デバイスに関連するコンフィギュレーション メモリ デバイスが自動的に表示されますが、必要であれば別のコンフィギュレーション メモリ デバイスを指定することもできます。
  • [Custom Memory Size] : ターゲットのコンフィギュレーション メモリ デバイスのサイズ制限を MB で指定します。2 のべき乗で指定する必要があります。
  • [Filename] : 作成するファイルの名前を指定します。ファイルの最後は、そのファイルに対して選択した [Format] で自動的に決まります。
  • [Options]
    • [Interface] : フラッシュ メモリ デバイスをプログラムするのに使用するインターフェイス フォーマットを指定します。デフォルトのインターフェイスは SMAPx8 です。
      重要: インターフェイス フォーマットは、フラッシュ メモリ デバイスが正しくプログラムするためには重要で、選択した [Memory Part] でサポートされるフォーマットと同じである必要があります。
    • [Load bitstream files] : チェック ボックスをオンにすると、CFGMEM ファイルの一部として任意の数のビットストリーム ファイルを含めることができます。
      • [Start address] : ビットストリームからのビット パターンを書き込むための開始アドレスを指定します。16 進数値として指定します。
      • [Direction] : ビットを追加する場合に、開始アドレスからカウント アップするか、カウント ダウンするかを指定します。
      • [Bitfile] : 読み込むビットストリーム ファイルの名前とパスを指定します。
        ヒント: ファイルを指定すると、[Add] ボタン () および [Remove] ボタン () が表示されるようになり、その他のビットストリームを追加するか前に追加したファイルを削除できます。複数のビットストリーム ファイルを読み込むと、ファイルがデイジー チェーンで結合されます。
    • [Load data files] : チェック ボックスをオンにすると、CFGMEM ファイルの一部として任意の数のデータ ファイルを含めることができます。データ ファイルは、フラッシュ メモリ デバイスに追加のフォーマットなしでそのまま追加されます。
      • [Start address] : データ ファイルからのビット パターンを書き込むための開始アドレスを指定します。16 進数値として指定します。
      • [Direction] : データ ビットを追加する場合に、開始アドレスからカウント アップするか、カウント ダウンするかを指定します。
      • [Datafile] : 読み込むデータ ファイルの名前とパスを指定します。
        ヒント: ファイルを指定すると、[Add] ボタン () および [Remove] ボタン () が表示されるようになり、その他のデータ ファイルを追加するか前に追加したファイルを削除できます。複数のデータ ファイルを読み込むと、ファイルがデイジー チェーンで結合されます。
    • [Write checksum] : CFGMEM ファイルの 32 ビット チェックサムが算出されます。このオプションは、フラッシュ メモリに正しいデータがプログラムされたことを確認するために使用してください。チェックサム データが CFGMEM ファイルと同時に作成されるログ ファイル (.prm) に書き込まれます。
    • [Disable bit swapping] : 読み込まれたビットストリーム ファイルのバイトのデフォルトのビット スワップをディスエーブルにします。
      重要: デフォルトでは、元の入力 BIT ファイルのバイトと比較して、バイト内のビットがスワップされます。
    • [Overwrite] : 既存のファイルが上書きされます。

[OK] をクリックすると、cfgmem ファイルが作成されます。
