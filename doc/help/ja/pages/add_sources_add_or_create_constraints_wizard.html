[Add or Create Constraints] ページ

[Add or Create Constraints] ページでは、制約ファイルを追加するための既存の制約セットを指定したり、新しい制約セットを定義したりできます。制約セットは、制約ファイルまたは管理されていない Tcl スクリプトのコレクションで、デザインのプロパティおよび特性が定義されています。制約には、デザイン ロジックをデバイス リソースに割り当てる物理制約、ロジック エレメントをデバイスのエリアに割り当てるフロアプランおよび I/O プランニング制約、またはデザインのスキューおよび遅延制限を指定するタイミング制約を含めることができます。

リストされる順番にファイルから制約が読み出されるので、制約セットのファイル順は重要です。リストの最初で定義される制約は、後で定義される制約で上書きされる可能性があります。制約の優先順位の詳細は、『Vivado® Design Suite ユーザー ガイド : 制約の使用』 (UG903) のこのセクションを参照してください。

  • [Specify constraint set] : 制約を追加する制約セットを指定します。デフォルトでは、アクティブ制約セットが選択されます。別の既存制約セットを選択するか、プロジェクト用に新しい制約セットを作成することもできます。新しい制約セットを作成するには、[Create Constraint Set] をクリックし、[Create Constraint Set] ダイアログ ボックスで制約セットの名前を指定します。
    • [Make Active] : 選択した制約セットをプロジェクトのアクティブ制約セットにします。アクティブな制約セットには、現在のデザインに適用される制約すべてが含まれ、新しい制約が保存されます。このチェックボックスは、アクティブな制約セットを編集中には使用できません。
  • [Add Files] : ファイルを選択するダイアログ ボックスが開き、追加する制約ファイル (.xdc) または管理されていない Tcl スクリプト (.tcl) を選択できます。複数のファイルを選択した場合は、ファイル リストに指定した順に追加されます。
    ヒント: をクリックして [Add Files] をクリックしても同じ操作を実行できます。
  • [Create File] : [Create Constraints File] ダイアログ ボックスが開き、ファイル タイプ、ファイル名、ファイルを保存するディレクトリを指定できます。空のファイルが作成されるので、ファイルを開いて編集したり、Timing Constraints ウィザードを使用して制約を定義したり、またはデザインの配置配線を変更してプロジェクトまたは制約を保存することにより、制約をこのファイルに追加できます。
    ヒント: をクリックして [Create Files] をクリックしても同じ操作を実行できます。
  • [Remove] () : 現在のプロジェクトから選択したファイルを削除します。ファイルは現在のプロジェクトから削除されますが、ハード ドライブからは削除されません。
  • [Move Up] : ファイル リストでファイルが選択されている場合、上矢印ボタン () が使用できるようになり、リスト内でファイルを上に移動できます。
  • [Move Down] : ファイル リストでファイルが選択されている場合、下矢印ボタン () が使用できるようになり、リスト内でファイルを下に移動できます。
  • [Copy constraints file into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定した制約ファイルをコピーします。ローカルおよびリモート ソースの使用に関する詳細は、『Vivado® Design Suite ユーザー ガイド : システム レベル デザイン入力』 (UG895) のこのセクションを参照してください。
