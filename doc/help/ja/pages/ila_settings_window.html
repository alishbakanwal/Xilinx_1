[ILA Settings] ウィンドウ

このウィンドウは、[Hardware ILA] ダッシュボードに表示されるデフォルトの 5 つのウィンドウの 1 つで、現在のデバイスにプログラムされたした ILA (Integrated Logic Analyzer) デバッグ コアを選択して、オプションを設定できます。ILA コアを使用すると、デザインの特定の信号のプローブ、プログラムされたハードウェア イベントでのトリガー、ザイリンクス FPGA デバイスからのリアルタイムでのデータ キャプチャを実行できます。

  • [Trigger Mode] : デバッグ コアのトリガー イベントのタイプを指定します。[ILA Trigger Setup] ウィンドウで使用できるコマンドおよびオプションは、トリガー モードを [BASIC] タイプのトリガーにするか [ADVANCED] タイプのトリガーにするかによって異なります。
    • [BASIC_ONLY] : トリガー条件は、関連の各 ILA プローブの各値を評価するのに使用したブール式の結果です。
    • [BASIC_OR_TRIG_IN] : ILA コアが、プローブ値を考慮するブール式またはコアの TRIG_IN ポートによりトリガーされます。
    • [ADVANCED_ONLY] : ILA コアが、ユーザー定義のトリガー ステート マシン (TSM) で定義されたアドバンス トリガー機能を持つよう設定されます。
    • [ADVANCED_OR_TRIG_IN] : ILA コアは、TMS またはコアの TRIG_IN ポートによりトリガーされます。
    • [TRIG_IN_ONLY] : ILA コアは、コアの TRIG_IN ポートによりトリガーされます。
  • [TRIG OUT mode] : ILA コアの TRIG_OUT をいつどのように遷移するのかを指定します。ほかの ILA デバッグ コアの場合、TRIG_OUT 信号は入力トリガー メカニズムとして使用できます。
    • [DISABLED] : ILA コアの TRIG_OUT ポートがディスエーブルになります。これがデフォルトです。
    • [TRIGGER_ONLY] : トリガー条件が満たされたときにのみ TRIG_OUT ポートが遷移されます。
    • [TRIG_IN_ONLY] : TRIG_IN 信号が遷移したときにのみ TRIG_OUT ポートが遷移されます。ILA コアのチェーンでトリガー イベントを伝搬する際に使用します。
    • [TRIGGER_OR_TRIG_IN] : トリガー条件が満たされたとき、または TRIG_IN が遷移したときに TRIG_OUT ポートを遷移します。
  • [Capture Mode] : [ALWAYS] か [BASIC] に設定します。[ALWAYS] に設定すると、デバッグ コアのプローブのデータ サンプルが常にキャプチャおよび格納されます。[BASIC] に設定すると、条件が true の場合にのみデータ サンプルがキャプチャおよび格納されます。[BASIC] に設定した場合は、各プローブのキャプチャ条件が [ILA Capture Setup] ウィンドウで定義できます。
  • [Number of windows] : キャプチャされたサンプル データを格納するためのデータ ウィンドウ数を指定します。最初のトリガー イベントからのデータで最初のウィンドウが一杯になると、続くウィンドウそれぞれにはその後のトリガー イベントまたはキャプチャ条件で順にデータが格納されます。
  • [Window data depth] : 選択した ILA デバッグ コアがデータ ウィンドウでキャプチャ可能なデータ サンプル数を定義します。データの深さはデフォルトでその ILA デバッグ コアの MAX_DATA_DEPTH プロパティになり、コアが作成されたときに定義されます。データの深さは 1 からコアの MAX_DATA_DEPTH まで、2 のべき乗の整数として指定できます。
  • [Trigger position in window] : トリガー イベントよりも前にサンプル データをキャプチャできるようにする [Window data depth] に関連する整数値です。たとえば、データの深さが 1024 の場合、トリガー ポジションを 100 に設定すると、トリガー前の 100 個のデータ サンプルと残りのトリガー時およびトリガー後の 155 個のデータ サンプルをキャプチャできます。
  • [General Settings] : 現在の ILA デバッグ コアに対する [Refresh rate] (更新頻度) をミリ秒で指定します。
