[Report Timing Summary] ダイアログ ボックス

[Report Timing Summary] ダイアログ ボックスでは、サインオフ品質の包括的なタイミング レポートを生成して、さまざまな側面からデザインを検証できます。タイミング サマリ レポートには、タイミング チェック、クロック定義、同じソースおよびデスティネーション クロックを使用するクロック間のタイミング パス、クロック ドメインをまたがるクロック間のタイミング パス、[Report Timing] コマンド同様の詳細なタイミング パスなどが含まれます。

重要: タイミング サインオフのためのタイミング サマリ レポートは、デザインが完全に配線された後にのみ使用できます。

デフォルトでは、[Report Timing Summary] ダイアログ ボックスでクロック ドメインごとまたはパス グループごとに 10 個のワースト タイミング パスがレポートされます。これは、[Maximum number of paths per clock or path group] オプションを設定すると、パス グループごとにより多くのタイミング パスやより少ないタイミング パスが表示されるように変更できます。また、タイミング サマリ レポートには、デフォルトで各終点オブジェクトごとに 1 つのパスが表示されます。これは、1 つの終点にデフォルトではレポートされない複数のエラーになるタイミング パスを含めることができることを意味します。ただし、これは [Maximum number of worst paths per endpoint] オプションを設定すると変更できます。これら 2 つのオプションにより、終点ごとおよびパス グループごとにレポートされるエラーになるタイミング パスの数が決定されます。

[Report Timing Summary] ダイアログ ボックスには、次のタブと共有オプションが含まれます。

  • [Options] タブ
  • [Advanced] タブ
  • [Timer Settings] タブ
  • 共通オプション

[Options] タブ

[Options] タブには、レポートの出力を設定するための次のフィールドが含まれます。

  • [Path delay type] : タイミング レポートを実行する際に解析する遅延のタイプを指定します。最大遅延はタイミング パスのセットアップ解析用で、最小遅延はホールド解析用です。セットアップおよびホールド解析両方の min_max 遅延を選択します。
  • [Report Unconstrained paths] : 現在のデザインに含まれる制約が適用されていないパスのタイミングをレポートします。デフォルトでは、レポートに制約の付いたパスのみが含まれます。
  • [Report datasheet] : クロックに対する入力 I/O のセットアップおよびホールド タイム、クロックから出力パッドへの最大/最小遅延、入力/出力バスのスキューが追加されます。この情報は、[Report Datasheet] コマンドでも表示されます。
  • [Maximum number of paths per clock or path group] : レポートに表示されるパス数を指定します。デフォルトでは、10 個のワースト タイミング パス、またはパス グループごとにワースト パスがレポートされます。
  • [Maximum number of worst paths per endpoint] : 終点ごとにレポートされるタイミング パス数を指定します。この数は、クロックまたはグループごとのパス数によって制限されます。
  • [Display paths with slack less than] : 必要のないパスがレポートから削除されます。この設定はサマリ レポートには影響せず、詳細にリストされるパスにのみ影響します。
    ヒント: [Use default] をクリックすると、デフォルト設定が使用されます。
  • [Significant digits] : タイミング レポートでレポートされる値の有効桁数を指定します。デフォルト値は 3 です。

[Advanced] タブ

[Advanced] タブには、次のフィールドが含まれます。

  • [Report from cell] : 指定した階層セル インスタンスのレベルからレポートを実行します。デフォルトでは、レポートはデザイン全体に適用されます。指定したセル内で開始、終了するパス、またはセルに完全に含まれるパスのみがレポートされます。
  • [Show input pins in path] : レポートに表示されるタイミング パスの詳細な説明に入力ピンが含まれるようになります。
  • [Report unique pins] : ピンの固有のセットそれぞれを使用して 1 つのタイミング パスのみがレポートされます。
  • [Write results to file] : タイミング サマリ レポートの結果が指定ファイルに書き込まれます。デフォルトは [Overwrite] で、同じ名前の既存ファイルが上書きされます。[Append] を使用して、指定ファイルに新しいレポートが追加されます。
  • [Ignore command errors (quiet mode)] : [Report Timing Summary] コマンドを実行する際に、エラーが表示されないようになります。
  • [Suspend message limits during command] : このレポートを実行する際に定義可能なメッセージ制限が無視されます。

[Timer Settings] タブ

[Timer Settings] タブには、レポートを生成する前にタイミング エンジンを設定するための次のフィールドが含まれます。

  • [Interconnect] : タイミング解析で使用するインターコネクト遅延を指定します。インプリメント済みデザインからすべての配線済み遅延を使用したり、合成またはインプリメント済みデザインで見積もり遅延を使用したり、インターコネクト遅延を使用しないようにして、ロジック遅延のみを検証できます。
  • [Speed grade] : 実際にターゲット パーツを変更することなく、タイミング解析中にデバイスのスピード グレードを変更できます。
  • [Multi-corner configuration] : 現在のデザインの Slow および Fast タイミング コーナーを設定できます。製造プロセス、電圧、および温度 (PVT) の変数により、デバイスの遅延が決定され、まとめられてタイミング コーナーが定義されます。これは、Tcl コマンドの config_timing_corners に関連します。
    • [Slow] : Slow コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延が使用されます。タイミング解析からこのコーナーを除外するには、none を使用します。
    • [Fast] : Fast コーナーに対して解析する遅延のタイプを指定します。セットアップ解析に max 遅延、ホールド解析に min 遅延、セットアップおよびホールド解析両方に min_max 遅延が使用されます。タイミング解析からこのコーナーを除外するには、none を使用します。
  • [Disable flight delays] : I/O 遅延の算出でパッケージ遅延が無視されます。フライト遅延は、パッケージ ピンとダイ パッド間で発生するパッケージ遅延です。これは、Tcl コマンドの config_timing_analysis に関連しています。

共通オプション

共通オプションのタブはありませんが、次はダイアログ ボックス内に常に表示されます。

  • [Results name] : Vivado® IDE の [Timing Results] ウィンドウのレポートに適用する名前を指定します。
  • [Command] : レポートを生成するために使用した Tcl コマンドと指定したオプションすべてが表示されます。
  • [Open in a new tab] : Vivado IDE の [Timing Results] ウィンドウに新しいタブでタイミング サマリが開きます。オフの場合は、新しいレポートで最後に開いたレポートが上書きされます。
  • [Open in Timing Analysis layout] : Vivado IDE の現在のレイアウト ビューがタイミング解析用のレイアウト ビューに変更されます。詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。

[OK] をクリックすると、表示された Tcl コマンドが実行されてレポートが生成されます。
