タイミング結果

[Timing Results] ウィンドウには、[Report Timing Summary] および [Report Timing] コマンドの結果が表示されます。このウィンドウは、2 つのセクションに分割されます。左側のツリー ビューには、サマリ、設定、実行したタイミング チェックのリスト、およびさまざまなパス レポートが表示されます。右側には、レポートまたは選択したカテゴリのサマリ結果が表示されます。タイミング値は、ナノ秒で表示されます。

[Timing Results] ウィンドウは表形式のウィンドウで、タイミング パスのタイミング結果およびクロック オブジェクトが列で表示されます。列のヘッダーを右クリックすると、列を表示および非表示にするコマンドと列幅を管理するコマンドが表示されます。表の情報は、列ヘッダーを 1 回または 2 回クリックすると並び替えることができます。詳細は、『Vivado® Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。

右クリック メニューには、選択したタイミング パスを詳しく調べるためのコマンドが含まれます。 [Schematic] をクリックすると選択したパスの回路図が開き、[View Path Report] をクリックすると [Timing Path Report] ウィンドウが開いてパスの詳細が表示されます。

[Timing Results] ウィンドウには、ツリー表示と表形式のツールバー メニューが含まれます。

  • タイミングの表のツールバーには、次のコマンドが表示されます。
    Search 検索パターンを入力する検索フィールドが開きます。検索パターンと一致するタイミング チェックまたはパスが表示されます。
    Collapse all ツリー ビューで表示されるタイミング チェックの階層の展開が閉じられます。
    Expand all ツリー ビューで表示されるタイミング チェックの階層が展開されます。
    Previous 前に選択していたタイミング チェックに戻ります。これを使用すると、検証するために選択したタイミングチェックをすばやくスクロールできます。
    Next [Previous] ボタンで戻る前に表示していたタイミング チェックが表示されます。これを使用すると、検証するために選択したタイミングチェックをすばやくスクロールできます。
    Show only failing checks 違反がないとレポートされたタイミング チェックが非表示になります。
    Rerun report 元のタイミング レポートを生成するのに使用した report_timing_summary または report_timing コマンドが実行し直されます。これは、デザインを修正またはアップデートした後に同じレポートを生成し直すために使用できます。
    ヒント: タイミング レポートを生成するのに使用した元のコマンドは、タイミング レポートの [General Information] セクションにリストされます。
    Save タイミング レポートを RPX ファイルに保存します。 このファイルを読み込み直すと、タイミング解析を実行し直さなくてもタイミングを再評価できます。
  • タイミングの表のツールバーには、次のコマンドが表示されます。
    Search 検索パターンを入力する検索フィールドが開きます。検索パターンと一致するタイミング チェックまたはパスが表示されます。
    Show only failing paths 違反がないとレポートされたタイミング パスが非表示になります。
    Remove selected path タイミング レポートから現在選択しているタイミング パスが削除されます。
    Schematic 選択したタイミング パスを含む回路図が開きます。
    Mark [Device] または [Netlist] ウィンドウなどのその他のウィンドウでタイミング パスをマークして、見つけやすくします。
    ヒント: 始点は緑色、終点は赤色、通過点は黄色で表示されます。
    Select Clock Paths when Timing Paths are selected ソースおよびデスティネーション クロック パスをデータパスと一緒に選択します。
