Partial Reconfiguration ウィザード

重要: このウィザードを使用する前に、まず [Tools] → [Enable Partial Reconfiguration] をクリックするか、プロジェクトで PR_FLOW プロパティを TRUE に設定し、[Create a Partition Definition] コマンドを実行しておく必要があります。

パーシャル リコンフィギュレーションでは、スタティック ロジックと RM の組み合わせに基づいて、リコンフィギャラブル モジュール (RM) とパーティション定義 (partitionDef) が入れ替えられて、独自のデザイン コンフィギュレーションが作成されます。PR デザイン フローでは、PR コンフィギュレーションをそれぞれインプリメンテーションする必要があります。 この結果、RM のパーシャル ビットストリームは作成されますが、統合された各コンフィギュレーションのビットストリーム全体は作成されません。

Partial Reconfiguration ウィザードでは、前に作成した partitionDef にプラグインする RM を定義し、PR コンフィギュレーションを定義し、PR コンフィギュレーションのインプリメンテーション run を定義する手順を実行できます。このウィザードに含まれるページは次のとおりです。

  • [Edit Reconfigurable Modules] ページ
  • [Edit Configurations] ページ
  • [Edit Configuration Runs] ページ
  • [Partial Reconfiguration Summary] ページ

[Next] をクリックして続行します。
