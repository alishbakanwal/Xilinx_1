[Packaging Your Block Design] ページ

  • [IP Location] : パッケージ IP を出力するディレクトリを指定します。新しい IP を別のプロジェクトで使用するには、このディレクトリを IP リポジトリとして Vivado® IP カタログに追加する必要があります。Repository Manager に関する詳細は、『Vivado Design Suite ユーザー ガイド : IP を使用した設計』 (UG896) のこのセクションを参照してください。
  • [Packaging IP in the project] : 次のオプションでは、ブロック デザインで使用された IP が新しく作成された最上位 IP の一部としてどのようにパッケージされるかを指定します。
    • [Include .xci files] : パッケージした最上位 IP に XCI ファイルのみを含めます。オンにすると、このIP を別のデザインで使用するとき、パッケージ IP で使用されるすべての IP の必要な出力ファイルを生成する必要があります。さらに、出力ファイルを生成するには、使用されている IP を Vivado IP カタログ に含まれる最新バージョンにアップデートする必要があります。ブロック デザインに含まれる IP のアップグレードに関する詳細は、『Vivado Design Suite ユーザー ガイド : IP インテグレーターを使用した IP サブシステムの設計』 (UG994) のこのセクションを参照してください。
      ヒント: カスタム IP に含まれる IP をアップデートする必要があると、カスタム IP での変更管理で問題が発生する可能性があります。
    • [Include IP generated files] : 含まれる IP をサポートするのに必要な出力ファイルすべてが含まれます。 これらは、パッケージした最上位 IP を使用したシミュレーションおよびインプリメンテーションでサポートされます。オンにすると、新しいパッケージ IP にはすべてが含まれているので、ほかのデザインで使用するために IP をアップグレードする必要がありません。これにより、含まれている IP カスタマイズのリファレンスが削除され、IP が新しくパッケージされる最上位 IP の階層プロジェクト ソース ファイルとして処理されます。
