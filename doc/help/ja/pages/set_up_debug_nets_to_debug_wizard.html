[Nets to Debug] ページ

Set Up Debug ウィザードの [Nets to Debug] ページでは、デバッグする現在のデザインのネットを指定できます。その他のウィンドウでネットをクリックして、[Nets to Debug] のリストに追加することもできます。[Nets to Debug] ページは、[Netlist] ウィンドウや [Schematic] ウィンドウなど、Vivado® IDE のほかのウィンドウでネットを選択して、これらのネットを [Nets to Debug] のリストに追加している間も開いたままになります。

ヒント: MARK_DEBUG プロパティの付いたネットは自動的にリストに追加されます。Set Up Debug ウィザードを起動したときに Vivado IDE で選択していたネットも [Nets to Debug] のリストに追加されます。

ネットがデバッグ リストに追加されると、Vivado Design Suite でそのネットに使用するのに最適なクロック ドメインが自動的に決定されます。クロック ドメインは、ILA にその ILA 上のネットと同期したクロックが供給されるようにするには重要で、これによりプローブされたネットに対してサイクル精度の高いデータが取得できます。共通のクロック ドメインを持つネットは、1 つの ILA コアからプローブできます。異なるクロック ドメインにあるネットには、別々の ILA コアが必要です。 これらは、Set Up Debug ウィザードで自動的に挿入されます。ネットを選択して Vivado ツールで自動的に選択されたものからクロック ドメインを変更することができます。ウィザードでは、プローブされるネットに対して選択されたクロック ドメインによって、クロック ドメインごとに固有の ILA コアが作成されます。

重要: ILA に選択するクロックは、フリーランニング クロックである必要があります。ときどきしかアクティブにならないクロックは、クロックがアクティブになったときにのみ動作がトリガーされてデータがキャプチャされます。

ILA プローブは、リストの各ネットまたは各バスの ILA コアにも追加されます。スカラー ネットのプローブ幅は 1 です。バスのプローブ幅は、そのバスの幅と同じです。 Vivado ツールでは、信号に対して最適なプローブ幅が自動的に決定されます。特定の信号に対するプローブ タイプを設定することもできます。プローブは、信号の動作をトリガーして、信号からデータをキャプチャできます。プローブは、動作をトリガーするかデータをキャプチャするためのみに設定することもできます。

ローカル ツールバーには、次のコマンドが含まれます。

Search 検索パターンを入力する検索フィールドが開きます。
Collapse all デバッグ ネットの階層が閉じられます。
Expand all デバッグ ネットの階層が展開されます。
Scroll [Netlist] ウィンドウや [Schematic] ウィンドウなど、別のウィンドウでネットを選択すると、選択した信号までスクロールされます。
Filter 現在クロック ドメインが定義されていないネットがフィルターされます。割り当てられていないネットのみを表示できます。
Select Clock Domain [Select Clock Domain] ダイアログ ボックスを開いて、選択したネットまたはバスに割り当てられたクロック ドメインを選択または変更できます。
Add selected nets [Netlist] ウィンドウや [Schematic] ウィンドウなど、別のウィンドウで現在選択されているネットを [Nets to Debug] のリストに追加します。
Remove nets 現在選択しているネットを [Nets to Debug] のリストから削除します。
