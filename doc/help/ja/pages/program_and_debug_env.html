Program and Debug 環境

Program and Debug 環境は、Flow Navigator で [Hardware Manager] をクリックするか、[Tcl Console] ウィンドウで open_hw を実行すると開きます。Program and Debug 環境を作成するには、Open New Hardware Target ウィザードを使用してハードウェア サーバーとハードウェア ターゲットを接続する必要があります。

Program and Debug 環境が既に開いている場合は、Flow Navigator の [Program and Debug] をクリックするとアクティベートできます。

Program and Debug 環境からは、次が実行できます。

  • ビットストリーム設定とデバイス コンフィギュレーション プロパティを指定し、ハードウェア ターゲットをプログラムするためのビットストリームを生成します。
  • ローカルまたはリモートで実行されるハードウェア サーバーに接続し、ハードウェア ターゲットを開きます。
  • JTAG を介してザイリンクス デバイスを直接プログラムします。
  • ILA (Integrated Logic Analyzer) を使用してデザインのインシステム デバッグをデバイスでリアルタイムで実行し、実行しているデバイスから信号を監視してデータをキャプチャします。
  • Virtual Input/Output (VIO) コアを使用して、内部 FPGA 信号を駆動し、リアルタイムでイベントをトリガーします。
  • IBERT (Integrated Bit Error Ratio Tester) Serial Analyzer デザインを使用して、高速シリアル I/O リンクを測定して最適化します。
  • JTAG-to-AXI Master コアを使用して、さまざまな AXI フルおよび AXI Lite スレーブ コアとやりとりする AXI トランザクションを生成し、AXI 信号をデバッグ/駆動します。
  • フラッシュ ベースのコンフィギュレーション メモリ デバイスを間接的にプログラムして、デザインのブート デバイスを作成します。
