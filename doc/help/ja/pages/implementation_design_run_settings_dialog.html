[Implementation Design Run Settings] ダイアログ ボックス

[Implementation Design Run Settings] ダイアログ ボックスでは、選択したインプリメンテーション run に使用するストラテジとオプションを定義できます。選択したストラテジのデフォルト オプションは表形式で表示され、カスタマイズするストラテジのオプションや値が変更できるようになっています。

  • [Run report methodology after routing] : オンにすると、配線プロセスの最後に設計手法チェックが実行され、タイミングまたはプロセス エラーが検出されるようになります。
  • [Incremental Compile] : 次のインプリメンテーション run のリファレンスとして使用するデザイン チェックポイント (DCP) を指定します。基準デザインは、通常はデザインの合成、配置、配線が完了した以前のバージョンです。ただし、部分的に配置配線されただけのチェックポイントを使用することもできます。詳細は、『Vivado Design Suite ユーザー ガイド : インプリメンテーション』 (UG904) のこのセクションを参照してください。
  • [Strategy] : 定義済み Vivado® Design Suite ストラテジを指定するか、選択したストラテジのオプションを変更して独自のストラテジを作成できます。ストラテジとは、合成またはインプリメンテーション run を構成する定義済みの Tcl コマンドの設定です。ストラテジの設定やオプションは、特定の目標を達成させるためにそれぞれ変更できます。合成およびインプリメンテーション コマンドの詳細と、使用可能なオプションと指示子については、『Vivado Design Suite Tcl コマンド リファレンス ガイド』 (UG835) で次のコマンドを参照してください。
    • synth_design
    • opt_design
    • power_opt_design
    • place_design
    • phys_opt_design
    • route_design
    • write_bitstream
    ヒント: 変更されたストラテジには、名前にアスタリスク (*) が付きます。[Save Strategy As] ボタン () をクリックすると、そのストラテジを新しいユーザー定義のストラテジとして保存できます。
  • [Description] : 選択したストラテジの目的のサマリを簡単に記述しておきます。この部分は、ユーザー定義のストラテジを保存する際に定義できます。
  • [Options] : 選択したインプリメンテーション ストラテジで使用可能なオプションが表形式で表示されます。表のオプションを選択すると、そのオプションの詳細が表示されます。[Options] の表では、インプリメンテーション ストラテジは、最適化、配置、および配線プロセスなどのさまざまな段階に分割されます。詳細は、『Vivado Design Suite ユーザー ガイド : インプリメンテーション』 (UG904) のこのセクションを参照してください。選択したストラテジの機能の中には、[Options] の表の設定を変更すると、変更できるものもあります。[Options] の表の [More Options] フィールドを使用すると、各コマンドにその他のコマンド オプションを追加できます。
    重要: インプリメンテーション ストラテジの多くで -directive オプションが使用されて、インプリメンテーションが設定されます。デフォルトではないオプションを使用するストラテジの場合、[Options] の表の設定を変更すると、インプリメンテーション中にエラーになる可能性がありますので、表では変更しないようにしてください。
