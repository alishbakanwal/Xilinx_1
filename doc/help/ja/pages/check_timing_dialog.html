[Check Timing] ダイアログ ボックス

このコマンドでは、現在のデザインのタイミング制約に対してポート、ピン、パスなどのデザイン エレメントがチェックされます。このコマンドは [Report Timing] コマンドを実行する前にデザイン データおよびタイミング制約に問題がないかどうかを確認するために使用します。

  • [Results name] : 作成するポートの名前を指定します。
  • [Timing Checks to Perform] : 実行されるチェックのリストが表示され、必要に応じてオン/オフにできます。タイミング チェックの詳細な説明については、関連項目の資料を参照してください。
  • [Write results to file] : [Check Timing] コマンドの結果が指定ファイルに書き込まれます。デフォルトは [Overwrite] で、同じ名前の既存ファイルが上書きされます。[Append] を使用して、指定ファイルに新しいレポートを追加します。
  • [Shows detailed information about potential problems] : 検出された問題の詳細も含め、実行したチェックからの詳細な結果が表示されます。
  • [Ignore command errors (quiet mode)] : コマンドをメッセージを表示せずに実行します。
  • [Command] : レポートを生成するために使用した Tcl コマンドと指定したオプションすべてが表示されます。
  • [Open in a new tab] : Vivado IDE の [Timing Results] ウィンドウに新しいタブで [Check Timing] の結果を開きます。オフの場合は、新しいレポートで最後に開いたレポートが上書きされます。
  • [Open in Timing Analysis layout] : Vivado IDE の現在のレイアウト ビューをタイミング解析用のレイアウト ビューに変更します。詳細は、『Vivado Design Suite ユーザー ガイド : Vivado IDE の使用』 (UG893) のこのセクションを参照してください。
