[Constraints Summary] ページ

タイミング制約ウィザードのサマリ ページには、定義されてデザインに追加できるようになった制約が表示されます。[Finish] をクリックすると、制約がターゲット XDC ファイルに保存され、ウィザードが閉じます。これらの制約は、ウィザードで次のページに進むたびに、インメモリ デザインに適用されますが、[Cancel] をクリックするとインメモリの制約も削除されます。

  • [View Timing Constraints] : [Timing Constraints] ウィンドウが開き、記述された制約が確認できます。
  • [Create Timing Summary Report] : 新しく定義されたタイミング制約を使用して、現在のデザインに対して [Report Timing Summary] コマンドが実行されます。入力した周期または I/O 遅延制約を満たすのが困難すぎる場合は、タイミング違反がレポートされることがあります。
  • [Create Check Timing Report] : 新しく定義されたタイミング制約を使用して現在のデザインに対して [Check Timing Report] コマンドが実行されます。
  • [Create DRC Report using only timing checks] : タイミング チェックのみを使用して現在のデザインに対して [Report DRC] コマンドが実行されます。

『UltraFast™ 設計手法ガイド (Vivado® Design Suite 用)』 (UG949) で説明されるように、タイミング制約を定義するプロセスは複数の段階に分けることができます。制約の定義を完了するには、タイミング制約ウィザードを何度が実行する必要があることもあります。
