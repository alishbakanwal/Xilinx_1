[Add Reconfigurable Module] ダイアログ ボックス

このダイアログ ボックスでは、名前、タイプ、モジュールの最上位階層を指定し、パーティション定義 (partitionDef) に関連付けて、新しいリコンフィギャラブル モジュール (RM) を定義して追加できます。

  • [Partition Definition] : RM を関連付ける partitionDef を指定します。
  • [Reconfigurable Module] : RM の名前をテキスト文字列で入力します。
  • [Top module name] : RM に関連付ける階層セルを指定します。これを指定すると、関連するソース ファイルで RM を定義する階層セルを指定できます。
  • [Sources have been synthesized] : RM のデザイン ソースに合成後のネットリスト (DCP または EDIF ネットリストのいずれか) があることを指定します。デザイン ソースが合成後のネットリストではない場合、アウト オブ コンテキスト (OOC) 合成 run がデザインに追加される RM ごとに作成されます。
このダイアログ ボックスでは、RM のソースの内容を定義するファイルを追加および削除することもできます。
ヒント: 新しい RM はモジュールのファイル内容を指定しなくても定義できますが、後でモジュールにデザイン ソース ファイルを追加する必要があります。
  • [Add Files] () : ファイル参照ダイアログ ボックスが開き、モジュールに含めるデザイン ソース ファイルを任意の数だけ選択できます。複数のファイルを選択すると、指定した順番でファイル リストに追加されます。
  • [Add Directories] () : ファイル参照ダイアログ ボックスが開き、デザイン ソース ファイルを検索するディレクトリを任意の数だけ選択できます。指定したディレクトリ内でデザイン ソース ファイルが検索されモジュールに追加されます。
  • [Remove] () : モジュールから選択したファイルが削除されます。
  • [Scan and add RTL include files into project] : オンにすると追加した RTL ファイルがスキャンされ、参照した Verilog のインクルード ファイルがモジュールに追加されます。
  • [Copy sources into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定したデザイン ソース ファイルをコピーできます。ローカルおよびリモート ソースの使用に関する詳細は、『Vivado® Design Suite ユーザー ガイド : システム レベル デザイン入力』 (UG895) のこのセクションを参照してください。
  • [Add sources from subdirectories] : [Add Directories] で指定したディレクトリのサブディレクトリに含まれるソース ファイルがすべて追加されます。

[OK] をクリックし、新しいリコンフィギャラブル モジュール (RM) を定義して、ソース ファイルに追加します。[Cancel] をクリックすると、RM を保存せずにウィザードが閉じます。
