[Program Configuration Memory Device] ダイアログ ボックス

Vivado ハードウェア マネージャーの [Write Configuration Memory File] コマンドを使用すると、特別なコンフィギュレーション メモリ ファイル (CFGMEM) が作成されます。 Vivado デバイス プログラマでは、このコンフィギュレーション メモリ ファイルが使用されて、JTAG を介して間接的にフラッシュ ベースのコンフィギュレーション メモリ デバイスがプログラムされます。コンフィギュレーション メモリ デバイスは、このダイアログ ボックスで指定したオプションを使用してプログラムできます。

  • [Memory Device] : コンフィギュレーション メモリ デバイスを指定します。
    ヒント: このフィールドには、[Add Configuration Memory Device] ダイアログ ボックスに基づいて、現在のハードウェア デバイスに関連するコンフィギュレーション メモリ デバイスが自動的に表示されますが、必要であれば別のコンフィギュレーション メモリ デバイスを指定することもできます。
  • [Configuration file] : フラッシュ メモリ デバイスをプログラムするのに使用する CFGMEM ファイルの名前とパスを指定します。
  • [PRM file] : CFGMEM ファイルと一緒に作成されるログ ファイルの名前とパスを指定します。[Write Configuration Memory File] ダイアログ ボックスの [Write checksum] オプションをオンにした場合、このログ ファイルにチェックサム データが含まれます。
  • [State of non-config mem I/O pins] : CFGMEM ファイルで未使用の I/O ピンのプル タイプ終端スキームを指定します。
    重要: これは、[Edit Device Properties] ダイアログ ボックスで指定した BITSTREAM.CONFIG.UNUSEDPIN プロパティ (デフォルト値は PULLDOWN) と同じにする必要があります。詳細は、『Vivado Design Suite ユーザー ガイド : プログラムおよびデバッグ』 (UG908) のこのセクションを参照してください。
  • [Program Operations]
    • [Address range] : 次に示すように、コンフィギュレーション メモリ デバイスの全アドレス範囲が削除、ブランク チェック、プログラム、および検証されるか、CFGMEM ファイルで必要とされるアドレス範囲だけがプログラムされます。
    • [Erase] : プログラム サイクルの一部としてコンフィギュレーション メモリ デバイスが削除されます。
    • [Blank Check] : プログラム サイクルの一部としてコンフィギュレーション メモリ デバイスのブランク チェックが実行され、完全に削除されるようになります。
    • [Program] : コンフィギュレーション メモリ デバイスを指定の CFGMEM ファイルでプログラムします。
    • [Verify] : コンフィギュレーション メモリ デバイスの内容が CFGMEM ファイルと一致するかどうかが検証されます。
    • [Verify Checksum] : [PRM file] に指定したチェックサム値に対して、コンフィギュレーション メモリ デバイスにプログラムされたデータのチェックサムが比較されます。
  • [SVF Options]
    • [Create SVF Only (no program operations)] : ザイリンクス デバイスとフラッシュ コンフィギュレーション メモリのリモート プログラム用に SVF ファイルが生成されます。SVF は業界標準のファイル フォーマットで、デバイス チェーンにシフト入力する必要のある情報を記述することにより JTAG チェーンの操作を記述するために使用されます。
      ヒント: SVF ファイルは、実際にコンフィギュレーション メモリ デバイスをプログラムする代わりに作成されます。
    • [SVF File] : 作成する SVF ファイルの名前とパスを指定します。

[OK] をクリックすると、コンフィギュレーション メモリ デバイスがプログラムされるか、SVF ファイルが作成されます。
