[Add Constraints] ページ (オプション)

[Add Constraints] ページでは、既存の制約セットを指定して制約ファイルを追加できます。制約セットは、制約ファイルまたは管理されていない Tcl スクリプトのコレクションで、デザインのプロパティおよび特性が定義されます。制約には、デザイン ロジックをデバイス リソースに割り当てる物理制約、ロジック エレメントをデバイスのエリアに割り当てるフロアプランおよび I/O プランニング制約、またはデザインのスキューおよび遅延制限を指定するタイミング制約を含めることができます。

リストされる順番にファイルから制約が読み出されるので、制約セットのファイル順は重要です。リストの最初で定義される制約は、後で定義される制約で上書きされる可能性があります。制約の優先順位の詳細は、『Vivado® Design Suite ユーザー ガイド : 制約の使用』 (UG903) のこのセクションを参照してください。

  • [Add Files] : ファイルを選択するダイアログ ボックスが開き、追加する制約ファイル (.xdc) または管理されていない Tcl スクリプト (.tcl) を選択できます。複数のファイルを選択した場合は、ファイル リストに指定した順に追加されます。
    ヒント: をクリックして [Add Files] をクリックしても同じ操作を実行できます。
  • [Create File] : [Create Constraints File] ダイアログ ボックスが開き、ファイルの種類、ファイル名、ファイルを保存するディレクトリを指定できます。空のファイルが作成されます。ファイルを開いて編集したり、Timing Constraints ウィザードを使用して制約を定義したり、またはデザインの配置配線を変更してプロジェクトまたは制約を保存することにより、制約をこのファイルに追加できます。
    ヒント: をクリックして [Create Files] をクリックしても同じ操作を実行できます。
  • [Remove] () : 現在のプロジェクトから選択したファイルが削除されます。ファイルは現在のプロジェクトから削除されますが、ハード ドライブからは削除されません。
  • [Move Up] : ファイル リストでファイルが選択されている場合、上矢印ボタン () が使用できるようになり、リスト内でファイルを上に移動できます。
  • [Move Down] : ファイル リストでファイルが選択されている場合、下矢印ボタン () が使用できるようになり、リスト内でファイルを下に移動できます。
  • [Copy constraints file into project] : ローカル プロジェクト ディレクトリの sources フォルダーに指定した制約ファイルをコピーできます。ローカルおよびリモート ソースの使用に関する詳細は、『Vivado® Design Suite ユーザー ガイド : システム レベル デザイン入力』 (UG895) のこのセクションを参照してください。
