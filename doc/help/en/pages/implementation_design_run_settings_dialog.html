Implementation Design Run Settings

The Implementation Design Run Settings dialog box lets you define the strategy as well as customize the options for the selected implementation run. The default options for the selected strategy are displayed in a table, and you can modify options or values to customize the strategy.

  • Run report methodology after routing: Enable this checkbox to run methodology checks at the end of the routing process to identify any timing or process errors.
  • Incremental Compile: Specifies a design checkpoint (DCP) to use as a reference for the next implementation run. The reference design is usually an earlier iteration or variation of the current design that has been synthesized, placed, and routed. However, you can also use a checkpoint that is only partially placed or routed. See this link in the Vivado Design Suite User Guide: Implementation (UG904) for more information.
  • Strategy: Specifies the predefined Vivado® Design Suite strategies, or lets you create your own strategy by modifying the options for the selected strategy. Strategies are predefined configurations of the Tcl commands that make up the synthesis or implementation runs. You can modify the individual settings or options for a strategy to meet your specific objectives. For more information on the synthesis and implementation commands, and available options and directives, refer to the following commands in the Vivado Design Suite Tcl Command Reference Guide (UG835):
    • synth_design
    • opt_design
    • power_opt_design
    • place_design
    • phys_opt_design
    • route_design
    • write_bitstream
    Tip: A modified strategy is indicated by an asterisk (*) in the strategy name. You can click the Save Strategy As button () to save the strategy as a new user-defined strategy.
  • Description: Provides a brief summary of the goal of the selected strategy. You can define the description when saving user-defined strategies.
  • Options: Shows a table view of the options available in the selected implementation strategy. Select an option in the table to display a description of the option. In the Options table, the implementation strategy is broken down into the various steps that make up the optimization, placement, and routing process. See this link in the Vivado Design Suite User Guide: Implementation (UG904) for more information. You can change some features of the selected strategy by changing the settings in the Options table. You can use the More Options field for each command in the Options table to add any additional command options.
    Important: Many implementation strategies use the -directive argument to configure the implementation. Changing the settings in the Options table is not recommended for strategies using non-default directives as this can result in errors during implementation.
