Define Module

The fields of the Define Module dialog box vary depending on the language of the module or entity you are defining: Verilog, SystemVerilog, or VHDL. The language is determined by the File Type setting in the Create Source File dialog box.

  • Module name/Entity name: Automatically populates with the File Name specified in the Create Source File dialog box, without the file extension. However, the module name does not need to match the file name, and you can edit in this field.
  • Architecture name: Specifies the architecture of the VHDL module being defined. The default is behavioral, but you can change this setting. The architecture provides the internal view of the circuit and describes the circuit behavior or structure.
    Tip: The specified architecture is declared in the module definition, but you must provide the details of the architecture.
  • IO Port Definitions: Lists the currently defined ports of the module. The table displays Port Name, Direction, Bus, MSB, and LSB for each port. You can edit each of these fields for a port in the table.
    • Port Name: Specifies the name for the port. You can type directly in the text field. Note that ports without name definitions are not written to the module.
    • Direction: Specifies the port as an Input, Output, or bidirectional (InOut) port.
    • Bus: Indicates that the port is a bus port. Define the width of the bus using the MSB and LSB options.
    • MSB: Defines the number of the most significant bit (MSB). This combines with the LSB field to determine the width of the bus being defined.
    • LSB: Defines the number of the least significant bit (LSB).
    You can add new ports to the module, remove existing ports, and reorder ports as follows:
    • Add: The green plus sign () adds a new port definition to the table. You can also add a new row to the table by clicking in an empty row of the table.
    • Remove: The red minus sign () lets you remove ports from the table. This command is available when one or more rows of the table are selected. Selected ports are removed from the module definition.
    • Move Up: The blue Up arrow button () lets you move selected ports up in the listed order. This command is available when one or more rows of the table are selected.
    • Move Down: The blue Down arrow button () lets you move selected ports down in the listed order. This command is available when one or more rows of the table are selected.
  • New Source Files: This section lists multiple new source files that need module definitions. You can select any of the files in this list to define the modules.
    Tip: The New Source Files section only appears when multiple new source files are specified by the Add Sources dialog box or the Add Sources wizard. It is not available to define a single module.
When you click OK, the new module or entity definition is written to its source file, and the file is added to the current project.
Important: If multiple source files are listed in the New Source Files section, all of the new module definitions are written at the same time. Complete the definition of all modules for all listed source files before clicking OK, or you will be required to manually define any undefined modules.
