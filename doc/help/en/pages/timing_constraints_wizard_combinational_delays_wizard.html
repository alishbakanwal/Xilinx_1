Combinational Delays

Understanding Combinational Delays

Figure: Example of a combinational path:



The input signal din[15] propagates through several levels of combinatorial logic and reaches the output port combOut without being latched by a register. The combinatorial delay through the device corresponds to the period of a virtual clock defined by the wizard minus the input and output delays provided by the user. If several delay constraints are defined on a port, only the most challenging one is used by the timer. For example: 

create_clock -period 10.0 -name virtual_clock
set_input_delay -clock virtual_clock 1.0 [get_ports din[15]] -min
set_input_delay -clock virtual_clock 3.0 [get_ports din[15]] -max
set_output_delay -clock virtual_clock 2.0 [get_ports combOut] -min
set_output_delay -clock virtual_clock 4.0 [get_ports combOut] -max

Alternatively, you can constrain combinational paths with set_max_delay and set_min_delay commands outside the Timing Constraints Wizard.

Understanding the Combinational Delays Page

  • Recommended Constraints: The Timing Constraints wizard recommends combinational delays for paths that propagate through combinatorial logic, from input to output, without passing through sequential logic. The wizard recommends a virtual clock constraint as well as delays for the input and output ports on the path.
  • Tcl Command Preview: Displays the Tcl commands used to define the combinational delays. These are the constraints that will be added to the design.
  • Existing Constraints: Displays currently defined constraints.

Each of the preceding sections also has a toolbar menu that provides access to one or more of the following commands:

Search Displays a text entry field to enter a search string for the listed constraints.
Select All Selects all constraints.
Edit selected row Opens the Combinational Constraints dialog box to define input and output delays. You can also edit these attributes by clicking directly on the field in the Recommended Constraints list.
Report Clock Networks Runs the Report Clock Networks command to generate a new clock network report.
Tip: The clock networks are reported in the Clock Networks window.
