Of Objects - Find

This dialog box lets you define a query to locate netlist and device objects using specified properties on the objects. The results of this query define the -of_objects option for the Find dialog box.
Tip: You can use this dialog box in a nested manner to find objects related to other objects. For example, you can find the package pins of specific I/O ports of specific I/O banks.
  • Find: Specifies the type of first class objects to find in the current design. The objects are listed as Netlist objects and Device objects. The available list of objects is narrowed by the parent dialog box.
  • Criteria: Provides three unlabeled drop-down menus listing Property, Operator, and Value sets to filter objects in the current design.
    • Property: The listed properties depend on the object type you specified in the Find field.
    • Operator: The listed operators depend on the specified Property and can be a string operator, a boolean function, or a numerical operator.
    • Value: The listed values are defined from the known or available values for the specified Property or can be a string or value that you enter directly. You can use the question mark (?) and asterisk (*) characters as wildcards in the Value field.
    • Add a new criterion (): Adds a new row to the Criteria section.
    • Remove this criterion (): Removes the selected row from the Criteria section.
    Tip: When there are multiple rows in the Criteria section, you must also specify the logical relationship between the properties as either AND or OR. AND returns objects that match all of the specified property values. OR returns objects that match any of the specified property values.
  • Regular Expressions: Defines the Value field as a regular expression (regexp) search pattern. See this link for more information on regular expression syntax.
  • Ignore case: Applies to regular expressions searches only. Search expressions are generally case-sensitive.
  • Search hierarchically: Searches through each level of the design hierarchy. The default search occurs on the current instance, or the top-level of the design hierarchy. The availability of this option is dependent on the type of object specified in the Find field.
  • Of objects: Limits the search to a list of selected objects. For example, find the package pins of specific I/O ports, or find the nets on specific cell pins. Use the browse button () to open another Of Objects - Find dialog box, or manually type in the appropriate Tcl command.
  • Command: Displays the Tcl command that is passed to the parent dialog box when you press the OK button.

Click OK to pass the Tcl command to the parent dialog box.
