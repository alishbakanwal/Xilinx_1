Edit Configurations

This page of the wizard lets you define PR Configurations that assign Reconfigurable Modules (RMs) to Partition Definitions (partitionDefs), and combine with the static logic of the core design to create a unique configuration of the design. This PR Configuration can then be implemented, and the bitstream generated.

The page displays a Configurations table letting you manage the project. In the center background of the table is a command link to automatically create configurations. Clicking this link is recommended, and automatically sets up all the configurations needed to map the partitionDefs and RMs in the design.

The table lists the PR configurations by name, and displays the partitionDefs in the project. For each configuration you can you specify an RM to assign to each partitionDef. The Configurations table also has a toolbar with the following commands:

  • Add (): Add a new PR Configuration to the current project. This command opens a small dialog box to specify the name of the configuration. The default name is config_<#>, numbered sequentially.
    Tip: You can edit the name of an existing configuration in the table by clicking the pencil icon () next to the name.
  • Remove (): Remove the selected configuration from the current project.
Click Next to proceed to the Edit Configuration Runs page of the Partial Reconfiguration wizard.
Note: The commands of the Partial Reconfiguration wizard are not run after each page, but are saved until all the pages have been visited and the whole setup is complete. Click Skip to Finish to go to the Partial Reconfiguration Summary page, and complete the setup without visiting all pages.
