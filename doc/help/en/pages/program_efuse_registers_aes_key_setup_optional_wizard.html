AES Key Setup (Optional)

For 7 series devices, this page of the Program eFUSE Registers wizard lets you program the FUSE_KEY and FUSE_USER registers of the current device. The Advanced Encryption Standard (AES) encryption key will be read from the specified key file and written to the 256-bit FUSE_KEY registers, to enable decrypting the bitstream when programming, verifying, or reading back the device. You can also specify HEX values to assign to the 32-bit FUSE_USER registers.

The implemented design must have encryption properties assigned prior to generating the bitstream with the write_bitstream command. You can add ENCRYPTION properties to the design using the Encryption page of the Edit Device Properties dialog box in the Vivado IDE.

  • Enable AES key programming:
    • AES key file (.nky): Specify the file name and path of the AES encryption key file.
      Tip: By default, this file is written to the implementation run directory at the time an encrypted bitstream is generated.
    • AES key: Displays the AES encryption key value that is read from the file specified above.
    • User Bits [7:0]: For 7 series devices you must program the lower 8-bits of the USER registers at the same time as the AES key, if you are going to program them at all. You cannot program these registers separately.
  • Enable Programming of User Bits [31:8]: This enables a text field to let you enter a hexadecimal value for the top 24-bits of the FUSE_USER register. The USER eFUSEs bits are provided to let you specify your own 32-bit ID for the design or revision on the device. The lower eight FUSE_USER bits are programmed at the same time as the 256-bit Advanced Encryption Engine (AES) key. The upper 24 user bits can be programmed concurrently with the AES key or at a later time
Programming a device with an encrypted bitstream is a two-step process that requires first programming the encryption key into the BBR or eFUSE registers, and then programming the encrypted bitstream into the device: 
program_hw_devices -key {efuse} [current_hw_device]
program_hw_device [current_hw_device]
