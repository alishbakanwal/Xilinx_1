Logically Exclusive Clock Groups with No Interaction

Understanding Logically Exclusive Clock Groups

Figure: Example of a logically exclusive clock group with no interaction:



The clocks clkA and clkB do not share any logical path between each other nor with any sequential cell driven by the multiplexer BUFGMUX. Therefore, it is safe to directly specify a logically exclusive clock group between clkA and clkB, because this does not disable a real timing path. For example: 

set_clock_groups -logically_exclusive -group clkA -group clkB

If the design contains logic that modifies the clocks in the fanout of BUFGMUX, it is also safe to apply the clock group constraint to the generated clocks. For example: 

set_clock_groups -logically_exclusive \
      -group [get_clocks -include_generated_clock clkA] \
      -group [get_clocks -include_generated_clock clkB]

Understanding the Logically Exclusive Clock Groups with No Interaction Page

  • Recommended Constraints: The Timing Constraints wizard identifies clocks that are defined on different source points but share part of their clock tree, due to a multiplexer for example. When these clocks do not share timing paths, the clocks have no interaction other than the logic connected to their shared clock tree.
  • Tcl Command Preview: Displays the set_clock_groups commands used to define the logically exclusive groups. These are the constraints that will be added to the design.
  • Existing Constraints: Displays currently defined constraints.

Each of the preceding sections also has a toolbar menu that provides access to one or more of the following commands:

Search Displays a text entry field to enter a search string.
Select All Selects all recommended constraints.
Clock Networks Displays a sub-menu to provide access to the Report Clock Networks command to generate a new clock network report and the View Clock Networks command to open an existing report.
Tip: The clock networks are reported in the Clock Networks window.
