Check Timing

This command checks the design elements of ports, pins, and paths, against the timing constraints in the current design. Use this command to identify possible problems with design data and timing constraints before running the Report Timing command.

  • Results name: Specifies the name of the report to create.
  • Timing Checks to Perform: Presents a list of checks to perform that you can enable or disable as needed. Refer to the documentation referenced below for a complete description of the timing checks.
  • Write results to file: Writes the results of the Check Timing command into the specified file. The default is to Overwrite an existing file of the same name. Use Append to append the new report to the specified file.
  • Shows detailed information about potential problems: Return more detailed results from the checks that are run, including details of any problems found.
  • Ignore command errors (quiet mode): Execute the command quietly, returning no messages from the command.
  • Command: Displays the Tcl command with all of the specified options, which is used to generate the report.
  • Open in a new tab: Opens the Check Timing results in a new tab of the Timing Results window in the Vivado IDE. If this checkbox is not enabled, the new report overwrites the last report that was opened.
  • Open in Timing Analysis layout: Changes the current layout view of the Vivado IDE to the Timing Analysis layout view. See this link in the Vivado Design Suite User Guide: Using the Vivado IDE (UG893) for more information.
