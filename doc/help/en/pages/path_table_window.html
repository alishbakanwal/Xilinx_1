Timing Path Report

The Timing Path Report window displays a detailed timing report of a specific path, including the start point, detailed datapath, and end point that define the path. The source clock and destination clock paths are also reported. The information is provided to help you understand what is causing a timing violation on the path. The window displays a Summary on the path, including the slack, timing requirement, total path delay, delay attributed to logic on the path, estimated or actual route delay, and clock path skew.

The Timing Path Report window has links to the details of specific values, such as Slack or Clock Path Skew. Click the value to see how it was calculated. Design objects on the timing path are also links that you can click to select and view the object in other windows, such as the Netlist or Device windows.

A complete discussion of the various elements of the Timing Path Report, and how to read it, can be found at this link in the Vivado® Design Suite User Guide: Design Analysis and Closure Techniques (UG906).
