Archive Project

The Archive Project dialog box lets you create a ZIP file archive of the current project. The ZIP file can be used for revision control, to share with team members, or just to relocate the design. The Archive Project command examines the hierarchy of the design, copies any remote source files and include files into the local directories, copies the design constraints, and optionally copies the results of the synthesis, simulation, and implementation runs.

  • Archive name: Specify the name of the project archive to create.
  • Archive location: Specify the directory location for the project archive.
  • Archive will be created at: This field simply displays the file name and path of the project archive that will be created.
  • Temporary location: Specifies the temp file space for copying and managing files and folders while creating the project archive. The temporary location will be emptied when the archive process is complete. By default, the Vivado tool creates a temporary directory inside of the current working directory.
    Tip: This option is for Windows only, and can be used to overcome file naming limitations on that operating system.
  • Include configuration settings: Add any initialization Tcl commands (init.tcl) to the archive file under the config_settings folder of the archive.
  • Include run results: Include the results of any synthesis or implementation runs. This option can significantly increase the size of the project archive. However, it also preserves the synthesis netlist and implemented design if necessary.

Click OK to create the project archive.
