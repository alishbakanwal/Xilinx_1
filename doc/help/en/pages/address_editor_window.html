Address Editor Window

The Address Editor window displays a tree view of master interfaces and connected peripherals (slaves) that can be addressed by that master. The memory address ranges for the slave and master interfaces in the block design are automatically assigned, but you can edit the addresses in this window.

The table view of the Address Editor displays the following columns:
  • Cell: Lists the address space master and the peripheral interfaces addressed by that master.
  • Slave Interface: Lists the name of the slave interface pin on the peripheral instance.
  • Base Name: Displays Mem or Reg type of memory.
  • Offset Address: Lists the memory offset from the start of the address block.
  • Range: Lists the size or amount of memory to allocate to the address space.
  • High Address: Displays the address calculated based on the offset and range.
You can manually assign address offsets and ranges or use the Auto Assign Address command ().
Tip: When multiple masters address the same slave, you can exclude the slave from a master's addressable memory space.
