Configure Implementation Runs

The Configure Implementation Runs page lets you add and configure new implementation runs to the current project. The page opens with a default implementation run to configure.

  • Add (): Add a new implementation run to the list of runs being configured.
  • Remove (): Remove one or more implementation runs from the list of runs to configure. This button is active when more than one implementation run is defined in the list of runs, and one or more of those runs is selected.
    Tip: The last run cannot be removed. Select Cancel if no runs are needed.
  • Name: The name of the implementation run to create and configure. A default name is provided based on the name of the currently defined implementation runs, but you can edit the name as needed.
  • Synth Name: The name of the synthesis run to associate with the implementation run.
  • Constraint Set: Specify the constraint set to use for the implementation run. This determines both the constraints that are used to drive placement and routing and the constraint set to write to after implementation is complete. You must select from a list of existing constraint sets. If you want to use a new constraint set, you must define it using the Edit Constraints Sets command from the Sources window.
  • Part: Specify the target part or platform board for the design. The default part for the current project is automatically selected. Click the Part field to select a new target part or board from the Select Device dialog box.
  • Strategy: Specify the implementation strategy to use for the run. You can define your own strategy for the run using Tools > Options > Strategies from the main menu.
  • Make Active: Makes the selected run the active run, which launches automatically when the Run Implementation command is used. The results for the active run display in the Messages, Compilation, Reports, and Project Summary windows.
