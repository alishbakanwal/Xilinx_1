Program Device

This dialog box lets you specify a bitstream file and probes file to download to the current device. The probes file corresponds to the debug cores contained in the design.

After connecting to the target hardware, to program the device you must associate the bitstream file (.bit, .rbt, .bin) from the implemented design with the device. The bitstream file is created after implementation using the Flow > Generate Bitstream command, or the write_bitstream Tcl command. This dialog box sets the PROGRAM.FILE property to define the bitstream, and the PROBES.FILE property on the current device.

  • Bitstream file: Specify the file name and path of the bitstream file associated with the current design.
    Important: If the bitstream file is encrypted to protect the design IP, then you must first load an encryption key into the BBR or the eFUSE key register to enable the device to decrypt the bitstream. This is done through the Program BBR Key dialog box, or the Program Efuse Registers wizard. Refer to this link in the Vivado Design Suite User Guide: Programming and Debug (UG908) for more information.
  • Debug probes file: Specify the file name and path of the probes file associated with the debug cores in the current design.
  • Enable end of startup check: Checks the startup status of the device when programming has completed. When enabled, the Hardware Manager will return an error if programming fails.

Click OK to set the PROGRAM.FILE and PROBES.FILE properties on the current device, and program the device with the specified bitstream.
