Write Configuration Memory File

The Vivado device programmer can indirectly program flash-based configuration memory devices via JTAG, using a special configuration memory file (cfgmem) created with the Generate Configuration Memory File command in the Vivado Hardware Manager. This dialog box lets you configure options related to writing the cfgmem file.

  • Format: Specify the format for the configuration memory file. The default format is MCS.
  • Memory Part: Specify the configuration memory device.
    Tip: This field is automatically populated by the configuration memory device associated with the current hardware device through the Add Configuration Memory Device dialog box. However, you can also specify a different configuration memory device if needed.
  • Custom Memory Size: Specify the size limit in MBytes of the configuration memory device that is being targeted. The size must be specified as a power of 2.
  • Filename: Specify the name of the file to create. The file suffix is automatically determined by the Format chosen for the file.
  • Options
    • Interface: Specify the interface format used to program the flash memory device. The default interface is SMAPx8.
      Important: The interface format is critical to properly programming the flash memory device, and must match the format supported by the selected Memory Part.
    • Load bitstream files: A checkbox to include one or more bitstream files as part of the cfgmem file.
      • Start address: Specify the starting address for writing the bit pattern from the bitstream. Specified as a hexadecimal value.
      • Direction: Add the bits by counting up from the starting address, or by counting down from the starting address.
      • Bitfile: Specify the file and path of the bitstream file to load.
        Tip: When a file is specified, an Add () and Remove () button is also displayed to let you add additional bitstreams, or remove previously added files. Loading multiple bitstream files causes the files to be concatenated in a daisy chain.
    • Load data files: A checkbox to include one or more data files as part of the cfgmem file. Data files are added to the flash memory device as is, with no additional formatting.
      • Start address: Specify the starting address for writing the bit pattern from the data file. Specified as a hexadecimal value.
      • Direction: Add data bits by counting up from the starting address, or by counting down from the starting address.
      • Datafile: Specify the file and path of the data file to load.
        Tip: When a file is specified, an Add () and Remove () button is also displayed to let you add additional data files, or remove previously added files. Loading multiple data files causes the files to be concatenated in a daisy chain.
    • Write checksum: Calculate a 32-bit checksum for the cfgmem file. Use this option to verify that correct data was programmed into the flash memory. The checksum data is written to the log file (.prm) that is created at the same time as the cfgmem file.
    • Disable bit swapping: Disable the default bit swapping for bytes in the loaded bitstream files.
      Important: By default the bits within a byte are bit-swapped, or mirrored, compared to bytes in the original input BIT files.
    • Overwrite: Overwrite an existing file.

Click OK to write the cfgmem file.
