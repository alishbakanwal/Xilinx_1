Report Clock Interaction

This dialog box lets you generate a report of the interaction between clocks in the design.

The dialog box includes the following tabs to configure the clock interaction report:
  • Options
  • Timer Settings

Options Tab

The Options tab provides the following fields to configure the output of the report:

  • Path delay type: Specifies the type of delay to analyze when running the Clock Interaction report. Max delay is for setup analysis of timing paths. Min delay is for hold analysis. Choose min_max delay for both setup and hold analysis.
  • Significant digits: Configures the significant digits for values reported in the Clock Interaction report. The default is 3.
  • Write results to file: Writes the results of the Clock Interaction report into the specified file. The default is to Overwrite an existing file of the same name. Use Append to append the new report to the specified file.

Timer Settings Tab

The Timer Settings tab provides the following fields to configure the timing engine prior to generating the report:

  • Interconnect: Specifies the interconnect delays to use in the timing analysis. You can use the full routed delays from an implemented design, use estimated delays in the synthesized or implemented design, or use no interconnect delays to examine just the logic delays.
  • Speed grade: Changes the speed grade of the device during analysis, without actually changing the target part.
  • Multi-corner configuration: Configures the Slow and Fast timing corners in the current design. Variables in the manufacturing process, voltage, and temperature (PVT) determine the delay across the device and combine to define a timing corner. This option relates to the config_timing_corners Tcl command.
    • Slow: Specifies the type of delay to analyze for the Slow corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
    • Fast: Specifies the type of delay to analyze for the Fast corner. Use max delay for setup analysis, min delay for hold analysis, or min_max delay for both setup and hold analysis. Use none to exclude this corner from timing analysis.
  • Disable flight delays: Ignores the package delay in I/O delay calculations. The flight delay is the package delay that occurs between the package pin and the die pad. This option relates to the config_timing_analysis Tcl command.

Common Options

The common options do not appear in a tab but are present on the dialog box at all times.

  • Results name: Specifies the name to apply to the report in the Clock Interaction window in the Vivado® IDE.
  • Command: Displays the Tcl command with all of the specified options, which is used to generate the report.
  • Open in a new tab: Opens the report in a new tab of the Clock Interaction window in the Vivado IDE. If this checkbox is not enabled, the new report overwrites the last report that was opened.

Click OK to run the displayed Tcl command and generate the report.
