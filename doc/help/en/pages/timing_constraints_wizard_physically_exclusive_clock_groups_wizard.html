Physically Exclusive Clock Groups

Understanding Physically Exclusive Clock Groups

Figure: Example of a physically exclusive clock group:



In this example, two primary clocks, clkA and clkB, are defined on the same input port clk. Because clkA and clkB cannot be active at a same time, a physically exclusive clock group is defined to prevent the analysis between the two clocks. 

set_clock_groups -physically_exclusive -group clkA -group clkB

In case the design contains logic that modifies the clocks in the fanout of BUFG, it is safe to also apply the clock group constraint to the generated clocks. For example: 

set_clock_groups -physically_exclusive \
      -group [get_clocks -include_generated_clock clkA] \
      -group [get_clocks -include_generated_clock clkB]

Understanding the Physically Exclusive Clock Groups Page

  • Recommended Constraints: The Timing Constraints wizard identifies clocks that share the same source point and propagate on the same clock tree. Then the wizard recommends a clock groups constraint to prevent unnecessary timing analysis on the clock domain crossing paths.
  • Tcl Command Preview: Displays the set_clock_groups commands used to define the physically exclusive groups. These are the constraints that will be added to the design.
  • Existing Constraints: Displays currently defined constraints.

Each of the preceding sections also has a toolbar menu that provides access to one or more of the following commands:

Search Displays a text entry field to enter a search string.
Select All Selects all recommended constraints.
Clock Networks Displays a sub-menu to provide access to the Report Clock Networks command to generate a new clock network report and the View Clock Networks command to open an existing report.
